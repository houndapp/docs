Mailpit polling: find the magic-link email the backend sent and pull out its sign-in code. Moved here (from tools/houndbot) so both houndbot and the scenario suite share one implementation.

Wire shapes verified empirically against a running axllent/mailpit (v1 API): GET /api/v1/search?query=to:"<email>" returns {"messages": [{"ID": "...", ...}], ...} (search results are metadata only — no body); GET /api/v1/message/{ID} returns the full message, whose Text field is the plain-text part. The backend’s magic-link email is multipart/alternative (services/backend/src/api/auth/magic.rs), so Text is exactly the part carrying the unadorned code line (the HTML part wraps the code in <b>, which would need stripping — Text avoids that entirely).

CODE_LINE_RE 🔒
Anchors on the plain-text body’s code line (services/backend/src/api/auth/magic.rs): "Or enter this code in the app: {code}". The same email’s link also embeds ?code=..., but this prefix is unambiguous and doesn’t require URL parsing. Alphabet excludes 0/O/1/I, so [A-Z2-9]{12} matches the code exactly.

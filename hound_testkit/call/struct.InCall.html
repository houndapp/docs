Struct InCall

pub struct InCall {
    participant: Participant,
    call_rx: Receiver<CallView>,
    stats_rx: Receiver<StatsSnapshot>,
}
Expand description

A joined call. Dropping this without calling InCall::leave leaves the call running server-side until the participant’s HoundClient is dropped or another join supersedes it — tests should always leave() explicitly.

Holds the watch_call/watch_stats receivers from before the join (see crate::participant::Participant::join): a receiver that exists from the start can never miss a publish, whatever the sender’s send-vs-send_replace discipline.

pub async fn expect_no_audio_growth(&self, quiet_for: Duration) -> Result<()>

Wait quiet_for, asserting inbound audio frames never grew — the “muted means silence” half of the mute assertion. A negative expectation, but still watch-driven: it wakes on every stats tick rather than blind-sleeping the whole window before checking once.

pub async fn expect_steady_audio( &self, min_peers: u32, window: Duration, ) -> Result<()>

The warmup-baseline audio-health gate — see [hound_client::call_health] for the shared monitor (the same one houndbot gates its verdict on). Observes for window, anchoring the starve-counter baseline once min_peers have been present for hound_client::call_health::WARMUP, then fails if any starvation counter grew past that baseline.

Source

pub async fn expect_sfu_node( &self, node_id_prefix: &str, deadline: Option<Duration>, ) -> Result<()>

Wait until this call’s transport session identifies itself as SFU node node_id_prefix (compare against crate::sfu_process::short_prefix’s output). The direct signal that a migration — the initial voice/join or a SfuMigrate- triggered re-establish — actually landed there: driven by the same set_session_identity write hound-client performs on every establish (crates/app/client/src/call/session.rs), so this observes the real client-visible outcome rather than just what the backend would currently route a fresh voice/join to.

Source

pub async fn expect_sfu_node_not( &self, node_id_prefix: &str, deadline: Option<Duration>, ) -> Result<()>

Wait until this call’s transport session has moved OFF SFU node node_id_prefix — the negative counterpart to Self::expect_sfu_node, used to observe a migrate-away (e.g. a node that started draining) without needing to know in advance which node it lands on next.

Source

pub async fn expect_audio_recovers( &self, min_peers: u32, within: Duration, ) -> Result<()>

After an EXPECTED disruption (e.g. an SFU migration’s brief media gap), poll for a clean Self::expect_steady_audio window within within. Each attempt re-anchors its own warmup baseline from the counter totals current at that attempt’s start, so a disruption caught mid-window doesn’t poison the next attempt. The invariant under test is BOUNDED recovery, not zero interruption — a single zero-tolerance starve gate spanning the disruption itself would always fail on an expected, transient blip.

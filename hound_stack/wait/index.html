Deadline-bounded polling — no bare sleeps beyond the poll interval (see AGENTS.md’s “Test determinism”). Every waiter here bounds total wall time and returns a descriptive eyre error on timeout instead of hanging or (worse) racing a downstream timeout with a less useful cause.

wait_for_http_or_exit
Poll an HTTP endpoint for reachability, bailing early if child exits first — mirrors the shell scripts’ kill -0 $PID || exit 2 guard inside their wait loops (an early-dead process is a far more useful cause than a plain timeout).
wait_for_log_line
Poll log_path for needle, bailing early if child exits first — the same early-exit guard as wait_for_http_or_exit, used for both the backend log’s SFU-registration line and the bot log’s “joined call” line.
wait_http_reachable
Any HTTP response (even a 4xx/5xx) proves the server is up — matches the shell scripts’ plain curl -s -o /dev/null check, which counts an auth-gated 401 as reachable (curl -f would not). Soft: callers that don’t have a hard requirement on reachability (mailpit) just ignore the Err, exactly like the shell scripts’ unchecked wait loop.

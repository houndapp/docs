Module compose

Expand description

Compose driver: resolves the same docker compose (plugin) → docker-compose (standalone) → mise-pinned fallback chain the shell harnesses used, so a CI host that ships bare docker (no plugin, no standalone binary — the X64 runner) still works without a host install.

Noise policy: lifecycle invocations (up, restart, teardown down) append their full output to a compose.log artifact and print nothing on success — the harness console gets one progress line per step, and a failure surfaces the log tail inline. The manual hound-stack down escape hatch uses Compose::down_volumes_inherit instead, since an operator invoking it directly wants to watch it happen.

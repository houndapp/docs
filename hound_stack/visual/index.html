hound-stack visual: bring the stack up, bootstrap the shared community via [hound_testkit::Env] (no bash curl/jq re-implementation — see its module doc), spawn one houndbot --testcard peer into the voice call, run the real-app Flutter integration test, then snapdiff compare against the accepted baselines. Mirrors the retired ui/integration_test/visual.sh.

All build phases (flutter debug prebuild, houndbot, the snapdiff report UI, the snapdiff binary) run quietly with output in build.log; the bot and Flutter test write their own runtime artifacts, and snapdiff-summary.json holds exactly snapdiff’s stdout JSON.

Exit code contract (unchanged — nightly.yaml’s visual job keys off this): 0 — ran to completion, every comparison passed (or was only new). 1 — ran to completion, at least one REAL visual diff — not an infra problem, just something to look at (or accept) in the report. 2 — infra/setup failure: the stack never came up, the bot never joined, or the integration test produced zero screenshots.

Module tee

Run a child process, teeing its stdout/stderr to both this process’s own stdout/stderr and a log file — the Rust equivalent of the shell scripts’ cmd 2>&1 | tee "$LOG".

Each stream keeps its own internal ordering; stdout and stderr interleave independently rather than through one merged pipe (an acceptable divergence from a literal 2>&1 | tee — the tools this wraps (nextest, flutter test, snapdiff) write their meaningful output to stdout).

Functions§

run_tee
Run cmd, teeing its stdout+stderr to log_path, and return its exit status. A nonzero exit is NOT an Err here — the caller propagates that as this stack’s own process exit code, exactly like the shell scripts’ set -o pipefail did.
run_tee_stdout
Like run_tee, but only stdout is teed to log_path (stderr passes straight through to the console) — for a tool whose stdout is a machine contract (snapdiff’s JSON summary) that must not be polluted by progress noise. The shell equivalent: cmd | tee "$LOG" without 2>&1.
tee_stream 🔒
Reads raw bytes (not lines-as-UTF8: BufRead::read_line would abort the whole stream on a single invalid-UTF8 byte, e.g. from an ANSI-heavy build tool) up to each newline and fans them out to the console + the shared log file.

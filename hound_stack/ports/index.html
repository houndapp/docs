Port preflight: fail fast if this run’s fixed host ports are already taken by a leftover/concurrent stack, instead of racing it through compose up/backend-startup timeouts.

Unlike the shell scripts’ lsof-based check (silently skipped when lsof isn’t installed, and scoped to LISTEN sockets only), this binds each port directly with SO_REUSEADDR set — the same “can we actually use this port” question, without an external tool dependency, and without false positives from a recently-closed socket still in TIME_WAIT.

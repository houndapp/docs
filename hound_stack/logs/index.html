Module logs 

Source
Expand description

Log-file helpers for the harness’s noise policy: every subprocess writes its full output to a per-concern artifact file, the console gets one progress line per lifecycle step, and a failure surfaces the relevant log’s tail inline — never the whole stream.

Functions§

append_section
Open log for appending and write a ── title ── section header, so one artifact file (e.g. build.log) can hold several steps’ output while staying navigable.
print_tail
Print log’s tail to stderr under a labeled header — the inline diagnostic attached to a failure, pointing at (not replacing) the full artifact file.
tail
The last lines lines of log, or empty if unreadable.

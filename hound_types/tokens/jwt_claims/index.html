Module jwt_claims 

Source
Expand description

Steam-link / magic-register ceremony claims.

Both are short-lived JWTs proving the bearer just completed an external identity check (Steam auth, an emailed magic-link code) for an identity that has no account yet, redeemed against the matching */register endpoint. Stateless on purpose — no in-memory challenge map, so it survives replicas. Backend-only today (the SFU never mints/verifies these); they live here anyway so every token format the system uses has one home.

encode/decode_* take the raw HMAC secret rather than the backend’s Jwt wrapper (which also owns session-token revocation, out of scope here) — callers pass the same secret bytes Jwt was constructed with.

Structs§

MagicRegisterClaims
Claims for a short-lived “magic register” token: proof that the bearer redeemed a magic-link code for email moments ago, minted when the email has no account yet. Redeemed against /auth/magic/register. Mirrors SteamLinkClaims exactly.
SteamLinkClaims
Claims for a short-lived “steam link” token: proof that the bearer completed Steam authentication for steam_id moments ago, minted when the Steam ID has no account yet. Redeemed against /auth/steam/register or /auth/identity/steam/link.

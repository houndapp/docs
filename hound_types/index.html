§hound-types

The backend<->client wire contract, defined once so both sides compile the same serde shape instead of hand-mirroring it, plus the token mint/verify formulas shared across service boundaries.

  • dto — HTTP/WS request/response DTOs. utoipa::ToSchema derives are gated behind the utoipa feature: the backend enables it to keep these types in its OpenAPI components without re-declaring the shapes; the client and the SFU (neither has an OpenAPI surface) leave it off.
  • tokens — short-lived token formats that cross a service boundary: the relay token (tokens::relay, the backend mints it, the SFU verifies it — always available) and the steam-link/magic-register ceremony claims (gated behind jwt-claims, backend-only today).

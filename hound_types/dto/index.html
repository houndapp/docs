Backend<->client wire DTOs. Field sets and serde attributes are byte-identical to what the backend used to declare locally (and the client used to hand-mirror) — this module is the single declaration both sides now share.

Not every backend wire type lives here: types that embed a DB-entity enum (sea_orm_active_enums::Permission/InstancePermission) or that the backend deliberately keeps non-generic for OpenAPI schema-naming reasons (FinishLoginResponse, ProfileResponse — see services/backend/src/api/auth.rs) stay backend-local; see the crate extraction report for the full list of deliberate exclusions.

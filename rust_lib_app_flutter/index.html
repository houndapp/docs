§hound-frb — the Flutter ⇄ Rust boundary

The flutter_rust_bridge binding over [hound_client]: a thin projection layer, not a second brain. All client logic — auth, REST, realtime, the call engine — lives in hound-client (UI-free, FFI-free); this crate only (a) exposes its commands as async fns returning Result<T, HoundError>, (b) mirrors its observable snapshots to Dart Streams via StreamSink, and (c) wires the video texture handshake to hound-render (the platform-uniform render facade; every render-side platform dependency lives there).

Because the logic is in hound-client, a second frontend (SwiftUI over UniFFI, a C ABI, …) is a parallel thin binding over the same core — not a rewrite. flutter_rust_bridge_codegen generate turns the api module into the Dart bindings + the frb_generated glue; the Flutter app (ui/) links this crate as a cdylib/staticlib.

The adapter from hound-passkey’s platform keyring store to hound-client’s SessionStore seam.

All the real storage lives in hound_passkey::storage (the storage feature): backend selection per OS (macOS Keychain / Linux Secret Service / mock elsewhere), the atomic one-blob token pair, and the Linux hop off the tokio runtime. hound-client never links a platform keyring (it injects persistence via the SessionStore trait), so this crate supplies the impl — a pure delegation plus the PasskeyError → HoundError mapping.

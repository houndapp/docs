A keyring-backed SessionStore for the FRB layer.

hound-client never links a platform keyring (it injects persistence via the SessionStore trait); this is the host-side impl the Flutter app supplies. It persists the access + refresh token pair as one JSON blob under a single keychain credential, so a refresh (which rotates BOTH tokens) can never leave a torn pair.

Backend per platform is selected at compile time: macOS uses the login Keychain (apple-native-keyring-store/keychain). Other targets get their backend in a parallel [target.*] table once they come online; until then this store is macOS-only (a build error elsewhere, not a silent fallback).

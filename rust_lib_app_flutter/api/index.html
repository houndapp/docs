Expand description

The bridge API surface the Flutter UI binds against.

Two halves:

  1. The app surface: an init_app global initialiser + thin async command wrappers over hound_client::HoundClient + StreamSink forwarders that mirror the facade’s watch snapshots to Dart Streams. Every command returns Result<T, HoundError>; HoundError mirrors to a Dart sealed class (so the UI matches on variants, not strings).
  2. The video/camera surface: the zero-copy texture handshake + live local-camera capture.

Projection rule (mirror by default; DTO only on a genuine reshape): the facade’s snapshots are hound-client’s own view-models. Rather than re-declare each one, we #[frb(mirror(T))] it — codegen emits the matching Dart class with zero field duplication, so a new field on the core type flows to Dart for free. A hand-written struct + project_* fn is written only where the shape genuinely differs from the core type:

  • LevelsDto — LevelsView.peers is a HashMap (FRB mirrors a Vec cleanly).
  • CommunityNode — the sidebar joins a community with its spaces (a composed reshape built from the mirrored Community/Space, still no duplication).
  • RegisterStartDto/LoginStartDto — command results, not snapshots.

Ids cross as uuid::Uuid (Dart UuidValue) via the FRB uuid feature.

Statics§

CLIENT 🔒
The process-wide HoundClient, set once by init_app. FRB free functions fetch it; calling a command before init_app is a usage error (surfaced as HoundError::InvalidState).
LOG_SINK 🔒
The live Dart log sink. Swapped on each init_log (e.g. a hot restart hands us a fresh sink); the tracing layer — installed exactly once — reads this per event, so it always forwards to the current sink and never to a dead one.

Functions§

attach_video_texture
Attach a stream key (e.g. “cam:42”, “self”, “screen:7”) to a native GPU texture handle the Flutter plugin registered, sized width×height. native_texture is an opaque pointer (an IOSurfaceRef / id<MTLTexture> on Apple) the Rust renderer will draw into. texture_id is the Flutter texture id the same createTexture returned: macOS’s shared render loop marks it “dirty” once it draws a new frame, and the plugin’s ONE shared CVDisplayLink re-pulls only dirty ids (so copyPixelBuffer runs at video-fps, not display refresh). The plugin allocates the buffer at this SAME width×height (the decoded frame’s size, so the render target matches the source aspect with no crop/stretch); Dart recreates + re-attaches at a new size when the decoded resolution changes. Returns once the render target is bound.
call_audio_debug_report
Render the agent-readable audio-quality debug report (a paste-able text block that pre-attributes a glitch to a pipeline stage) from the latest stats snapshot. For the “Copy audio report” affordance; returns a “healthy” report between calls.

call_set_screen_quality
Set the live screen-share quality (motion↔legibility + a max resolution/fps cap). Reconfigures a running share’s ladder live — no restart — and is remembered for the next share + across a reconnect. Harmless with no active call.
call_set_stream_volume
Set one remote audio stream’s playback volume — the tile right-click volume/mute control. source is Source::Microphone (the peer’s voice) or Source::Screen (their share’s app audio); volume is 0.0 (muted) … 1.0 (unity), clamped ≤ 2.0. Local-only (the sender is not told); applies within ~20 ms and survives reconnects/layer switches.
call_set_tile_quality
Report a remote video tile’s rendered pixel size + visibility, for peer peer_id’s source (Source::Camera | Source::Screen). Small or hidden tiles are switched to the LOW simulcast layer (and large/visible ones to the full source), so a receiver only pays to decode what it’s actually showing. Call from each tile’s layout/visibility. No-op when not in a call or for a non-video source.
call_set_video_paused
Pause/resume inbound video (audio keeps flowing). Call from the app lifecycle observer: paused = true when the app is backgrounded / the call surface is not visible, so we stop receiving + rendering remote video (notably a heavy screenshare) that nobody can see. No-op when not in a call.

Build the HoundClient bound to base_url with a keyring-backed session store and install it as the process-wide handle.

The /auth/{register,login}/{start,finish} request/response DTOs — defined ONCE here so the relying party (backend) and the caller (client) share the exact serde shape. snake_case fields, matching the live wire; the options/credential fields use the typed crate::proto wire types, not serde_json::Value.

The finish responses carry a profile. The profile shape is relying-party defined, so the finish response is generic over it (ProfileResponse is the default, matching the backend’s /profile/global shape). The backend plugs in its own profile type (which keeps its OpenAPI schema + From<Model> impl); the client decodes into ProfileResponse.

With the utoipa feature the DTOs derive ToSchema, so the backend can keep them in its OpenAPI components without re-declaring the shapes.

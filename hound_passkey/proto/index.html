The canonical WebAuthn wire types — re-exported from webauthn-rs-proto so every caller (the relying party AND the client authenticator seam) shares ONE vocabulary rather than passing opaque serde_json::Values around.

These are the exact types webauthn-rs itself produces/consumes (it re-exports the same webauthn-rs-proto types through its core), so a DTO built on them (de)serialises byte-identically to the live /auth/* wire.

  • CreationChallengeResponse — server → client: feed to navigator.credentials.create() (register/start).
  • RegisterPublicKeyCredential — client → server: the created credential (register/finish).
  • RequestChallengeResponse — server → client: feed to navigator.credentials.get() (login/start).
  • PublicKeyCredential — client → server: the assertion (login/finish).

CreationChallengeResponse
A JSON serializable challenge which is issued to the user’s web browser for handling. This is meant to be opaque, that is, you should not need to inspect or alter the content of the struct - you should serialise it and transmit it to the client only.
PublicKeyCredential
A client response to an authentication challenge. This contains all required information to asses and assert trust in a credentials legitimacy, followed by authentication to a user.
RegisterPublicKeyCredential
A client response to a registration challenge. This contains all required information to assess and assert trust in a credential’s legitimacy, followed by registration to a user.

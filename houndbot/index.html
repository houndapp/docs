houndbot — a headless call-participant bot for CI e2e call-smoke tests.

Signs in via magic-link (email code, read out of Mailpit), joins the first Voice-kind space in the signed-in user’s first community using the engine’s synthetic (camera-free/mic-free) capture sources, samples call stats for a fixed window, leaves, and prints a pass/fail verdict as JSON to stdout. Exit code: 0 pass, 1 fail (ran to completion under the bar), 2 setup error (never got far enough to run the actual test).

§Auth-flow choice

[HoundClient::magic_login_begin] binds a loopback callback listener (hound_browser_auth::MagicLogin::begin, 127.0.0.1:0 — OS-ephemeral, so it’s not actually a fixed-port concurrency hazard) and then blocks on that listener for a link click. A headless bot never clicks the emailed link — it reads the code straight out of Mailpit — so that whole listener is dead weight: this bot bypasses it entirely. It builds its own throwaway [RestClient] and calls magic_start directly (dummy port/state — the link the email carries is never opened, only the code line is read), then feeds the polled code into the real HoundClient via [HoundClient::magic_login_code]. Net effect: zero loopback listeners anywhere in this binary, which also makes “run two bots concurrently on one machine” trivially safe (no shared port of any kind, ephemeral or not).

Modules§

mailpit 🔒
Mailpit polling: find the magic-link email the backend sent and pull out its sign-in code.
verdict 🔒
The pass/fail verdict — a pure function of the signals the run observed, so it’s unit-testable without a live backend/network. main gathers the inputs (auth outcome, join outcome, stats samples) and calls compute_verdict once at the end of the run.

sign_in 🔒

houndbot — a headless call-participant bot for CI e2e call-smoke tests.

Signs in via magic-link (email code, read out of Mailpit), joins the first Voice-kind space in the signed-in user’s first community using the engine’s synthetic (camera-free/mic-free) capture sources, samples call stats for a fixed window, leaves, and prints a pass/fail verdict as JSON to stdout. Exit code: 0 pass, 1 fail (ran to completion under the bar), 2 setup error (never got far enough to run the actual test).

Sign-in delegates to [hound_testkit::auth] (see [hound_testkit::auth::sign_in_magic_link] for why this bypasses HoundClient’s loopback-listener magic-link flow — a headless bot never clicks the emailed link). Stats sampling drives [hound_client::call_health] directly — the same warmup-anchored starvation monitor the hound-testkit scenario suite gates its expect_steady_audio assertion on.

verdict 🔒
The pass/fail verdict — a pure function of the signals the run observed, so it’s unit-testable without a live backend/network. main gathers the inputs (auth outcome, join outcome, stats samples) and calls compute_verdict once at the end of the run.

run 🔒
Run the full bot flow, returning the verdict to print and the process exit code (0 pass / 1 fail / 2 setup error). Never panics on an expected-shape failure (auth, join, missing voice space) — those become a signed_in/joined-reflecting best-effort verdict with failures explaining what happened, per the setup-error contract.

Build a best-effort verdict for a run that never reached the sampling window — every gated signal is at its worst (0 peers / 0 frames), so compute_verdict already fails it; this just adds the concrete reason.

Pure I420 → RGBA8888 conversion with nearest-neighbor scaling.

The non-macOS render seam (Windows/Linux) hands decoded I420 to a CPU pixel-buffer texture: there is no GPU shader (the macOS path renders via wgpu into an IOSurface), so the convert + fit-to-texture happens here on the render thread and writes straight into the plugin’s RGBA buffer.

Pure + deterministic, so the pixel math is unit-tested on ANY host (including macOS builds, which never use it) — the part most likely to be wrong when a platform can’t be exercised locally. Scaling is a plain nearest-neighbor stretch to the fixed texture size; Flutter’s FittedBox applies the tile fit, so aspect handling (and any SIMD convert) stays out of this seam.

i420_to_rgba_scaled 🔒
Convert a tightly-packed I420 source (sw×sh: Y(sw*sh) ‖ U ‖ V, even extents, stride == width) to RGBA8888 (dw×dh, row-major, 4 bytes/px) written into out, nearest-neighbor scaled. BT.601 limited-range YUV→RGB (the engine’s encode/decode matrix). No-ops (leaves out unchanged) on a malformed source or an undersized out, so a single bad frame never panics the render thread.

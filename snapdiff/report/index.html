Module report

Expand description

Report DATA, not markup: the report is a static SvelteKit SPA (tools/snapdiff/ui/, built with pnpm build) that renders window.__SNAPDIFF__ at runtime. This module’s job is just: serialize the outcomes into that payload (data.js), copy the built UI’s assets into the report directory, and copy each pair’s baseline/current/diff PNGs into imgs/<name>/ as plain files — relative <img src> paths, no base64, so opening the report is one <img> fetch per image (no CORS to fight; only the JS module-script CORS restriction under file:// needed solving, and that’s handled entirely on the UI side — see ui/svelte.config.js’s bundleStrategy: 'inline').

Plain recursive file copy (no crate dependency for this — it’s a dozen lines and pulling in e.g. fs_extra for just this would be overkill). Skips anything that isn’t a regular file or directory (i.e. symlinks — a Vite build shouldn’t produce any, so this is a safety net, not a feature).
default_ui_dist 🔒
Default built-UI location: tools/snapdiff/ui/build, resolved relative to THIS crate’s own manifest dir (a compile-time constant) rather than the process’s current working directory, so snapdiff finds it regardless of where it’s invoked from.
image_subdir 🔒
The per-pair image subdirectory: the pair name minus its .png suffix (e.g. login.png -> login) — pair names are already unique, so this alone is enough to avoid collisions, no further sanitization needed.

Write bytes (if present) to dir/filename, creating dir first. Returns Ok(None) (not an error) when bytes is None — a pair legally has no baseline/current/diff image (e.g. Missing/New).

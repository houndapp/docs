snapdiff — perceptual visual-regression comparison for the Layer-2 snapshot system (see ui/integration_test/visual_test.dart + the harness in ui/integration_test/visual.sh). Pairs PNGs by name across a baselines dir and a current dir, scores each pair with image_compare::rgb_hybrid_compare, and emits a JSON summary (stdout) plus one self-contained HTML report.

Modules§

compare 🔒
Pairing + perceptual comparison of two directories of PNG snapshots.
report 🔒
Report DATA, not markup: the report is a static SvelteKit SPA (crates/snapdiff/ui/, built with pnpm build) that renders window.__SNAPDIFF__ at runtime. This module’s job is just: serialize the outcomes into that payload (data.js), copy the built UI’s assets into the report directory, and copy each pair’s baseline/current/diff PNGs into imgs/<name>/ as plain files — relative <img src> paths, no base64, so opening the report is one <img> fetch per image (no CORS to fight; only the JS module-script CORS restriction under file:// needed solving, and that’s handled entirely on the UI side — see ui/svelte.config.js’s bundleStrategy: 'inline').

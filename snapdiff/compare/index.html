Module compare 

Source
Expand description

Pairing + perceptual comparison of two directories of PNG snapshots.

Pairing is by filename: <baselines>/foo.png vs <current>/foo.png. A baseline may carry a same-named <name-without-.png>.mask.png painting regions to ignore (BLACK = ignore, anything else = compare) — e.g. a timestamp or cursor blink that’s expected to vary run to run. Comparison itself is image_compare::rgb_hybrid_compare (SSIM on luma + RMS on chroma): a score in [0,1], 1.0 = identical, is cheap to threshold and comes with a ready-made per-pixel diff visualization ([image_compare::Similarity::image]) — no separate pass needed to build the report’s diff image.

accept_current
--update: copy every current image over its baseline (creating new baselines for New pairs, replacing changed ones for Fail/Pass). Masks are untouched — they’re an accepted-baseline artifact, not a per-run one. Missing/Error pairs have no current file to copy and are skipped.
apply_mask 🔒
Zero out masked (ignore) regions in both images so they can never contribute a diff. mask pixels with luma < 128 are ignored (painted BLACK in the mask file); everything else is compared normally. Panics only on a logic error (mismatched dims) — callers must check first.
compare_all
Compare every discovered name, returning outcomes sorted with the most actionable statuses first (fail/missing/error, then new, then pass) and alphabetically within each — the report and terminal summary should lead with what needs attention.
compare_existing 🔒
The (true, true) arm of compare_pair, split out to keep that match readable: both files exist, so decode, mask, and run the perceptual comparison.
compare_pair
Compare one named pair. Never panics on bad input (corrupt PNG, missing file raced out from under us) — every failure mode becomes a Status::Error outcome, not a propagated Result, so one bad pair doesn’t abort the whole run.
discover_names
Union of snapshot names present in either directory, sorted — the full pair worklist (a name may be New or Missing on one side).

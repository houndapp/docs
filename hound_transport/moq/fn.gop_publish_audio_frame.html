Publish one audio frame as an object in a COUNT-ROLLED group (the ordered-substream model — see AUDIO_GROUP_OBJECTS). The first frame and every AUDIO_GROUP_OBJECTS-th frame opens a fresh group (new ordered QUIC stream); the rest append to the open group, so a window of frames rides one in-order stream. The frame’s keyframe bit is IGNORED — every Opus frame is independently decodable, so audio grouping is driven by count, not by a keyframe flag. Shared by MoqPublisher and MoqTrackPublisher.

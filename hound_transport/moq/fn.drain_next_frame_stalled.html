drain_next_frame bounded by a stall budget, for VIDEO tracks.

With on-demand keyframes a GOP (= one group) can span many seconds, and a lost object inside the group being drained head-of-line-blocks every frame behind it until QUIC retransmits — the tile freezes, then fast-forwards. This pull instead treats “no object within stall” as a stuck group: abandon it (skip its remaining objects) and report VideoPull::Stalled so the caller can request a keyframe; the producer’s IDR opens a fresh group that next_group jumps to, bounding the freeze at ~stall instead of a retransmit tail.

Cancel-safe: the moq-net consumers are poll-based state machines whose cursors advance only on Ready, so the timed-out future is dropped without losing a frame — and a mid-object timeout only ever discards the group it abandons. Never use for audio (audio must drain every group).

Expand description

MoQ publisher over an iroh connection. Owns the moq-net session (held alive for the track’s lifetime), the broadcast producer, and the open track producer. Hand it encoded [Frame]s via MoqPublisher::publish_frame; it maps each onto the GOP group / object model in §2.

§_session: Session§_broadcast: BroadcastProducer§track: TrackProducer§open_group: Option<GroupProducer>

The currently open group (a GOP for video, a rolled audio window for audio). None until the first frame opens one.

§next_group_seq: u64

Sequence of the NEXT group to open (advanced per group roll).

§objects_in_group: usize

Objects written to the open group — drives the audio count-roll (gop_publish_audio_frame). Unused by the video keyframe-driven path.

pub async fn connect( conn: Connection, broadcast_path: &str, track_name: &str, ) -> Result<Self, MoqError>

Wrap an established iroh Connection as a MoQ session and announce one broadcast with one track. Returns once the moq-net client handshake completes. (The peer’s SUBSCRIBE may not have landed yet — gate the first frame on Self::wait_subscribed.)

Source

pub async fn connect_relay( conn: Connection, relay_token: &str, broadcast_path: &str, track_name: &str, ) -> Result<Self, MoqError>

Like Self::connect but for the MoQ relay path (services/sfu::moq_relay): perform the relay-token auth handshake first, then announce. The relay scopes this publisher’s announces to the space in relay_token, so broadcast_path must live under space/{space_id} (the crate::moq namespace convention) or the relay drops it.

The token is presented on a dedicated bi-stream and the relay’s accept verdict is awaited before the MoQ handshake — so a rejected token fails fast with MoqError::RelayAuthRejected and no broadcast is announced.

Source

async fn connect_inner( conn: Connection, broadcast_path: &str, track_name: &str, ) -> Result<Self, MoqError>

Shared body of Self::connect / Self::connect_relay: wrap the conn as a raw-QUIC MoQ session, announce one broadcast + track, run the moq-net Client handshake. The caller has already done any auth handshake.

Source

pub async fn wait_subscribed(&self) -> Result<(), MoqError>

Block until at least one subscriber has attached to the track. Awaiting this before producing the first keyframe guarantees the opening group is delivered (moq-net live-join gate; see module docs). Deterministic — no sleep/yield.

Source

pub fn publish_frame(&mut self, frame: &Frame) -> Result<(), MoqError>

Map one encoded [Frame] onto the MoQ group/object model and publish it.

  • keyframe → finish the open group (if any) and create_group at the next sequence; the keyframe is the first object in that group.
  • delta → append an object to the open keyframe group. A delta before any keyframe is MoqError::DeltaBeforeKeyframe.

The encoded payload is forwarded byte-for-byte (an OBJ_HEADER_LEN-byte hound_media::Frame envelope is prepended so the subscriber can rebuild the Frame; the relay never reads it). Per §2 a GOP is one group: the keyframe opens it and deltas ride the same ordered stream until the next keyframe rolls the boundary.

Source

pub fn publish_audio_frame(&mut self, frame: &Frame) -> Result<(), MoqError>

Publish one AUDIO frame as an object in a count-rolled ordered group (gop_publish_audio_frame) — the ordered-substream model that keeps successive Opus frames on one in-order QUIC stream instead of one stream/frame.

Source

pub fn finish(&mut self) -> Result<(), MoqError>

Finish the open group and signal end-of-track so the subscriber’s drain loop terminates cleanly. Call when the media flow ends.

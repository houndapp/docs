Client-side session state derived purely from ServerMessages.

SessionView is a deterministic reducer: feed it the server messages as they arrive and it maintains the current roster of peers and tracks. It holds no I/O and makes no requests — the UI and media engine read from it, and a ServerMessage::Welcome fully resyncs it (used on reconnect), so a dropped socket never leaves stale state behind.

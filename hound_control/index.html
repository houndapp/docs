§hound-control

The call control plane: a structured, versioned message protocol for managing a media session — tracks, codecs, mute, keyframe requests, and reconnect/resync — shared by the client and the SFU/backend.

It sits above the participant roster (which the backend already provides via HTTP join + the socket.io /community namespace) and beside the media plane (hound_media, which carries the actual packets). This crate is sans-IO: it defines the wire messages and a pure state reducer, with no sockets or async, so both ends share one source of truth and every transition is deterministically testable.

  • proto — versioned proto::ClientMessage / proto::ServerMessage and their payload types (proto::TrackInfo, proto::PeerInfo, …).
  • session — session::SessionView, a client reducer that turns the stream of ServerMessages into the current roster of peers and tracks (a Welcome fully resyncs it on reconnect).
  • server — server::SessionServer, the authoritative side: assigns ids, validates ClientMessages, and emits routed ServerMessages (server::Outgoing / server::Target) for the transport to deliver.

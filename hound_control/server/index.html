Authoritative, sans-IO server logic for one call.

SessionServer is the counterpart to the client’s crate::session::SessionView: it owns the source-of-truth roster, assigns PeerId/TrackIds, validates each ClientMessage, and emits the ServerMessages that result — each tagged with who should receive it (Target). It performs no I/O: the transport (the backend’s socket.io namespace) feeds it messages and routes the Outgoings it returns.

Keeping this pure means the routing rules (a publish reaches everyone, a keyframe request reaches only the producer, a leave drops the peer’s tracks) are tested deterministically, independent of the socket layer.

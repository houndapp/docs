§AV1 OBU parsing + av1C configuration-record synthesis

VideoToolbox can decode AV1 in hardware (A17 Pro / M3 and later), but unlike H.264/HEVC there is no CMVideoFormatDescriptionCreateFromAV1ParameterSets in objc2-core-media 0.3.2. The AV1 format description must instead be built with the generic CMVideoFormatDescriptionCreate, carrying an av1C configuration-record (the AV1 Codec ISO-BMFF binding, [AV1CodecConfigurationRecord]) as a sample-description extension atom. This module is the codec-only half of that: it has no VideoToolbox / objc2 dependency, so it is unit-testable on any host and shared cleanly with the decoder.

§What we parse, and what we deliberately don’t

AV1’s “low-overhead bitstream format” (what rav1e emits, and what goes on the MoQ wire) is a sequence of OBUs:

 obu_header + (optional) leb128 obu_size + payload

To synthesise an av1C we need exactly the fields the config record carries, all of which live near the front of the sequence-header OBU: seq_profile, seq_level_idx[0], seq_tier[0], the bit-depth flags (high_bitdepth/twelve_bit), mono_chrome, the chroma subsampling flags, and chroma_sample_position. We parse just far enough to reach those — through the (skippable) timing_info / decoder_model_info / operating_points preamble — and stop. We do not decode frames; libaom/dav1d/VideoToolbox does that.

§av1C layout (AV1-ISOBMFF §2.3.3, AV1CodecConfigurationRecord)

marker(1)=1, version(7)=1
seq_profile(3), seq_level_idx_0(5)
seq_tier_0(1), high_bitdepth(1), twelve_bit(1), monochrome(1),
  chroma_subsampling_x(1), chroma_subsampling_y(1), chroma_sample_position(2)
reserved(3)=0, initial_presentation_delay_present(1)=0,
  initial_presentation_delay_minus_one(4) OR reserved(4)=0
configOBUs[]  // the sequence-header OBU, verbatim (header + size + payload)

VideoToolbox wants this atom under the "av1C" key inside kCMFormatDescriptionExtension_SampleDescriptionExtensionAtoms.

Structs§

BitReader 🔒
A minimal MSB-first bit reader over a byte slice. AV1 syntax is big-endian bit-packed (f(n) reads n bits MSB-first); this reads exactly that. Every read is bounds-checked and returns None past the end rather than panicking — the parser runs on attacker-influenced wire data.
Obu
One OBU located inside an AV1 bitstream, as a borrowed slice of the whole OBU (header + size field + payload) plus its decoded type. Keeping the whole OBU lets us copy the sequence header verbatim into the av1C configOBUs.
SeqHeaderConfig
The fields extracted from a sequence-header OBU that the av1C record needs.

parse_obus
Split an AV1 low-overhead OBU stream into its OBUs (the format rav1e emits and that travels on our wire). Each OBU is obu_header (1–2 bytes) + optional leb128 obu_size + payload. We require obu_has_size_field (rav1e sets it; the low-overhead format mandates it for all but the last), and parse defensively: a malformed length stops the walk rather than panicking.
parse_sequence_header
Parse the fields the av1C record needs out of a sequence-header OBU payload.
read_leb128 🔒
Decode an unsigned LEB128 value at data[pos..], returning (value, bytes_read). AV1 uses LEB128 for obu_size and the temporal-unit/frame-unit framing of the length-delimited “Annex-B”-ish variants. Bounded to 8 bytes per the spec.

§Opus encode / decode (libopus via audiopus)

The runnable codec for the native audio endpoint. OpusEncoder turns a 20 ms PcmFrame (960 samples @ 48 kHz mono i16) into one OpusPacket; OpusDecoder reverses it, including PLC (packet-loss concealment) on a None packet. Both wrap audiopus, which bundles libopus from source (no system lib / cmake / pkg-config — see Cargo.toml).

§Config (the endpoint’s contract)

  • 48 kHz, mono — Opus’s native rate, no internal resample, half-bitrate of stereo for voice.
  • 20 ms / 960-sample frames — SAMPLES_PER_FRAME.
  • Per-source application mode (OpusProfile) — Voip (speech-optimised) for the mic, Audio (general-audio) for shared screen/system audio. Using the speech mode on music is what makes it sound thin/“old-codec”.
  • Configurable bitrate — ~24 kbps for mono voice; music-grade (≥96 kbps) for shared audio. The mode AND the rate must match the source.
  • complexity 5 — mid; encode stays microseconds-cheap. (10 is max quality / most CPU; for real-time voice 5 is a fine default and keeps headroom on mobile.)
  • FEC on, 10% expected loss — in-band forward error correction so the decoder can reconstruct a lost frame from the next packet. Harmless on the lossless loopback; correct default for the MoQ transport step.

§Real-time safety

encode/decode run on a worker thread, never the cpal device callback. Each call writes into a reused scratch buffer (Vec<u8>) and then clones the exact-length slice into the packet — Opus encode at 24 kbps is well under a few hundred bytes, so the one alloc per packet is trivial and off the RT thread. (A zero-alloc pool is a production refinement, not needed for the endpoint.)

Enums§

OpusProfile
What the encoder is tuned for. Opus’s Application mode materially changes how it sounds: Voip applies speech-optimised psychoacoustics (great for a mic, but it crushes music into a thin, “old-codec” sound), while Audio preserves general-audio fidelity (the right choice for shared screen/system audio). Pick by the source, not globally.

Constants§

DEFAULT_BITRATE
Default voice bitrate. 24 kbps is comfortable mono speech; callers can override it at construction or retarget live.
MAX_PACKET_BYTES 🔒
Max bytes an Opus packet for one 20 ms frame can occupy. libopus’s documented upper bound is 1275 bytes/frame for a single stream; we size the scratch buffer to 1500 for headroom. The encoder returns the true length.

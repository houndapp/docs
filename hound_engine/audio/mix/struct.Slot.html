struct Slot {
    state: Mutex<PlayoutState>,
    carry: Mutex<VecDeque<i16>>,
    starves: Arc<AtomicU64>,
    own_starves: AtomicU64,
    label: &'static str,
    starve: Mutex<StarveTracker>,
    peer_tag: u32,
}
Expand description

One producer’s per-peer slot. The PUMP pushes decoded, timestamped PCM into state.buffer (the jitter buffer); the BUS THREAD pops the due frames on its own real-time clock into carry and drains MIX_CHUNK_SAMPLES per tick. The two mutexes are each held only for a bounded op, never across IO; the producer is the async per-peer pump, the consumer is the bus’s own thread (neither is a cpal RT callback).

Fields§

§state: Mutex<PlayoutState>

Jitter buffer + adaptive depth. Pump writes (push), bus reads (pop_due).

§carry: Mutex<VecDeque<i16>>

Sample carry-queue between the 20 ms popped frames and the 10 ms drained chunks. Touched ONLY by the bus thread, so it is uncontended — the mutex is for the Arc<Slot> interior mutability, not cross-thread coordination.

§starves: Arc<AtomicU64>

Shared bus counter of mix STARVATION events: a tick where this slot was primed (not idle) but its carry came up SHORT of a full chunk — a per-peer audible gap. Held as a shared Arc so a dropped slot’s events still count toward the bus total. THIS is the per-peer stutter signal (distinct from the output-ring underrun, which is the device-side gap). With the pull model a rising count means the JITTER BUFFER itself ran dry (real network loss / total starvation), not a producer-scheduling race.

§own_starves: AtomicU64

THIS slot’s own cumulative starve count (the shared starves above is the bus-wide sum). Used only to attribute the aggregate to a source kind (Self::label) so the stutter log can say WHICH audio starves — e.g. a screen-share’s intermittent app audio vs the remote mic (voice).

§label: &'static str

A human label for the slot’s source kind (“mic” / “screen_audio”), for the per-kind starve breakdown. Static — the call sites pass a string literal.

§starve: Mutex<StarveTracker>

DTX-aware starvation classifier (warmup gate + gap-onset window + recent-energy silence discriminator). Touched ONLY by the bus thread each tick, so the mutex is uncontended interior mutability over the Arc<Slot>, never cross-thread coordination. See crate::audio::starve.

§peer_tag: u32

The control-plane peer id this slot’s audio belongs to (NO_PEER_TAG if registered untagged), so a recorded underflow AudioGlitch can name the peer.

fn push(&self, now_us: i64, ts_us: i64, samples: &[i16]) -> PushStatus

Producer side (pump): fold this frame’s arrival into the adaptive jitter depth and push it into the jitter buffer, both under one lock so the depth retarget and the insert stay consistent. Returns the post-push depth + jitter estimate for the caller’s per-peer stats. now_us is the local receive time and ts_us the sender’s capture timestamp — BOTH on the shared [CaptureClock] the bus thread pops against.

Source

fn pull_into(&self, now_us: i64, out: &mut [i32]) -> PullOutcome

Bus side (one tick): release every frame whose playout time has arrived (up to now_us + MIX_PULL_LOOKAHEAD_US) into the carry-queue, then accumulate up to out.len() carry samples into out (summing in i32), leaving the rest of out untouched. Returns a PullOutcome (samples contributed + whether this was an underflow), so the bus loop can record a per-peer glitch. Bumps the shared starvation counter when a primed, non-idle slot comes up short — which, under the pull model, means the jitter buffer genuinely had nothing due (network starvation), not a producer-scheduling race.

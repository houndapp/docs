Expand description

The live-call audio-processing module: shared echo canceller + noise suppressor + gain control. Capture frames (mic) and render frames (far-end playout) are both fed through it; it subtracts the render reference from the capture so speaker audio does not echo.

One instance is shared (Arc<Apm>) across the capture thread and every per-peer playback pump. Internally a Mutex<AudioProcessing> — the WebRTC APM is Send but single-threaded, and capture/render must serialize against each other anyway (they share the canceller’s adaptive state).

impl Apm

Source

pub fn new() -> Option<Self>

Build the APM for the endpoint’s fixed 48 kHz mono format with AEC3, noise suppression (high), and AGC2 (adaptive digital) enabled, seeded with the initial stream-delay estimate. Returns None if the APM cannot be built (so the caller falls back to pass-through and the call still works — AEC failing must never break audio).

Source

pub fn process_capture(&self, frame: &mut PcmFrame)

Process one captured mic PcmFrame in place: AEC (subtract the render reference) + noise suppression + gain. The 20 ms frame is split into 10 ms APM chunks; each is processed in place. A frame whose length is not a whole number of 10 ms chunks is left untouched (defensive — the live path always hands exact 960-sample frames). The lock is held only for the short processing call, never across IO.

Source

pub fn process_render(&self, frame: &PcmFrame)

Feed one render (far-end / playout) PcmFrame to the APM as the echo reference. Called once per mix tick by the mixed-playout bus with the SUMMED 10/20 ms chunk — the single coherent stream the speaker plays. The APM accumulates it as the reverse stream it subtracts from capture; output is discarded (we do not alter what we play, only inform the canceller). Split into 10 ms chunks like capture.

There is exactly one render feed by construction (the bus), so AEC works for any number of peers — no per-source gate. (Before the bus, N per-peer pumps fed this directly and the interleaved references corrupted AEC3; that gate is gone.)

Source

fn chunkable(&self, samples: &[i16]) -> bool

Whether samples is mono and a whole number of 10 ms APM chunks (so it can be split cleanly). The live path always satisfies this (960 = 2 × 480); anything else is passed through unprocessed.

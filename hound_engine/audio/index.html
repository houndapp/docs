§Portable audio engine services

The pieces of the audio stack that are pure engine logic — built ON TOP of the platform backends, not part of them:

  • apm — the full-duplex audio-processing module (AEC3 + NS + AGC2, pure-Rust sonora), shared between the mic capture path (process_capture) and the playout bus (process_render).
  • mix — the mixed-playout bus: ONE output device stream per call and ONE coherent AEC render reference, summing every remote peer’s due PCM.
  • level_from_rms — the perceptual speaking-level mapping for UI meters.

All of it is platform-agnostic (the only device contact is through the injected output sink), which is why a new platform backend gets working multi-party audio + AEC for free.

Constants§

ROUTE_WARMUP_DELAY
One-shot reopen delay for audio endpoints on a non-48 kHz device (a Bluetooth headset in HFP/hands-free mode). The SCO link is frequently still establishing when a stream first opens; the stream then runs SILENTLY — no error fires, so the dead-stream reopen never triggers. A single reopen this long after open lands on the settled route. Applies to BOTH directions (playout in the mix bus, capture in the client mic loop); 48 kHz devices never arm it.

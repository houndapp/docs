§Engine error taxonomy

Per-domain error enums with typed fields where structure exists. The two domains mirror the two pipelines: AudioError for the mic/speaker/codec path and CaptureError for the camera/encode/decode path.

Design rules:

  • Permission and absence are distinct, typed variants — a denied device must never read as a silent zero stream, and the DeviceKind on PermissionDenied names which device was denied.
  • Whole error classes are unrepresentable by construction: “capturer not started” cannot exist because open-is-construct (a handle that exists is running), and “odd dimensions” cannot exist because crate::types::Dimensions is validated at construction.
  • Device-layer strings (Device, NoDevice, …) stay strings on purpose: they wrap opaque OS/driver error text that has no structure to type.

The app maps these to its UI-facing error type at its own boundary; the serialized UI contract lives there, not here.

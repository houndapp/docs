§Typed, zero-cost media vocabulary

The unit-bearing newtypes every engine API speaks in, plus the shared frame types the pipelines exchange. The point of this module is to move whole classes of errors to compile time:

  • a MicId cannot be passed where a SpeakerId or CameraId is expected (the “opened the wrong device kind” class is unrepresentable);
  • a Dimensions is always even and non-zero (validated at construction), so 4:2:0 plane math downstream can never see an odd geometry;
  • a SampleRate/ChannelCount/Fps/Bitrate can never be zero, and can never be confused with each other or with a bare count;
  • a TimestampUs is explicitly microseconds — no more “is this ms or µs” at a call site.

All newtypes are Copy wrappers over NonZero*/primitive values: zero-cost — they compile to the underlying integer. Conversions are explicit (.hz(), .bps(), .micros(), …); there are no Deref tricks and no implicit numeric coercions.

The literal-friendly constructors (Dimensions::new, Fps::new, Bitrate::kbps, …) are const fns that panic on invalid input — in a const context that panic is a compile error, which is exactly where a bad hard-coded geometry should fail. Runtime-sourced values go through the fallible try_new constructors instead.

Constants§

CHANNELS
Channel count. Mono for voice — half the bitrate of stereo and what AEC expects on the capture side. If a device only offers stereo input we downmix to mono before encode.
ENCODE_PRIORITY
Codec preference order, highest quality first: AV1 → HEVC → H.264. The ladder queries encode/decode support in this order; H.264 is the universal floor.
FRAME_MS
Opus frame duration. 20 ms is the voice sweet spot: 960 samples @ 48 kHz, low algorithmic delay, the default the WebRTC/Opus VoIP stacks use.
SAMPLES_PER_FRAME
Samples per channel in one PcmFrame / one Opus frame: 48000 × 20 / 1000 = 960. Opus only accepts exact frame sizes; the capture side must hand it exactly this.
SAMPLE_RATE
Sample rate for the whole audio engine. 48 kHz is Opus’s native internal rate (no internal resample) and the CoreAudio default on Macs — picking it end to end avoids a resample on either the capture or the playback side.

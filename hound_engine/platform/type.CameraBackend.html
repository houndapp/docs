Fields§

§stream: Stream<'static>

The running mmap capture stream. Held alongside the device; its Drop does the STREAMOFF + unmap.

§_device: Device

The device, kept alive for the stream’s lifetime. MmapStream borrows the device only at construction (it dups the fd internally), so holding the device here is for the lifetime/ownership tie, not an active borrow.

§capture_format: CaptureFormat

The format actually negotiated (post set_format) — which raw layout the dequeued buffers carry, and at what native geometry/stride.

§capture_native: Dimensions

Native capture geometry the device negotiated (what the mmap buffers carry, before the scale to the encode target).

§native_stride: usize

The bytes-per-row the driver reported for the (Y/luma) plane. Used by the NV12 de-interleave to drop row padding; YUYV uses width*2 directly.

§encode_target: Dimensions

The ENCODE-target geometry the caller requested. Every emitted frame is scaled to this so the encoder (built from this size) always agrees.

§neg_fps: Fps

Actual negotiated fps (from the stream params), falling back to the request when the driver doesn’t report a usable interval.

§clock: CaptureClock

Stamps each frame on the session timeline (A/V sync), shared with audio.

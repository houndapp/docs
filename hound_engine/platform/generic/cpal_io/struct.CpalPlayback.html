Expand description

Speaker playback via cpal. The output callback drains the ring; the trait’s AudioOutput::play pushes into it (non-blocking, bounded). Dropping the sink stops the stream and releases the device.

Fields§

§out: PlaybackProducer

Producer half of the wait-free playback ring (the RT callback owns the consumer).

§_stream: Stream

Held to keep the device stream alive; dropped (released) with self.

§resampler: MonoResampler

48 kHz → device-rate conversion (pass-through on 48 k devices). Runs in AudioOutput::play on the caller’s thread, NEVER in the RT callback (the ring stays device-rate mono).

§scratch: Vec<i16>

Reused conversion output buffer (no per-frame allocation).

§opened_name: Option<String>§dev_rate: u32

The device’s sample rate — to convert the ring’s buffered sample count to milliseconds for the mix bus’s output-cushion controller.

fn take_output_underruns(&self) -> u64

Read and reset the count of output samples the device’s RT callback had to silence-pad because the ring ran dry since the last call — the definitive end-of-pipeline AUDIBLE GAP (a choppy dropout). Steady state is 0; a rising count is the playout stutter, made measurable. Default 0 for sinks that do not (or cannot) track it.
Source§

fn buffered_ms(&self) -> u32

Current buffered playout depth in milliseconds (the device ring’s fill). The mix bus targets a small cushion here so the ring never drains to zero (which is what made it underrun on pacing jitter). Default 0 for sinks that don’t track it — the bus then falls back to one chunk per tick.
Source§

fn is_dead(&self) -> bool

Whether the underlying device stream has errored (e.g. the default output device changed / disconnected) and the sink must be reopened. The cpal stream is dead after such an error but this object lives on, so the mix bus checks this each tick and reopens — otherwise output stays silent. Default false.
Source§

fn device_rate(&self) -> u32

The opened device’s native sample rate (Hz). The mix bus uses this only as a heuristic: a rate other than the engine rate means the device was opened resampled — in practice a Bluetooth headset in hands-free (HFP) mode, whose SCO link is often not yet carrying audio when the stream first opens (silent first connect that a reopen fixes). Default = engine rate, so sinks that don’t report it (and every test fake) opt out of the warmup reopen entirely.

fn drop(&mut self)

Releasing the cpal stream stops the device and drops the RT callback (and with it the ring consumer); the producer drops with self. No explicit stop needed — the SPSC ring has no blocked consumer to wake (unlike the capture ring’s condvar).

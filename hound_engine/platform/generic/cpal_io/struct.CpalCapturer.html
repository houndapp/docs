Expand description

Microphone capture via cpal. The cpal Stream is !Send on CoreAudio, so it is created and held on the thread that calls Self::open; the ring it feeds is Send/Sync and is what AudioInput::next_frame reads. Dropping the capturer stops the stream and releases the device (on the owning thread — the !Send contract).

Fields§

§ring: Arc<Ring>§_stream: Stream

Held to keep the device stream alive; dropped (released) with self.

§clock: CaptureClock§negotiated: (SampleRate, ChannelCount)

The device truth: what the device negotiated (we DELIVER 48 k mono).

§opened_name: Option<String>

The name of the device actually opened, for the hot-swap path to log what it landed on. None if the device name could not be read.

§resampler: MonoResampler

Device rate → 48 kHz conversion (pass-through when the device is 48 k). Runs on the consumer thread in AudioInput::next_frame, NEVER in the RT callback.

§pending_48k: Vec<i16>

Converted 48 kHz samples awaiting frame assembly (FIFO tail of the resampler output that did not fill a whole 20 ms frame yet).

Open the best available input device and begin capture, stamping frames on clock. Tries input_candidates in order (named → Linux shared server PCM → host default), so a busy/exclusive or vanished device falls through to the next rather than failing the call — non-exclusive access by default.

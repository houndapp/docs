§Bounded producer/consumer ring (portable)

A small, fixed-capacity queue shared between a capture producer (a platform callback queue, or a bridge thread draining a foreign capturer) and a single consumer (the capture/encode thread’s blocking pull). One shape serves every video capture backend — AVFoundation’s delegate ring, ScreenCaptureKit’s frame ring, and the WGC / portal-PipeWire bridge rings all reduce to this.

§The one invariant that matters

The producer never blocks on the consumer. A capture callback runs on a platform-owned realtime queue; making it wait would stall the OS capture graph. So on overflow BoundedRing::push drops the OLDEST item (the consumer is behind) and returns. The queue is small on purpose — frames are large, so a deep queue is just latency.

The consumer blocks via a Condvar: BoundedRing::pop_blocking waits for an item or stream stop; BoundedRing::pop_timeout is the bounded-wait variant that distinguishes a still-running-but-idle source (Idle) from a stopped one (Ended) — the signal a damage-driven screen capturer needs (a static window legitimately produces nothing).

Arc<BoundedRing<T>> is Send + Sync when T: Send, so a platform delegate reaches it through a raw pointer without retaining a non-Send foreign object.

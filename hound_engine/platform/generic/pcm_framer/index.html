§Sans-io 20 ms PCM framer (portable)

The shared assembler that turns a ragged stream of 48 kHz mono i16 samples — however the OS capture callback batches them — into the engine’s fixed crate::types::SAMPLES_PER_FRAME-sample (20 ms) [PcmFrame]s, each carrying a unique, monotonically advancing capture-clock timestamp.

It is pure (now_us is passed in; no clock, no I/O, no locks), so every cadence rule below is exercised by the unit tests with plain integers — no audio device required. Both screen-share audio backends feed it: macOS ScreenCaptureKit ([crate::platform::apple::screen]) and Windows WASAPI loopback ([crate::platform::windows::wasapi_loopback]); the microphone path frames at the device boundary instead. One copy of the cadence math, one set of tests.

§Why the timestamp cadence is not just clock.now()

A capture callback does not hand you 20 ms at a time — it delivers whatever the OS batched (often several frames’ worth at once). Stamping every block carved out of one callback with the same now() produces duplicate timestamps, which the receiver’s playout buffer discards as duplicates → choppy audio. So the framer advances an internal cursor by exactly one frame (20 ms) per emitted block: a single fat callback yields evenly spaced, unique timestamps. The cursor re-anchors to now_us on the first block and whenever it has drifted more than [REANCHOR_US] from wall-clock (a capture stall/pause), so the cadence tracks real time without lagging forever.

Structs§

PcmFramer 🔒
Accumulates 48 kHz mono i16 samples and emits fixed 20 ms PcmFrames with a monotonic, re-anchoring timestamp cadence. Sans-io: drive it with Self::push, passing the current capture-clock time; it never blocks, never allocates beyond the pending tail, and holds no OS handle.

Constants§

FRAME_US 🔒
Wall-clock span of one 20 ms frame, in microseconds (960 / 48 000 s).
REANCHOR_US 🔒
Re-anchor the cadence cursor to now_us once it drifts this far from wall-clock — a capture stall/pause. Keeps timestamps tracking real time while a few-frame batch still gets a smooth 20 ms cadence.

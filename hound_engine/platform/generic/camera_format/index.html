§Camera capture-format selection (pure, platform-agnostic)

The preference ordering a camera backend uses to pick which pixel format to negotiate, plus the mapping from a raw FourCC to a known CaptureFormat. This is deliberately pure (no V4L2, no platform types) so the preference logic — the part most likely to harbour a “we picked MJPEG when a raw format was available” bug — is unit-tested on every host, including this macOS dev box where the Linux V4L2 capturer itself cannot compile.

§Preference order (best first)

  1. NV12 — biplanar 4:2:0, the exact layout the HW encoders and the decoders speak; the cheapest path to tightly-packed I420 (a plane de-interleave, no color math). Same format the AVFoundation capturer forces on Apple.
  2. YUYV (YUY2) — packed 4:2:2, the near-universal UVC webcam format. A cheap repack + a 2-row chroma average reaches I420 with no JPEG decode.
  3. MJPEG — last resort. Decoding it would pull a JPEG dependency we want to avoid; a camera that offers only MJPEG is recorded as a known follow-up rather than supported here (see CaptureFormat::needs_decoder).

Anything we don’t recognise is ignored (never selected) so the capturer only ever opens a format it can actually convert.

Enums§

CaptureFormat 🔒
A camera pixel format the engine knows how to turn into I420, ordered by how cheaply it converts (the Ord derive ranks NV12 < Yuyv < Mjpeg, i.e. NV12 is “smallest”/best — preferred = the minimum).
OpenErrorKind 🔒
How a device-open error should be classified for the engine’s typed error taxonomy. Derived purely from the raw OS errno so it is testable without opening a real device (errno values are stable across Unix).

Functions§

classify_open_errno 🔒
Classify a raw OS errno from a camera-device open into OpenErrorKind. EACCES=13, ENOENT=2, ENODEV=19 are the same on every Unix the engine targets, so this is host-independent and unit-tested on macOS.
fps_from_interval 🔒
Compute integer fps from a V4L2 frame-interval fraction (numerator/denominator seconds per frame ⇒ fps = denominator / numerator). None on a degenerate interval (zero on either side, or a sub-1-fps result) so the caller falls back to the requested fps.
select_capture_format 🔒
Pick the best capture format from the device’s offered FourCCs, applying the preference order (NV12 > YUYV > MJPEG) and ignoring anything unrecognised.

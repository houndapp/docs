Expand description

Generates a moving test pattern as I420 frames at a fixed geometry/fps. Open is construct (Self::open): a value that exists is producing, so a “synthetic source not started” error class is unrepresentable.

Fields§

§dims: Dimensions

Encode-target geometry. The source produces I420 directly at this size, so there is never a capture-vs-encode mismatch and no scale step — capture_native == encode_target and Self::is_scaling is false.

§fps: Fps§clock: CaptureClock§frame_idx: u64

Monotonic frame index, drives the animation phase.

§last_emit: Option<Instant>

When the previous frame was emitted, for fps pacing. None before the first frame.

pub(crate) fn open( width: u32, height: u32, fps: Fps, clock: CaptureClock, ) -> Result<Self, CaptureError>

Begin producing frames at as close to width×height as I420 allows (odd requests are even-clamped, matching real capturers) at fps, stamped on clock. Infallible in practice — kept Result to mirror the real device openers it stands in for.

Source

pub(crate) fn negotiated_fps(&self) -> Fps

The fps the source honours — always the request, exactly. Diagnostic; only the tests read it (a live join reads geometry off Self::negotiated).

Source

pub(crate) fn capture_native(&self) -> Dimensions

The synthetic source generates at the encode target, so the “native” capture resolution equals the encode resolution. Diagnostic; test-only caller.

Source

pub(crate) fn is_scaling(&self) -> bool

Always false: the source produces frames at the encode target directly. Diagnostic; test-only caller.

Source

fn render(&self, w: usize, h: usize) -> Vec<u8> ⓘ

Build one I420 frame for the current frame_idx. Pure CPU, fallible-free: every write is to a pre-sized buffer at an in-bounds index.

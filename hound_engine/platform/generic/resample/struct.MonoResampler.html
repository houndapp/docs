Expand description

Streaming mono i16 resampler with an internal input FIFO.

rubato’s fixed-input resampler consumes exact chunk_in-sized blocks; this wrapper accepts arbitrary-length input (decoded frames can be ragged after a defensive truncation), buffers the remainder, and appends converted samples to the caller’s output buffer. Latency cost is at most one chunk (20 ms).

pub fn new(in_rate: u32, out_rate: u32) -> Result<Self, AudioError>

A resampler converting in_rate → out_rate (mono). Equal rates yield a pass-through. Block size is 20 ms of input — every standard audio rate (8/16/22.05/24/32/44.1/48/96 kHz) is divisible by 50, so the block is exact.

The fixed input block size (0 for pass-through). The capture side pulls device samples in blocks of this size so each pull converts immediately.

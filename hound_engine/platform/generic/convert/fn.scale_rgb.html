Resize packed RGB (src_w*src_h*3 bytes, R,G,B) to dst_w×dst_h, returning a new packed-RGB buffer of dst_w*dst_h*3 bytes.

Bridges the camera’s negotiated capture resolution to the requested encode resolution so the encoder always receives the geometry it was built for. Uses [fast_image_resize] with a triangle (bilinear) filter — a good quality/speed tradeoff for real-time downscale; SIMD-accelerated where the crate detects it. This is the pre-step to rgb_to_i420.

Returns None on a malformed input (zero dimension or a short source buffer); the caller treats that as a frame to skip rather than corrupt planes. A no-op same-size resize still copies (the caller short-circuits the equal-dimension case before calling, so this stays a pure resize).

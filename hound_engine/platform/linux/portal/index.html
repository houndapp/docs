§xdg-desktop-portal ScreenCast session (Linux)

The source-selection + PipeWire handoff for a Linux screen share. Wayland deliberately gives applications no window/display inventory: the compositor’s own picker (the portal dialog) selects what is shared, and the portal hands back a PipeWire video node id plus a connection fd scoped to that stream. This module owns exactly that round-trip; the frames themselves are consumed by [super::portal_screen] over the returned fd.

§Threading

The portal is D-Bus (zbus), the engine’s one async edge on Linux. A dedicated portal thread owns a current-thread tokio runtime, the zbus connection, and the live portal session for the whole share: zbus needs a polled executor for as long as the session must stay open (dropping the connection closes the session, which ends the compositor’s share overlay and destroys the PipeWire node). [open] blocks its caller only for the setup round-trip — including however long the user looks at the picker — and the thread then parks in a stop-poll until the [PortalSession] guard drops, closing the session cleanly on its owning thread.

§Restore tokens

The portal can skip the picker on a re-share: Start returns a restore token which, passed to the next SelectSources, restores the previous selection (PersistMode::ExplicitlyRevoked keeps it valid across app restarts; we cache per share kind for the process lifetime). A share restart — the session’s answer to a window resize — therefore does not re-prompt the user.

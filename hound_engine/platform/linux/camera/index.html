§V4L2 camera capturer (Linux)

The camera VideoCapture for Linux, built on V4L2 via the [v4l] crate (safe Rust over the kernel ioctl ABI). It mirrors the AVFoundation capturer’s contract exactly so the engine facade treats the two identically: construct-is-start, next_frame blocks for the next frame, drop stops the stream and releases the device.

§Pipeline

/dev/videoN ─► Device::with_path ─► enum_formats ─► rank (NV12>YUYV>MJPEG)
       │                                                   │
       │            set_format(best @ native size) ◄───────┘
       │            set_params(1/fps)
       ▼
  MmapStream::with_buffers (REQBUFS/QBUF) ── kernel mmap ring
       │
  next_frame(): DQBUF ─► convert (YUYV/NV12 → tightly-packed I420)
                         ─► scale native → encode target (scale_i420)
                         ─► stamp on CaptureClock ─► RawFrame ── QBUF (recycle)

Unlike AVFoundation (whose delegate fires on an Apple-owned dispatch queue, forcing a bounded ring + panic containment across an FFI callback boundary), V4L2’s MmapStream::next() is a blocking pull on our own thread — there is no foreign callback thread, so next_frame simply dequeues, converts, and requeues inline. The kernel’s mmap buffer ring (REQBUFS count) is the backpressure: if we fall behind, the kernel overwrites the oldest buffer (the V4L2 analogue of the AVFoundation drop-oldest policy).

§Format negotiation (prefer raw, avoid a JPEG dep)

We rank the device’s offered formats with the shared, pure select_capture_format (NV12 > YUYV > MJPEG). NV12 is a plane de-interleave to I420; YUYV is a cheap repack + chroma average — both need no extra dependency. A camera that offers only MJPEG would need a JPEG decoder we deliberately do not pull in, so it surfaces a typed CaptureError::UnsupportedFormat naming MJPEG as the follow-up rather than silently pulling a dep. (Most UVC webcams offer YUYV alongside MJPEG, so this is rare.)

§Typed errors (never a silent zero stream)

  • EACCES opening the device ⇒ CaptureError::PermissionDenied (the user is not in the video group / lacks device perms) — distinct from absence.
  • No /dev/video* / a vanished named device with no default ⇒ CaptureError::NoDevice.
  • No convertible format ⇒ CaptureError::UnsupportedFormat.
  • Any other ioctl failure ⇒ CaptureError::Device with the OS error text.

§Threading

Like the other capturers it is !Send in our contract: construct, drive (next_frame), and drop on the one dedicated capture thread. Drop stops the stream (the v4l Stream’s own Drop issues STREAMOFF and unmaps the buffers — RAII), so there is no explicit teardown to forget.

Structs§

V4l2Camera 🔒
V4L2 camera backend (Linux). Holds the open device + the running mmap stream; pulls a frame per VideoCapture::next_frame, converts it to I420, and scales it to the encode target. Not Send (the V4L2 stream/device are tied to this capture thread). Open is construct (Self::open); dropping it stops the stream (STREAMOFF + unmap, via the v4l Stream’s Drop).

Constants§

MMAP_BUFFER_COUNT 🔒
How many mmap buffers to request from the kernel. A few frames of slack so a brief consumer hiccup does not stall capture, but small (video frames are large): a deep queue here is just latency. Matches the AVFoundation ring depth (4).

Functions§

build_stream 🔒
Build the mmap capture stream for device.
convert_native 🔒
Convert one raw V4L2 buffer (in the negotiated format) to a tightly-packed I420 buffer at the native geometry. A free function over snapshotted config (not a &self method) so it does not borrow the capturer while the mmap buffer slice from stream.next() is still alive (that slice mutably borrows self.stream, so any concurrent &self borrow would not type-check). None (a skippable frame) on a malformed/short buffer rather than a panic.
default_device_path 🔒
The default camera device path: the lowest-numbered /dev/video* that is a capture device. Uses the v4l context enumeration so it matches what the kernel reports, falling back to a /dev/video* scan if that yields nothing.
enumerate_cameras 🔒
Enumerate the connected V4L2 cameras (/dev/video*). Best-effort: returns an empty list if nothing is connected; never panics, never opens a stream (so it is safe from a blocking pool). The typed CameraId is the device PATH (stable for a fixed USB port; the human name comes from the driver). Only devices that report a video-capture capability are listed (a /dev/videoN can be a metadata or output node).
format_name 🔒
Human-readable name for a capture format, for error/log text.
map_open_error 🔒
Map a device-open io::Error to a typed CaptureError, via the shared (host-independent, unit-tested) classify_open_errno: EACCES ⇒ PermissionDenied (not in the video group), ENOENT/ENODEV ⇒ NoDevice, anything else ⇒ Device with the OS text.

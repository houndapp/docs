Expand description

A running portal screen capture: the portal session guard keeps the cast alive, the bridge thread fills the ring, this handle pulls I420 frames. !Send-by-use (drive and drop on one capture thread, like every backend).

pub(crate) fn open( source: &ScreenSource, config: ScreenCaptureConfig, clock: CaptureClock, ) -> Result<(Self, Option<PipeWireAppAudio>), CaptureError>

Open a share for source’s kind (the portal dialog picks the concrete target) and begin delivery. The encode target is the source’s native size aspect-fit into config.dimensions (a 1080p-class cap).

The audio leg follows the picked kind when config.capture_audio: display → whole-system (default-sink monitor); window → None (no owner identity on Wayland — never silently widened). An audio-leg failure degrades to video-only, parity with the other platforms.

fn next_frame_timeout( &mut self, timeout: Duration, ) -> Result<Option<RawFrame>, CaptureError>

Bounded-wait variant of Self::next_frame: Ok(None) when timeout elapses with no frame. Screen capture is damage-driven (a static source legitimately produces nothing, unlike a camera), so consumers that need liveness — preview tees, keyframe synthesis for late subscribers — poll with this instead of blocking indefinitely.
Source§

fn source_alive(&self) -> bool

Whether the captured SOURCE still exists (the shared window / display is still in the shareable inventory). Damage-driven capture cannot distinguish “static content” from “source gone” by frame flow alone, so an idle consumer probes this (throttled) and ENDS the share when it reports false — otherwise the last frame is re-emitted forever. Default true for backends without an inventory query (their stop/error path covers source loss).
Source§

fn content_size(&self) -> Option<(u32, u32)>

The source’s current content size (points) as observed on delivered frames, when the platform reports it. A running capture never resizes itself, so a change means the shared window was resized — the session’s cue to restart the share at the new geometry. None = no frame seen yet, or no such signal on this platform.

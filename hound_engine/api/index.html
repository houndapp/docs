§The high-level, platform-agnostic engine API

This is the only surface the app consumes (together with crate::types): open devices by typed id, list the device inventory, build codecs by typed configuration, and query what the running platform can do. No platform type (cpal, AVFoundation, VideoToolbox, …) is reachable from here — the handles below wrap the compile-time-selected backend and expose only engine vocabulary.

§Zero-cost facade

Engine is a zero-sized value; every handle is a thin newtype over the platform backend chosen by cfg at compile time. There is no dyn on the per-frame hot paths (the one boxed seam is the playout bus’s output sink, which is injected for testability and called 100×/s, not per sample).

§Capability, not panic

Platforms differ (no camera backend on a server build, no HW HEVC off Apple). Codec differences are queried at runtime via Engine::encode_caps / Engine::decode_caps; every other miss surfaces as the typed CaptureError::Unsupported a constructor returns when a caller asks anyway — never a panic, never a silently-dead pipeline.

Camera
A running camera: a started, platform-selected video capture delivering I420 frames at the requested encode geometry. Obtained from Engine::open_camera; dropping it stops the session and releases the device (on the owning thread — the !Send contract).
DeviceDescriptor
One enumerated device, typed by what it can be used for: a DeviceDescriptor<MicId> can only ever select a microphone.
Engine
The platform media engine — the app’s single entry point to devices and codecs. Zero-sized: the backend is selected at compile time, so Engine is free to construct, copy, and keep anywhere.
Microphone
A running microphone: a started, platform-selected audio input delivering fixed 20 ms 48 kHz mono PCM frames. Obtained from Engine::open_microphone; dropping it stops capture and releases the device (on the owning thread — the !Send contract).
ScreenAudio
The app-audio leg of a ScreenShare: the shared application’s audio as fixed 20 ms 48 kHz mono PCM frames — the same shape as a Microphone, so the Opus encoder and publish path reuse the mic plumbing. Obtained as the Some(_) second element of Engine::open_screen_capture when audio was requested for a window source on a supporting platform. Not Send: lives on the screen capture thread alongside its video parent.
ScreenShare
A running screen share: either a platform screen/window capture, or a camera shared as a screen (the picker’s “Devices” tab — a webcam, capture card, or virtual camera streamed onto a screen tile). Obtained from Engine::open_screen_capture; dropping it stops the session and (with it) any paired ScreenAudio leg. Not Send: drive and drop on the one capture thread (the !Send contract).
VideoDecoder
A video decoder for one inbound track, by wire codec, hiding whether the hardware (VideoToolbox) or the software (openh264) path was selected. Obtained from Engine::video_decoder; dropping it tears the decode session down. Not Send: lives on the decode thread.
VideoEncoder
A real-time video encoder for a negotiated codec, hiding whether the hardware (VideoToolbox: H.264 / HEVC / AV1) or the software (openh264, H.264 only) path was selected. Obtained from Engine::video_encoder (or the H.264-specific Engine::h264_encoder); query VideoEncode::codec for the wire codec. Not Send: construct, drive, and drop on the one encode thread.

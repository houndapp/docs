§Linux-only capture extensions

App-scoped screen-share audio by explicit application identity.

On macOS and Windows a window share carries the owning app’s audio automatically, because those platforms can map a window to its process. Wayland cannot: the xdg-desktop-portal picker never reveals which application owns the selected window, so nothing on Linux can auto-scope audio to “the shared window’s app” (OBS and Discord hit the same wall and make the user pick the app). This escape hatch is that pick: given an explicit identity — a root pid (matched with its whole process tree) and/or an application binary/name — it captures exactly that application’s PipeWire output streams as a standard crate::api::ScreenAudio leg, tracked live as the app starts and stops streams. The scope never widens: an identity that matches nothing yields silence, not system audio.

AppAudioIdentity
The application whose audio an app-scoped share leg captures. At least one criterion must be set; they are OR’d (pid-tree first, then the case-insensitive binary / application-name fallbacks).

Open an app-scoped share-audio leg for identity, delivering the standard 48 kHz mono 20 ms frames (ScreenAudio::next_frame). Pair it with a window share whose audio the portal could not scope automatically.

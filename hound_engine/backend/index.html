§The platform SPI (sealed)

The traits a platform backend implements to plug its devices and codecs into the engine. Sealed: the private supertrait below can only be implemented inside this crate, so the set of backends is closed at the crate boundary — the app can call through these traits (via the crate::api facade) but can never implement them or smuggle a foreign “device” into the engine. Adding Windows/Linux parity means adding an in-crate platform::windows / platform::linux module that implements these (see the crate-root “adding a backend” notes).

§Lifecycle: open is construct, drop is stop

There is deliberately no start() on these traits. A backend value that exists is running: construction performs the open/negotiate (and is where permission/absence errors surface, typed), and Drop releases the device/session on the owning thread. That makes a “called next_frame before start” runtime error unrepresentable.

§Threading contract (!Send by design)

Most backend values are !Send (CoreAudio streams, AVFoundation session graphs, VideoToolbox sessions are tied to their setup context). The contract everywhere is: construct, drive, and drop on one dedicated OS thread — never an async executor, never a UI thread, never moved.

Traits§

AppAudio
A running per-application audio source delivering fixed-size 48 kHz mono PCM frames — the audio leg of a ScreenCapture window/app share.
AudioInput
A running microphone source delivering fixed-size 48 kHz mono PCM frames.
AudioOutput
A running speaker sink that plays 48 kHz mono PCM frames.
ScreenCapture
A running screen/window source producing timestamped raw frames, optionally paired with an AppAudio stream of the source application’s audio.
VideoCapture
A running camera/screen source producing timestamped raw frames.
VideoDecode
A video decoder: encoded chunks in, raw frames out. The receive-side complement to VideoEncode; one per inbound video track.
VideoEncode
A video encoder: raw frames in, encoded chunks out, with keyframe and bitrate control. One per outbound video track.

pub fn allocate_send_budget(video_budget_bps: u64, desired: &[u64]) -> Vec<u64>
Expand description

Split a video budget across the active video tracks, max-min-fair with a floor.

video_budget_bps is the connection total minus the audio reserve. desired is each active video track’s current uncapped loss/RTT target (0 = the track isn’t producing yet / hasn’t decided). The returned vector is the per-track cap, in the SAME order as desired. Contract:

  • If sum(desired) <= video_budget OR sum == 0: return video_budget for EACH track — a generous ceiling, so no track is artificially limited and its OWN adaptation governs. (The coordinator only bites when the aggregate genuinely exceeds the budget.)
  • Else: scale each track’s cap proportionally to its desire, but never below MIN_VIDEO_BPS. If the floors alone can’t all fit in video_budget, fall back to an equal split (video_budget / n) so the outcome is still total and deterministic rather than handing out floors that oversubscribe.

Pure: no time, no state. The sum of the returned caps is ≈ video_budget in the over-budget case (within rounding), so the coordinator hands out exactly the estimated wire, no more.

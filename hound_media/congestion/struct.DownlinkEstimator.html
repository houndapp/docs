Expand description

Receiver-side downlink bitrate estimator for simulcast layer selection.

RateController sizes the uplink on the sender from receiver-reported loss; this is its mirror on the receive side. A receiver subscribed to one simulcast layer sees its own downlink loss (from the sequence tracker) and runs the same loss-based AIMD to estimate the bitrate its link can sustain. A LayerSelector then maps that estimate to a layer: a clean link climbs (probing the next layer up), a lossy one backs off, so each receiver independently converges on the richest layer its downlink carries — instead of every receiver being pinned to the worst leg by the single send-side ladder.

Pure/sans-IO: time is an argument, signals are explicit, the output is the estimate. Wraps a RateController so the AIMD behaviour is identical to (and shares the tests of) the send path.

Fields§

§rate: RateController§delay: DelayBasedBwe

Delay-based half (GCC): senses a filling bottleneck queue from the one-way delay trend before loss appears. The estimate is the MIN of this and the loss-based rate — whichever controller is more worried governs.

pub fn new(start_bps: u64, max_bps: u64) -> Self

start_bps seeds the estimate (typically the currently-subscribed layer’s bitrate); max_bps caps it at the top layer’s bitrate (× the selector’s climb margin) so it never probes past the richest layer the sender offers.

Source

pub fn on_report(&mut self, received: u32, lost: u32, now_us: i64) -> u64

Fold one reception-report interval into the LOSS-based estimate. received/ lost are the frame deltas the sequence tracker reports (the same pair sent upstream as MediaFeedback); now_us is the receive clock. An interval with no frames at all carries no signal and leaves the estimate untouched (no phantom climb). Returns the combined estimate (bits/sec).

Source

pub fn on_arrival(&mut self, recv_us: i64, send_us: i64) -> u64

Fold one frame’s arrival into the DELAY-based estimate. recv_us is the local arrival time, send_us the sender’s capture timestamp (only their differences are used, so the clocks need not be synchronised). Returns the combined estimate. Drives the early back-off that loss-based AIMD can’t see.

Source

pub fn estimate_bps(&self) -> u64

The current downlink estimate (bits/sec): the lower of the loss-based and delay-based controllers (GCC’s min-combine).

Source

pub fn delay_usage(&self) -> Usage

The current delay-based congestion verdict (for stats/debug).

Source

pub fn set_max_bps(&mut self, max_bps: u64)

Move the ceiling when the offered layer set changes (mirrors RateController::set_max_bps).

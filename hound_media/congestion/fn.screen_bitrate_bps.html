fn screen_bitrate_bps(height: u32, fps: u32) -> u64
Expand description

Target bitrate (bits/s) for one screen rung at height×fps. A single model — pixels × fps^0.72 × k, rounded to 100 kbps — instead of a hand-tuned table per (res, fps), staying monotonic in both axes (the ladder ordering relies on it).

Calibrated for SCREEN content, not camera video. Screen content (text, UI, large static regions, sharp intra-friendly edges) compresses several times better than motion video; a camera-video rate table (1080p30 → ~8 Mbps) oversizes a screenshare and saturates the shared connection, starving everything else. k is set so 1080p30 ≈ 2.4 Mbps — in line with production-RTC screenshare rates over codecs less efficient than H.265, and ample for crisp text. Rungs: 720p15≈0.6M, 1080p15≈1.5M, 1080p30≈2.4M, 1080p60≈4.0M. The BWE ceiling tracks the top rung, so a fast link reaches it.

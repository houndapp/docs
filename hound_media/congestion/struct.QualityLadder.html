Expand description

Maps available bitrate to a Rung with hysteresis and a manual ceiling.

Fields§

§rungs: Vec<Rung>

Rungs sorted ascending by bitrate.

§current: usize§audio_only: bool

Whether the last Self::update decided Quality::AudioOnly (the available bitrate fell below the lowest rung). Kept so Self::current is a faithful projection of the last decision rather than silently reporting Video(rungs[0]) while video is actually dropped.

§ceiling: Option<usize>

Manual maximum rung index (inclusive); None = no cap.

§climb_margin: f64

Headroom required to climb a rung (e.g. 1.15 = need 15% above its cost).

pub fn standard() -> Self

The default camera ladder: 180p15 → 360p20 → 360p30 → 540p30 → 720p30.

Source

pub fn screen() -> Self

The default screen-share ladder — DegradePriority::Legibility capped at 1080p60, i.e. screen_for(ScreenQuality::default()): 720p15 → 1080p15 → 1080p30 → 1080p60 (climb resolution before fps; on the way down, drop fps before resolution). Kept as a named constructor for callers that don’t expose the per-stream control; Self::screen_for is the configurable one.

Source

pub fn screen_for(quality: ScreenQuality) -> Self

Build a screen-share ladder for a per-stream ScreenQuality preference (degrade priority + resolution/fps caps). See screen_rungs for the rung construction; this just wraps the result. Pair with Self::reconfigure to change the preference on a live stream without losing the bitrate position.

Source

pub fn reconfigure(&mut self, rungs: Vec<Rung>)

Swap in a new rung set (e.g. from a mid-stream ScreenQuality change) while preserving the bitrate position: the next Self::update re-picks from the live available bitrate, so the only state to carry is “are we paused (audio-only)”. current is clamped into the new ladder; a stale ceiling index is dropped (the new ladder’s caps are baked into its rungs).

Source

pub fn bwe_ceiling_bps(&self) -> u64

The BWE estimate at which the top rung becomes selectable: the top rung’s bitrate plus the Self::update climb-margin headroom. The rate controller seeds its ceiling from this so it can actually climb to the richest rung on a fast link. Without it a flat BWE cap below the top rung × margin would strand the high rungs entirely — e.g. a 1440p120 top needs ~20 Mbps, so a 4 Mbps cap would top out at 1080p15 and never offer the quality a gaming LAN can carry.

Source

pub fn new(rungs: Vec<Rung>) -> Self

rungs must be non-empty and sorted ascending by bitrate.

Source

pub fn set_ceiling(&mut self, idx: Option<usize>)

Set the manual ceiling by rung index (clamped). None removes the cap.

Source

pub fn update(&mut self, available_bps: u64) -> Quality

Pick the quality for the given available bitrate. Drops as many rungs as needed immediately, but climbs at most one rung per call and only with climb_margin headroom (and never above the ceiling).

Source

pub fn current(&self) -> Quality

The quality from the last Self::update (a faithful projection: reports Quality::AudioOnly if that was the last decision). Before any update, reports the lowest video rung (the starting state).

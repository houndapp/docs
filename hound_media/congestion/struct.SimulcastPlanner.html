Expand description

Producer-side aggregator: tracks each receiver’s standing QualityRequest for one published video track and yields the current LayerPlan via plan_layers.

The sender feeds it every QualityRequested (peer → cap) and every peer-leave, and acts on the returned plan only when it changes — so a steady call costs nothing and a flapping receiver (already damped by its own send-side hysteresis) can at most toggle one layer. Peers are keyed by a plain u32 (the session passes PeerId.0) so this core stays free of the control-plane types. Pure/sans-IO.

Fields§

§source_bps: u64§caps: BTreeMap<u32, QualityRequest>

Per-receiver standing request. A peer absent here has not asked for anything → it is assumed to take the source (the default before any request arrives).

§desired: LayerPlan

The plan the requests imply — what the sender would publish if every receiver could consume the layer it asked for. Tracked even while actuation is off so the demand pipeline stays fully observable (logged on every change).

§actuate: bool

Whether the sender may actually act on a non-default desired (drop the source / spin up a low layer). This is only SAFE once a constrained receiver can switch its subscription from the source track to the low track — until that exists, dropping the source would starve receivers still subscribed to it, and any low layer is consumed by nobody. While false, Self::plan clamps to LayerPlan::SOURCE_ONLY, so the encode path is byte-identical to a build without simulcast regardless of what receivers request.

pub fn new(source_bps: u64) -> Self

Start with the given source bitrate and no receiver requests (plan = source only — the zero-cost common case). Actuation is OFF: see Self::set_actuation.

pub fn set_actuation(&mut self, actuate: bool) -> bool

Enable or disable acting on the demand-driven plan. Pass true only once constrained receivers actually move their subscription to the low layer (so dropping the source can’t starve anyone). Returns true if the EFFECTIVE plan (Self::plan) changed as a result — the caller must then re-publish it.

pub fn set_request(&mut self, peer: u32, req: QualityRequest) -> bool

Record a receiver’s standing request; returns true if the resulting DESIRED plan changed (the sender logs the new demand). Whether that translates into a reconfiguration depends on actuation — see Self::plan vs Self::desired.

pub fn remove_peer(&mut self, peer: u32) -> bool

Drop a receiver (it left or unsubscribed); returns true if the plan changed.

Source

pub fn retain_peers(&mut self, keep: impl Fn(u32) -> bool) -> bool

Reconcile against the live roster: drop caps for any peer not in keep (a receiver that left without retracting its request). Returns true if the plan changed. keep membership is tested via the closure so the caller needn’t allocate a set.

Source

pub fn set_source_bps(&mut self, source_bps: u64) -> bool

Update the source bitrate (the source rung re-negotiated); returns true if the plan changed (a cap that was below the old source may now be ≥ the new one).

Source

pub fn capped_peers(&self) -> BTreeSet<u32>

The peer ids currently requesting a CAPPED (below-source) layer — the receivers the shared low layer serves. The sender’s feedback partitioner excludes these from the SOURCE adaptation so a low-layer receiver can’t drag the source down (see the simulcast notes). Independent of actuation; the caller gates on its own flag.

Source

pub fn plan(&self) -> LayerPlan

The EFFECTIVE plan the sender must act on: the demand-driven Self::desired once actuation is enabled, otherwise the safe LayerPlan::SOURCE_ONLY (so a constrained receiver can never trigger a source drop while it is still subscribed to the source track).

Source

pub fn desired(&self) -> LayerPlan

The plan the current requests imply, regardless of actuation — for logging and observing the demand pipeline before the receive-side switch exists.

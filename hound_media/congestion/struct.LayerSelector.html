Expand description

Picks which published simulcast layer a receiver should subscribe to, given its DownlinkEstimator output. The layers are the sender’s published bitrates, ascending. Same anti-thrash contract as QualityLadder: drop fast (step down as far as needed the moment the estimate falls below the current layer) but climb slow (one step at a time, and only with climb_margin headroom over the next layer’s cost) so a flapping link can’t thrash subscriptions.

impl LayerSelector

Source

pub fn new(layer_bps: Vec<u64>) -> Self

layer_bps is the published layers’ bitrates; it is sorted ascending here, so the returned index is into the sorted order (lowest layer = 0). Starts at the lowest layer (safe bootstrap; climbs as the estimate proves headroom).

Source

pub fn update(&mut self, estimate_bps: u64) -> usize

The layer to subscribe to for estimate_bps. Drops as many steps as needed to fit; climbs at most one step, and only with headroom.

Source

pub fn current(&self) -> usize

The layer chosen by the last Self::update.

Expand description

A playout item carries a capture-clock timestamp the buffer schedules against.

Implemented for Frame (the audio/encoded-media path) and for any caller-defined wrapper (e.g. a decoded video frame on the receive path), so the playout policy lives here once and is never re-rolled in the I/O shell.

fn timestamp_us(&self) -> i64

Capture-clock timestamp (µs) this item should play out at, relative to the sender’s timeline. The buffer maps it onto the local clock on first push.

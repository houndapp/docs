const MAX_FRAME_GAP: u32 = 10_000;
Expand description

Bound on a single observed frame_seq gap. A restart (a fresh producer’s frame_seq starts back at 0) or a corrupt/wild value must not be misread as millions of lost frames — far above any gap a live call could produce (a stalled pipeline is reaped and re-subscribed long before frame_seq could legitimately jump this far).

§Capture clock

One monotonic timestamp source per session, shared by both the audio and video capture paths. Every captured frame is stamped from the same CaptureClock, so audio and video timestamps live on one timeline and the receiver can align them (A/V sync).

This directly fixes the root cause of the sync freeze in the old pipeline: video timestamps came from one capture API and audio from another, on unrelated epochs, so the two were never comparable.

§Design

The pure, testable core is CaptureClock::stamp, which converts a monotonic Instant reading into microseconds since the session epoch — the value carried in crate::packet::PacketHeader::timestamp_us. Tests pass synthetic instants (epoch + Duration); production passes Instant::now() (or uses the CaptureClock::now convenience).

The clock is Copy, so both capture tasks hold the same epoch and produce comparable stamps — the type makes “one clock” the easy path.

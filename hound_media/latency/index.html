§Latency histogram (sans-IO)

A fixed-bucket histogram for per-stage media latency, the measurement primitive the pipeline instruments each stage with (decode time, jitter-buffer dwell, capture→encode, publish-queue, the clock-synced network leg). Percentiles, not means, because the tail is what causes stutter: a p95 dwell spike matters even when the mean looks fine.

Deterministic and pure — record(us) slots a sample into a log-spaced bucket, percentile(q) interpolates within the bucket holding the q-th sample, and drain_percentiles reads + clears for a windowed (e.g. 1 Hz) emit. No clock, no atomics, no I/O: the IO shell wraps it (a Mutex at frame cadence is free) and feeds it durations. Bucket bounds span 0.25 ms … 2 s — fine resolution where real-time stages live (1–100 ms), coarse in the tail where only “it’s bad” matters.

Constants§

BOUNDS_US 🔒
Upper bounds (µs) of each bucket, ascending. A sample lands in the first bucket whose bound is ≥ it; anything larger saturates the top bucket. DENSE through the 10–150 ms band where the real-time stages and frame-interval cushions live (16/33/67 ms boundaries among them), so a value clustered at a frame interval reads back tightly rather than smearing across a wide bucket on interpolation; coarse in the tail where only “it’s bad” matters.

pub(crate) fn init_sentry(environment: Environment) -> Option<ClientInitGuard>
Expand description

Install Sentry crash/error reporting from SENTRY_DSN; None (nothing bound) when the var is absent or empty — Sentry is either fully configured or fully off, never a half-configured warning state.

The panic integration (see the sentry feature list in Cargo.toml) turns any Rust panic, on any thread, into a Sentry event; the sentry tracing layer wired into LogFormats::init_tracing_subscriber turns tracing::error! calls into events too. rustls-no-provider (the event-upload transport’s TLS feature) needs a process-default CryptoProvider installed before its reqwest::Client builds inside sentry::init, so install ring first — matches crates/app/frb’s init_sentry exactly.

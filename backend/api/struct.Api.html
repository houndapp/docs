async fn register_account( &self, user_id: Uuid, provider: IdentityProvider, external_id: String, display_name: String, ) -> ApiResult<FinishLoginResponse>

Shared tail of the “create a brand new account with a single login method” ceremonies (Steam register, magic-link register — passkey register additionally inserts a passkey mid-transaction, so it stays bespoke in finish_register): user row + identity + first-user instance-permission grant + global profile + default-community join, one transaction. The instance row is fetched first: it runs on its own pooled connection, and a write from a second connection deadlocks against the open transaction. The first-user grant itself is decided by Api::claim_first_user_grant (a race-safe conditional update), not a user count, so two concurrent registrations can’t both believe they’re first.

pub(crate) async fn resolve_channel_permission( &self, space: &Model, user_id: Uuid, permission: Permission, ) -> DbResult<bool>

Resolves the effective value of a channel-applicable permission for a user in a specific space:

  1. Organiser → always true (bypasses all checks).
  2. Community default — the user’s community-level role_permission rows.
  3. @everyone channel overwrite — overrides the community default for this space.
  4. Role channel overwrites — all of the user’s roles (excl. @everyone) combined: deny bits applied first, then allow bits (allow wins across roles).
  5. User channel overwrite — final, highest-priority override.

pub(crate) async fn get_or_create_instance(&self) -> DbResult<Model>

Returns the single instance row, inserting a default one if it doesn’t exist yet. The insert is ON CONFLICT DO NOTHING (portable across sqlite/postgres) rather than a plain insert: two concurrent first callers (e.g. two racing first registrations) both take the “missing” branch, and only one insert may land on the fixed id = 1 row — the other must not 500 on the resulting unique-key violation, it must just see the winner’s row.

Source

pub(crate) async fn claim_first_user_grant( &self, conn: &impl ConnectionTrait, ) -> DbResult<bool>

Atomically claims the one-time “first user” instance-permission grant slot. Returns true for exactly one caller across any number of concurrent registrations — the arbiter is the conditional UPDATE ... WHERE first_user_granted = false’s rows-affected count, not a preceding SELECT, so there is no read-then-write window for two callers to both observe “not yet granted”. Portable across sqlite/postgres: relevant only through standard UPDATE ... WHERE row-affected semantics (a concurrent updater targeting the same row either serializes behind the winner, on sqlite, or is re-evaluated against the winner’s committed row, on postgres — both land on 0 rows affected for the loser). Must run in the same transaction as the caller’s user-row insert so the claim rolls back with it on failure. The caller must have already ensured the singleton instance row exists (via Self::get_or_create_instance, on its own pooled connection, before opening the transaction — every caller does, to read default_community) — a claim UPDATE against a not-yet-existing row affects 0 rows, which would wrongly read as “already granted”.

pub(crate) fn generate_relay_token( &self, space_id: Uuid, user_id: Uuid, ) -> String

Generate a relay token that proves user_id may join the voice channel for space_id.

§Token format
{user_id}:{space_id}:{expiry_unix_secs}:{hex_hmac_sha256}

The token is verified by the SFU using the same shared secret (RELAY_HMAC_SECRET on the backend / RELAY_SECRET on the SFU) via the shared formula in hound_types::tokens::relay. Tokens expire after 15 minutes — short enough that a leaked token cannot be reused long after an access-control change, long enough to survive normal page-load latency.

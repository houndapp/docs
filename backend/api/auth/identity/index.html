Module identity

Expand description

Login-method management for a signed-in account: list identities, link an email (the authenticated twin of passkey registration), and unlink.

Two invariants, both enforced in Api::remove_identity so no endpoint can drift:

  • never-strand: removal must leave at least one identity on the account. Every provider counts unconditionally — a bare email identity (no passkeys) used to be excluded, but magic-link (api/auth/magic.rs) now makes it sign-in-able on its own, so it counts like Steam always did.
  • deleting the email identity deletes the account’s passkeys with it (they are unreachable without an email to start a passkey login ceremony — magic-link doesn’t need one).

backend::api::auth

Module magic 

Source
Expand description

Email magic-link login: a one-shot code emailed to the address, redeemed at /auth/magic/verify for either a session (known email) or a stateless registration token (unknown email — mirrors the Steam link-token flow in super::steam).

The pending-code + rate-limit maps live in-memory (same tier as Auth::inflights/auth_states) — fine for a single replica, not for horizontal scale.

/auth/magic/start never reveals whether the email has an account (always 202s on success, and does not query the DB at all): existence is only resolved at verify, after the caller has proven receipt of the code.

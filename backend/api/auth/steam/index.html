backend::api::auth

Module steam 

Source
Expand description

Steam login: Steamworks session-ticket validation (launched through Steam) and OpenID 2.0 assertion verification (browser flow), both resolving to a SteamID64 that maps onto an identity row.

Login endpoints return either a session (Steam ID already linked) or a short-lived signed link_token (unknown Steam ID) that the client redeems against /auth/steam/register (new account) or /auth/identity/steam/link (attach to the signed-in account). The token is stateless — no in-memory challenge maps, so it works across replicas and for both the ticket and OpenID paths.

Talks to the Steam Web API directly over the workspace reqwest. The candidate crates were rejected on inspection: steam-rs performs blocking HTTP (minreq) inside async fns and cannot pass the ticket identity parameter; steam-openid requires a static return_to; steamopenid links libcurl.

The /control call control plane.

Unlike /community (a socket.io namespace serving browsers), the control plane’s client is the Rust app, so this is a plain authenticated axum WebSocket — no socket.io. The transport-agnostic hound_control::SessionServer holds the authoritative per-call state; this module is only the connection, routing, and authorization glue around it.

§Endpoint & wire shape

GET /space/{space_id}/control?token=<jwt> upgrades to a WebSocket. A “call” is a space (keyed by space_id). Every WS text frame is JSON: inbound a hound_control::ClientMessage, outbound a hound_control::ServerMessage.

The first inbound frame MUST be ClientMessage::Hello { protocol_version }; the peer is admitted via SessionServer::join. Subsequent frames are fed to SessionServer::handle. A ClientMessage::Bye, the stream ending, or a transport error all run the leave path.

§Authorization (trust boundary)

The control plane is privileged. We authenticate & authorize before accepting the upgrade: the ?token= query param must be a valid, unrevoked access token, and claims.sub must be a member of the space’s community (is_community_member). Any failure returns 401/403 and the upgrade never happens — no PeerId is assigned, no ControlCall is created.

§State / lifecycle (SPOF flag)

Per-call state lives in ControlCall in Inner::control_calls, created on the first join for a space and dropped when the last peer leaves (SessionServer::is_empty()). This is in-memory and per-process — it does not survive a backend redeploy and does not coordinate across horizontally scaled replicas. /control is only correct on a single backend replica.

§Routing

Each admitted peer gets an mpsc::UnboundedSender<ServerMessage> registered in ControlCall::conns[peer]; a per-connection forward task drains the receiver to the WS sink. route resolves each Outgoing’s Target to recipient PeerIds against conns and pushes the message onto each sender. Because the transport tracks each connection’s own sender, BroadcastExcept(self) is a genuine exclusion (no join-ordering trick needed).

§Locking

The per-call SessionServer and conns are behind std Mutexes. We NEVER hold either across an .await: lock → call the sync join/handle/leave and collect recipients → drop the lock → then send. mpsc::UnboundedSender::send is sync and non-blocking, so routing never awaits while locked.

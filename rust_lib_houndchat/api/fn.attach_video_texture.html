Expand description

Attach a stream key (e.g. “cam:42”, “self”, “screen:7”) to a native GPU texture handle the Flutter plugin registered, sized width×height. native_texture is an opaque pointer (an IOSurfaceRef / id<MTLTexture> on Apple) the Rust renderer will draw into. texture_id is the Flutter texture id the same createTexture returned: macOS’s shared render loop marks it “dirty” once it draws a new frame, and the plugin’s ONE shared CVDisplayLink re-pulls only dirty ids (so copyPixelBuffer runs at video-fps, not display refresh). The plugin allocates the buffer at this SAME width×height (the decoded frame’s size, so the render target matches the source aspect with no crop/stretch); Dart recreates + re-attaches at a new size when the decoded resolution changes. Returns once the render target is bound.

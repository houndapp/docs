§Frame header encoding

Every packet sent through the SFU — whether it travels as a QUIC datagram (unreliable) or a QUIC stream (reliable) — starts with a 16-byte header.

§Why a fixed-size header?

Parsing is branchless and allocation-free: we read exactly 16 bytes, copy them into fields, and we’re done. The router in the SFU can inspect the header to decide where to forward the frame without ever looking at the payload.

§Header layout (all fields little-endian)

Offset  Size  Field         Description
──────  ────  ───────────   ─────────────────────────────────────────────
 0–1     2    peer_id       SFU-assigned ID for the sending participant.
                            The SFU replaces this with its own assignment
                            before forwarding, so receivers always see the
                            SFU's canonical ID for each peer.
 2–3     2    stream_seq    Wrapping sequence number per track. Used by
                            receivers to detect packet loss and trigger
                            error-concealment (e.g. Opus PLC for audio).
  4      1    frag_index    0-based index of this fragment. 0 for the first
                            (or only) fragment of a frame.
  5      1    frag_total    Total number of fragments. 1 means not fragmented.
                            frag_index < frag_total must always hold.
  6      1    track_type    Which A/V track this belongs to. See TrackType.
  7      1    flags         Bit flags: bit 0 = is_keyframe.
 8–15    8    timestamp_us  Capture time in microseconds. i64 LE, negative
                            values are valid (clock not yet synchronised).

Total: 16 bytes.

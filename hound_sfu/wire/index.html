§Wire protocol

Everything in this module is about turning bytes on the wire into typed Rust values and back again. No network I/O happens here — callers hand us &[u8] slices and we hand back structs.

§Layout

frame.rs  — 16-byte FrameHeader (peer_id, seq, track type, timestamp, …)
codec.rs  — codec negotiation types (VideoCodecId, AudioCodecId, …)
error.rs  — WireError: every parse failure with a clear description

The separation between frame.rs and codec.rs is intentional: a frame header knows which track a payload belongs to, but it doesn’t care about how the payload was encoded. Codec information is negotiated once at connection start (via the Hello/HelloAck handshake) and then remembered by the room; it is not repeated in every packet.

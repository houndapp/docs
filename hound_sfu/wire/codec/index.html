§Codec negotiation types

Before any media flows, the client and SFU agree on which codecs to use. This negotiation happens once during connection setup via the Hello/HelloAck JSON handshake. After that, codec information is stored in the Room and is not repeated in every packet.

§Why these codecs?

  • Camera video — AV1: Good compression, royalty-free, hardware support on all modern devices. String form used by the WebCodecs API: "av01.0.04M.08".

  • Screen video — H.264: The openh264 software encoder in Rust is well-tested for this use case. String form: "avc1.42001f". We don’t use AV1 for screen capture because openh264 is simpler to integrate and screen content is already low-entropy (encoder still wins on quality-per-bit).

  • Audio — Opus: The standard for interactive voice. 20 ms frames, comfortable at 32 kbps, built-in packet loss concealment (PLC).

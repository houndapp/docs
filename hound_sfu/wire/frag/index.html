§Frame fragmentation and reassembly

QUIC datagrams are bounded by the path MTU (typically ~1200 bytes after QUIC/UDP/IP overhead). Most audio packets and delta video frames fit comfortably within this limit, but keyframes can be several hundred kilobytes and must be split into fragments.

This module provides:

  • fragment — split a large frame payload into MTU-sized wire packets.
  • Reassembler — hold arriving fragments and return the complete payload once all pieces have arrived.

Both are pure, sans-IO: no network calls, no async, no locks. They take and return byte slices and Vec<u8>. The connection handler calls them and then does whatever network writing is required.

§Fragment format

Every fragment is a complete wire packet: a 16-byte FrameHeader followed by the payload chunk. The FrameHeader::frag_index and FrameHeader::frag_total fields identify which fragment this is and how many there are in total.

┌──────────────────────────┬────────────────────────────┐
│ FrameHeader (16 B)       │ payload chunk (≤ max_payload│
│  frag_index = i          │   bytes)                   │
│  frag_total = n          │                            │
└──────────────────────────┴────────────────────────────┘

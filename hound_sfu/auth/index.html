§Relay token verification

Every participant must present a relay token when connecting to the SFU. The token is issued by the backend and proves that:

  1. The user is authenticated (the backend checked their JWT).
  2. They are a member of the community that owns the voice channel.
  3. The token has not expired.

§Token format

{user_id}:{space_id}:{expiry_unix_secs}:{hex_mac}

Where hex_mac is HMAC-SHA256(secret, "{user_id}:{space_id}:{expiry}").

The backend and the SFU share the same RELAY_HMAC_SECRET environment variable. This secret is never sent over the wire — only the token is.

§Why HMAC instead of JWT?

Relay tokens are short-lived (15 minutes) and carry only three fields. HMAC is simpler, has no library complexity, and is easy to verify from any language. The backend’s JWT infrastructure is reused for user auth; we didn’t want to add a second JWT dependency to the SFU.

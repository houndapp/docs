§Room state and frame forwarding

A Room represents a single voice channel. It holds the list of connected participants and implements the forwarding logic: when a participant sends a frame, the room rebroadcasts it to every other participant.

§Sans-IO design

The room never writes to the network directly. Instead, each participant is represented by a FrameSink — a trait with two methods:

  • try_send_datagram — best-effort, unreliable (for delta frames and audio).
  • try_send_reliable — guaranteed delivery (for keyframes).

In production the sink wraps an iroh QUIC connection. In tests we use [MockSink], which pushes frames into a Vec so tests can assert on exact payloads without any networking.

§Peer IDs

The SFU assigns each participant a compact u16 peer ID when they join. This ID appears in every forwarded FrameHeader so receivers can match frames to participants without UUID lookups on the hot path.

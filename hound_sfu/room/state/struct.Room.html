Expand description

A single voice channel: tracks participants and forwards media frames.

§Lifetime

A Room is created when the first participant joins a voice channel and is dropped when the last participant leaves (checked via Room::is_empty). There is one Room per active voice channel across the whole SFU process.

§Thread safety

Room is not Sync on its own — it is always held behind an Arc<Mutex<Room>> in the connection handler so that multiple iroh tasks can operate on it concurrently.

Fields§

§peers: HashMap<PeerId, PeerInfo>

Participants currently in the room, keyed by their SFU-assigned peer ID.

§user_ids: HashMap<UserId, PeerId>

Reverse map: look up a peer ID by user UUID.

Used to deduplicate reconnects: if a user joins while they are already in the room (e.g. page reload), we evict their old peer first.

§next_peer_id: PeerId

Monotonically increasing counter for assigning peer IDs.

We never reuse an ID within a room’s lifetime, so a receiver can tell whether a peer_id it hasn’t seen before is new or is a replayed old one.

§codecs: NegotiatedCodecs

Codecs agreed upon during the Hello/HelloAck handshake.

Stored here so newly-joined participants can be told which codec the room is already using (we don’t renegotiate mid-room).

pub fn assign_peer_id( &mut self, user_id: UserId, sink: Arc<dyn FrameSink>, ) -> PeerId

Add a participant to the room and return their assigned PeerId.

If the user was already in the room (e.g. a reconnect after a page reload), the old peer entry is removed first to avoid ghost participants.

The sink is stored behind an Arc so the same sink can be shared with the iroh writer task without cloning the underlying connection state.

Remove a participant by their PeerId.

Source

pub fn forward(&self, from: PeerId, frame: Bytes, is_keyframe: bool)

Forward frame to all participants except the one who sent it.

from is the sender’s PeerId. The frame is forwarded to every other peer in the room using the appropriate delivery method:

  • keyframes → try_send_reliable (QUIC stream, must arrive)
  • all other frames → try_send_datagram (QUIC datagram, ok to drop)

Whether the frame is a keyframe is determined by inspecting is_keyframe. The frame bytes themselves are not parsed again here — the caller already parsed the header and passes the result as a parameter.

§Performance note

frame is a [Bytes] (reference-counted) so cloning it to send to multiple peers is O(1) — no copies of the payload are made.

Source

pub fn is_empty(&self) -> bool

Returns true if there are no participants in the room.

The connection handler uses this to decide when to drop the Room after the last participant disconnects.

pub fn peer_summaries(&self) -> Vec<PeerSummary>

Returns a snapshot of all current participants as [PeerSummary] values.

Used by the connection handler to populate the HelloAck.peers list so a newly-joined client can render placeholder tiles immediately.

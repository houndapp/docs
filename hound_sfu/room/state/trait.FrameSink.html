Expand description

A sink that accepts outgoing frames for one participant.

§Why a trait?

The room state machine is completely independent of iroh or any other networking library. It calls try_send_datagram / try_send_reliable and does not care how the bytes reach the other side.

Production code provides an implementation backed by iroh QUIC connections. Test code provides [MockSink], which stores frames in-memory so tests can assert on forwarding behaviour without standing up real network connections.

§Send semantics

Both methods are non-blocking. They return false if the frame was dropped (e.g. because the send buffer is full). The room’s forwarding loop logs drops but does not retry — real-time media prefers freshness over completeness.

fn try_send_datagram(&self, frame: Bytes) -> bool

Send frame as a QUIC datagram (unreliable, unordered).

Use for delta video frames and audio packets. Returns true if the frame was queued, false if it was dropped.

Source

fn try_send_reliable(&self, frame: Bytes) -> bool

Send frame over a QUIC stream (reliable, ordered).

Use for keyframes that a decoder must receive. Returns true if the frame was queued, false if it was dropped.

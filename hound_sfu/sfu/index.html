§SFU connection handler

This module owns the iroh Endpoint and the accept() loop. When a new client connects it:

  1. Opens the signalling stream and reads the Hello message.
  2. Verifies the relay token and negotiates codecs.
  3. Assigns the client a peer ID and sends HelloAck.
  4. Spawns two tasks per connection:
    • reader — receives QUIC datagrams and reliable streams, parses the frame header, and calls Room::forward.
    • writer — reads from a mpsc channel of outgoing frames and writes them as datagrams or reliable streams depending on the keyframe flag.

§Sans-IO seam

The forwarding logic lives in crate::room::state::Room and is fully tested without iroh via [crate::room::state::MockSink]. This module is the thin glue that wires iroh connections to that pure state machine.

§Connection lifecycle

accept() ──► handshake() ──► spawn reader + writer tasks
               │                     │
               ▼                     ▼
          Hello / HelloAck     Room::forward
          (signalling stream)  (frame routing)

Constants§

HANDSHAKE_TIMEOUT 🔒
How long the SFU waits, after accepting a connection, for the client’s Hello to arrive over the signalling stream before giving up. Generous enough to absorb relay/hole-punching setup, short enough that a client that connected but can’t deliver data doesn’t pin a handler task indefinitely.
HOUND_ALPN
ALPN protocol identifier for hound SFU connections.
NACK_DRAIN_TIMEOUT 🔒
How long reject waits for a rejected client to read its NACK and close the connection before the handler gives up and tears everything down.

Application-level QUIC close code ingress_watchdog sends when it closes a connection for exceeding IngressLimits::max_bytes_per_sec — distinct from the generic 0 code the auth/goodbye paths use, so a client inspecting the code/reason on conn.closed() can tell “the relay enforced a policy” apart from “auth failed” or “the network died”.

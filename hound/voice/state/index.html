§Voice session state

VoiceState is stored in Tauri’s managed state (app.manage(...)) and is accessible from every Tauri command via State<VoiceState>.

§Thread safety

The inner Option<ActiveSession> is protected by a tokio::sync::Mutex so async commands can await it without blocking the executor.

§Why not a plain RwLock?

Commands that start or stop a session need to mutate state while potentially awaiting async operations (HTTP calls, etc.). tokio::sync::Mutex allows holding the lock across .await points; std::sync::Mutex does not.

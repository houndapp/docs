§Platform-agnostic screen capture

Screen capture works very differently on each OS:

  • macOS — ScreenCaptureKit (Swift/Obj-C, requires entitlements)
  • Windows — DXGI + WASAPI
  • Linux — PipeWire or X11 Shared Memory

Rather than scattering #[cfg(target_os = "...")] throughout the codebase, we define a single ScreenCapturer trait here. Adding support for a new platform means adding a new impl ScreenCapturer in a new file — callers never change.

§Adding a new platform

  1. Create src/voice/screen/<platform>.rs.
  2. Implement ScreenCapturer for your type.
  3. Add a #[cfg(target_os = "<platform>")] alias to PlatformCapturer at the bottom of this file.

That’s it — commands.rs uses PlatformCapturer without knowing which implementation it got.

§Tauri voice commands

These are the functions the TypeScript frontend calls via invoke(...). Each command maps to one specific action in the voice lifecycle:

voice_join  → calls backend /space/{id}/voice/join, stores relay token
voice_leave → calls backend /space/{id}/voice/leave, clears state
voice_list_capture_targets → lists capturable screens/windows

Commands that require an active session check VoiceState first and return VoiceError::NotConnected if there is no active session.

§Error handling

All commands return Result<T, VoiceError>. On the TypeScript side, invoke rejects with a VoiceError object { kind, message } that can be pattern-matched on kind for user-facing messages.

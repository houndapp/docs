§Phase 0.5 spike S1 — native → webview render path

Validates the riskiest assumption of the native media engine: that we can push frames from native Rust into the webview and render them, on CEF desktop and an iPhone 15 Pro (WKWebView). See docs/voice/spikes/S1-render.md.

Throwaway spike code (no real decoder): it streams a synthetic RGBA test pattern at a caller-chosen resolution and frame rate over a Tauri [Channel] (Tauri sends large payloads via the efficient fetch path); the frontend draws each frame to a canvas and measures sustained fps / drops. Resolution and fps are parameters so the render-path ceiling can be probed live (720p30, 720p60, 1080p60, …) without rebuilding.

The realtime plane: a socket.io client for the backend /community namespace (server is socketioxide 0.18 — Socket.IO v5 / Engine.IO v4, which [rust_socketio] 0.6 speaks). It connects with { token } handshake auth, lets the caller subscribe/unsubscribe to per-space rooms, and decodes every server event into a typed RealtimeEvent delivered over a [tokio::sync::mpsc] channel — so the reducer consumes a single typed stream, never raw JSON.

Threading: rust_socketio’s async handlers are Fn closures returning BoxFuture<'static, ()>, so they cannot borrow. Each registered handler captures a clone of the event sender and a parser fn; on an event it parses the payload and forwards a RealtimeEvent (or a RealtimeEvent::Decode marker on malformed input — loud, never silent). The socket.io v5 wire delivers an event’s argument as the first element of a JSON array, surfaced by rust_socketio as [Payload::Text](Vec<Value>); we read values[0].

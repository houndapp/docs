The orchestration facade: ties the REST data plane, the session lifecycle, and the realtime (/community) plane into one observable, UI-free app core.

HoundClient is what the FRB layer (and any future native UI) drives. It owns the pieces and an AppState of tokio::sync::watch channels — one per observable snapshot, latest-wins, which is exactly what a UI wants (it never needs the history, only the current view). REST loads seed the snapshots; a single background task consumes the typed RealtimeEvent stream, folds each event through the pure reducers (CommunityTree/RosterView/MessageList), and republishes the mutated snapshot onto the matching watch sender.

Commands are async and return Result<_, HoundError>; observable getters return [watch::Receiver]s the binding subscribes. No FFI and no UI types live here — the crate stays standalone-buildable.

AppState 🔒
The observable app state: one watch channel per snapshot the UI renders. Each holds the latest value (a new subscriber sees it immediately) and notifies on change. Per-space message lists are created lazily on HoundClient::open_space.
HoundClient
The headless application core. Cheap to share via the returned Arc<Self>; every command takes &self. Holds the REST client, the session, the observable state, and the live realtime connection (replaced on each (re)connect).

pub struct HoundClient {
    rest: Arc<RestClient>,
    base_url: String,
    session: Session,
    state: AppState,
    realtime: Mutex<Option<RealtimeClient>>,
    call: Mutex<Option<CallSession>>,
    video_sink: Mutex<Arc<dyn VideoSink>>,
}
Expand description

The headless application core. Cheap to share via the returned Arc<Self>; every command takes &self. Holds the REST client, the session, the observable state, and the live realtime connection (replaced on each (re)connect).

Fields§

§rest: Arc<RestClient>§base_url: String

The base URL (kept so the call orchestration can derive the control WS URL).

§session: Session§state: AppState§realtime: Mutex<Option<RealtimeClient>>

The live realtime connection, if connected. Swapped on connect; dropped (and its background task ended) on disconnect/logout.

§call: Mutex<Option<CallSession>>

The live voice/video call, if joined. At most one at a time.

§video_sink: Mutex<Arc<dyn VideoSink>>

The decode→render sink the call hands decoded video to (keyed by peer). The FRB layer installs one whose body calls video_render::push_frame(...); until then a NullVideoSink keeps the call running (audio + state) headless.

Join the voice/video call for space_id. Establishes the control WS, dials the MoQ relay, publishes mic (+ camera if camera_enabled), and starts the subscribe loop. Drives the watch_call/watch_connection/watch_levels snapshots. Leaving any prior call first (one call at a time).

Pause/resume inbound video (audio keeps flowing). The UI calls this on app lifecycle changes — paused when backgrounded / the call surface isn’t visible — so we stop receiving + rendering remote video (notably a heavy screenshare) nobody can see. No-op when not in a call.

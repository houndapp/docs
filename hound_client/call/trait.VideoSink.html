Expand description

The decode→render seam. The orchestration decodes remote video into tightly packed I420 and hands it here, keyed by a stream key ("self", "cam:{peer}", "screen:{peer}"). The FRB layer supplies an impl whose body calls video_render::push_frame(key, w, h, i420) — so this crate stays free of any texture/FFI dependency. The sink must be cheap and non-blocking (drop-oldest); it is called on a decode task, never on a real-time audio thread.

fn push_frame(&self, stream_key: &str, w: u32, h: u32, i420: Vec<u8>)

Hand one decoded I420 frame to the renderer for stream_key. i420 is tightly packed: w*h + 2*(w/2 * h/2) bytes.

fn push_surface(&self, stream_key: &str, surface: RenderSurfaceHandle) -> bool

Zero-copy fast path: hand one decoded frame that is still GPU-resident (RenderSurfaceHandle) so the renderer can sample it without a CPU readback / colour conversion / re-upload. Returns true if the sink took it. The default returns false (“I can’t take a surface”), so the caller falls back to Self::push_frame with materialized I420 — meaning software decoders, non-GPU sinks (tests/headless), and platforms without a surface renderer all keep working unchanged. The handle is valid only for the duration of the call; an impl that keeps it must retain synchronously.

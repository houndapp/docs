Expand description

One live voice/video call. The supervisor task owns the live session(s) + every capture thread; this struct holds the cross-reconnect command seams + the supervisor handle. Dropping it (or CallSession::leave) cancels + tears down.

Construct with CallSession::join. The HoundClient holds at most one at a time (a [Mutex<Option<CallSession>>], see crate::app).

pub async fn join( rest: Arc<RestClient>, base_url: String, space_id: Uuid, sink: Arc<dyn VideoSink>, camera_enabled: bool, view: Sender<CallView>, connection: Sender<ConnectionState>, levels: Sender<LevelsView>, stats_out: Sender<StatsSnapshot>, identities: PeerIdentities, ) -> Result<Self>

Join the voice channel for space_id: run the first voice/join + establish_live, then spawn the auto-reconnect spawn_supervisor. Drives the three observable snapshots through channels. The sink receives decoded video frames. Device selection / mute / camera-enabled survive a reconnect (their shared cells live on ConnectionParams).

On the FIRST establish failing the call is NOT created (the error is returned and ConnectionState::Failed is published). After that, a layer drop heals via the supervisor without the user re-joining.

Source

pub fn set_muted(&self, muted: bool)

Mute or unmute the local mic. Gates the encoder locally (no audio is sent) AND tells the control plane so the roster reflects it for everyone. Routes to the CURRENT session via the shared outbound (survives a reconnect).

Source

pub fn set_camera_enabled(&self, enabled: bool)

Enable/disable the local camera. The capture task watches this flag and starts/stops capturing (no re-join needed); the flag is shared across reconnects, so the camera state survives a heal.

Also announces/withdraws the camera track on the control plane: the establish-time publish only fires when the camera is already on at join, so a mid-call enable must Publish here — otherwise capture + self-preview run but the call view has no camera track and peers never learn of it. A freshly published track is un-muted, so the announce alone makes the self tile render video.

Source

pub fn set_video_paused(&self, paused: bool)

Pause/resume inbound video. Paused, the subscribe loop drops every remote video subscription (camera + screen); audio keeps flowing. Driven by app foreground/visibility so a backgrounded call stops paying to receive + render a screenshare nobody is looking at. Idempotent; survives reconnects.

Source

pub fn set_tile_quality( &self, peer: u32, source: Source, width: u32, height: u32, visible: bool, )

Report a remote video tile’s rendered size + visibility, which the subscribe loop maps to a quality demand (small/hidden ⇒ the low layer, large ⇒ source) and folds across reconnects. Called by the UI per tile as it lays out / scrolls.

pub async fn screen_share_start(&self, source_id: String) -> Result<()>

Start sharing the screen source identified by source_id (a token from list_screen_sources). The command is routed to the supervisor (which OWNS the running share so it can republish it after a reconnect); we await its open result on a oneshot so a permission/unsupported error surfaces to the caller. Replaces any current share. A runtime capture end (window closed, OS “stop sharing”) just stops the track.

Source

pub fn screen_share_stop(&self)

Stop the active screen share (if any). Fire-and-forget to the supervisor, which tears down the capture/publish tasks and Unpublishes the screen track(s).

Source

pub fn set_screen_quality(&self, quality: ScreenQuality)

Set the live screen-share quality (motion↔legibility + a max resolution/fps cap). Takes effect on the next adaptation tick of a running share (the ladder is reconfigured live, NO restart) and is remembered for the next share + across a reconnect. Harmless with no active share — the preference is call-scoped and the next share seeds from it.

Source

pub fn set_stream_volume(&self, peer: u32, source: Source, volume: f32)

Set one remote audio stream’s playback volume (0.0 = muted, 1.0 = unity, clamped to ≤ 2.0) — the tile right-click volume/mute control. Applied by the stream’s decode pipeline before the mix, so it affects only this stream and takes effect within one audio frame (~20 ms). Local-only (the sender is not told); survives reconnects and layer switches.

Source

pub fn set_screen_audio_enabled(&self, enabled: bool)

Toggle whether an active screen share broadcasts its app audio (the broadcaster’s “share audio” control). Gates the publish loop before encode — a muted share costs zero CPU and zero wire data (the DTX-aware receivers read the silence as a quiet source, not a fault). Survives reconnects and share restarts; a share started later honours the current setting.

Source

pub async fn leave(self)

Best-effort voice/leave broadcast, then cancel + await the supervisor (which tears down the live session). Idempotent with drop.

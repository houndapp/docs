Handle one ScreenCommand inside the supervisor loop, against the CURRENT live session. Start replaces any running share (stop-old-first + unpublish), starts a fresh pipeline, records the desired source for cross-reconnect republish, and replies on the oneshot. Stop and Ended both tear the pipeline down + unpublish the tracks + drop the desire (a clean user/OS stop). Resized/Retry RESTART the same share in place (the share must NOT end on a resize or a transient glitch).

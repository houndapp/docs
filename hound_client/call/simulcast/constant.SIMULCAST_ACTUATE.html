const SIMULCAST_ACTUATE: bool = true;
Expand description

Gate for demand-driven simulcast. While false, [SimulcastPlanner] clamps to SOURCE_ONLY (the low layer is never spun up) — the demand pipeline still runs and is observable, but the publish path is byte-identical to a no-simulcast build.

true enables the low layer. This is SAFE to enable because the source encode is never dropped (the send_source=false optimization is intentionally not wired): a receiver only ever switches to the low track for a small/hidden tile it explicitly requested, so the blast radius of any low-layer issue is a thumbnail, never the full-quality source. Set live for the screen path; camera has no low producer yet (a camera demand harmlessly falls back to the source track).

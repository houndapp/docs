Expand description

Start the screen-share publish pipeline against the live relay session: spawn the !Send capture+encode thread, await its open (surfacing a permission/unsupported error), announce the screen video track (+ screen_audio when the source has an app-audio leg), and spawn the publish task(s). Returns the handle that stops it.

Mirrors spawn_cam_publish with screen-tuned differences: the encode geometry is the source aspect fitted to 1080p-class (not the fixed cam size); a paired app-audio leg is published on its own track; and a runtime capture end (window closed / OS “stop sharing”) just ends the track rather than being an error.

The screen adaptation reads the sharer’s SharedScreenQuality each tick and reconfigures its ladder live when it changes; the capture thread classifies its exit (window close = Ended, a held resize = Resized, a transient fault = Retry) and reports it via notify so the supervisor restarts (resize/retry) or stops (ended) the share — a share never ends on a resize or a transient glitch.

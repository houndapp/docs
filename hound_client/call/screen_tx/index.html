Module screen_tx 

Source
Expand description

Outbound screen share: open → !Send capture/encode thread → publish, plus the LOW simulcast rung and the classified exit protocol back to the supervisor (ScreenCaptureExit: a clean end vs a held resize vs a transient fault — a share must NOT end on a resize or a glitch). Static content is re-emitted on a slow cadence for fresh subscribers, and an idle source is liveness-probed so a terminated window ends the share instead of freezing on its last frame.

map_screen_capture_error 🔒
Map an engine screen-capture open error onto a typed HoundError the UI can route — a Screen-Recording TCC denial gets a distinct, actionable message.
screen_capture_encode_loop 🔒
The !Send screen capture+encode thread: open the engine ScreenShare (+ maybe ScreenAudio) and the H.264 encoder HERE (so neither crosses a thread boundary), report readiness (handing back the audio leg), then pull → self-preview tee → encode → forward. A static source produces no frames, so on an idle poll the last frame is re-emitted (timestamp advanced) at the static-refresh cadence so a fresh subscriber isn’t stuck on a black tile.
spawn_screen_publish 🔒
Start the screen-share publish pipeline against the live relay session: spawn the !Send capture+encode thread, await its open (surfacing a permission/unsupported error), announce the screen video track (+ screen_audio when the source has an app-audio leg), and spawn the publish task(s). Returns the handle that stops it.
spawn_screen_video_publish 🔒
The async screen VIDEO publish task: acquire the screen track, gate on a live subscriber, then forward each encoded AU while running the 1 s adaptation tick (screen ladder). Mirrors the camera publish task. Reads screen_quality each tick and reconfigures its ladder live (set_screen_quality) when the sharer changes it — no restart.
unpublish_screen_tracks 🔒
Unpublish our screen video + audio tracks on the control plane (best-effort) so the roster drops them for everyone. Resolved from the published CallView (the same roster own_track reads) against the live session’s outbound.

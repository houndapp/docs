Module subscribe 

Source
Expand description

The subscribe loop: reconcile remote tracks → decode → sink / mix bus. Diffs the SessionView roster into per-track pipelines (start_pipeline), picks source vs low layer per tile demand (debounced through the policy hysteresis), and self-heals: every pipeline carries a PipelineEndGuard whose Drop reports the death so the reconcile reaps + re-subscribes it (restart-paced). Invariant: audio subscribes with arrival-order drain and the smaller moq priority value — audio must strictly outrank video on the shared connection.

Structs§

DownlinkCell 🔒
Lock-free single-slot handle for one video pipeline’s downlink-driven demand: the video pull loop (video_pipeline) writes its latest [hound_media::QualityRequest] here as frames arrive (via [hound_media::DownlinkDemand]); the reconcile loop below reads it every tick and folds it with the tile demand via [hound_media::most_constrained]. A plain atomic (not a lock) so the per-frame write on the pull loop never blocks. None (audio, or a pipeline that hasn’t decided yet) reads as [hound_media::QualityRequest::Source] — the optimistic default, matching [hound_media::DownlinkDemand::new].
PipelineEndGuard 🔒
Fires (track_id, epoch) on the reap channel when the owning pipeline task ends — by ANY path: clean end-of-track, a transport error, a panic unwind, or an abort (tokio drops the future at an await point, running destructors). This is what lets the subscribe loop REAP a self-terminated pipeline: without it, a dead track would stay in active forever, every reconcile would skip it as “already subscribed”, and its tile would freeze permanently.
Subscription 🔒
One active remote subscription: the decode task + a stop flag.

Constants§

KEYFRAME_NAG_ATTEMPTS 🔒
KEYFRAME_NAG_INTERVAL 🔒
How often the receiver re-sends RequestKeyframe while a fresh video subscription is still awaiting its first keyframe, and how many times. The subscribe-time request can RACE the producer’s PLI-forced IDR (the IDR lands in a group the moq live-join skips), leaving the receiver black indefinitely on a keyframe that never comes. With the screen encoder on an on-demand cadence (no frequent periodic IDR to paper over the race), this bounded re-request is the fast recovery: producer-side the keyframe throttle coalesces duplicates, so nagging is cheap. Once the attempts are exhausted the pipeline’s [FIRST_KEYFRAME_DEADLINE] ends it — a fresh, ladder-paced resubscribe replaces a black tile that would otherwise wait on the safety-net IDR.
SOURCE_SENTINEL 🔒

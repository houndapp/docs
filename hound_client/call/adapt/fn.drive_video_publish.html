Drive one video publish loop: the 1 s adaptation tick + frame forwarding, with the live gate deciding when forwarding starts. The tick runs from the first iteration — before any subscriber — so PLIs are serviced and the BWE keeps state even while dark; only forwarding waits for proof of a receiver.

Live = the moq subscriber signal (MoqTrackPublisher::subscribed) OR control-plane demand (adapt.demand_seen, latched by the keyframe listener). The fallback matters when a broadcast is replaced on the same path (a share restart): moq-net keeps the newcomer as a backup until the old broadcast closes, so used() can arrive late or not at all — while the receiver’s subscribe always sends RequestKeyframe. Going live forces a fresh IDR, so the first delivered group always opens decodable; frames before the gate are dropped (a dropped keyframe re-arms the forced IDR).

before_tick runs ahead of each adaptation tick (the screen path reloads its live quality preference there). Returns when the encode channel closes or the track dies.

struct TransportStats {
    state: AtomicU8,
    reconnect_attempts: AtomicU32,
    rtt_us: AtomicU64,
    self_peer_id: AtomicU32,
    relay_url: Mutex<String>,
    sfu_node_id_short: Mutex<String>,
    playout_output_underruns: AtomicU64,
    playout_mix_starves: AtomicU64,
    tick_late_p50_us: AtomicU32,
    tick_late_p95_us: AtomicU32,
}
Expand description

Transport-section counters/gauges. reconnect_attempts is the only field that must survive a reconnect (it counts them), so the whole CallStats registry is created once per call and shared into every establish; CallStats::reset_session clears the per-establish fields but leaves this counter alone.

Fields§

§state: AtomicU8

Coarse connection state (see TransportState). Gauge.

§reconnect_attempts: AtomicU32

How many times the supervisor has re-established the session this call. Survives a reconnect (it counts them).

§rtt_us: AtomicU64

Last RTT sample in microseconds (written by the adaptation tick each second from conn.rtt). 0 means “not yet sampled / no path”. Gauge.

§self_peer_id: AtomicU32

Our control-plane peer id for the current session. Gauge (re-set each establish — a reconnect mints a fresh peer id).

§relay_url: Mutex<String>

The relay URL + short SFU node id, set once per establish. A Mutex<String> rather than an atomic because they are strings; written once on establish, read once per second by the emitter — no contention.

§sfu_node_id_short: Mutex<String>§playout_output_underruns: AtomicU64

CUMULATIVE playout-health counters mirrored from the mix bus each emit tick (CallStats::set_playout_health); the snapshot diffs them to a per-window rate. playout_output_underruns = output samples the device RT callback had to silence-pad (device-side gap); playout_mix_starves = per-peer mix slots that came up short at a 10 ms tick (per-peer gap). Both 0 in steady state; rising = audio stutter, localized (device path vs per-peer pump cadence).

§playout_mix_starves: AtomicU64§tick_late_p50_us: AtomicU32

Mix-tick lateness percentiles (µs) for the last window, mirrored from the bus each emit. The RT-health signal: ~0 when the bus thread keeps its 10 ms cadence, high when it is descheduled (CPU/encode contention) — which drains the device ring and stutters ALL audio at once. Gauges.

§tick_late_p95_us: AtomicU32

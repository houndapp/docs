Module rest

Expand description

The REST data plane: a thin, typed wrapper over the backend HTTP API. Holds the session bearer token in a lock-free cell (read on every request, republished by the auth refresh task) and maps transport/status failures onto HoundError.

Wire mapping discipline: responses are decoded against private wire DTOs (the backend’s serde, snake_case) and mapped into the client view-models in crate::model via into_model()/From. Wire quirks — the pagination envelope (contents/current/next/prev), the nested community spaces, the messages+users split, /profile/global, /community/members?id= — are confined to this module. The exact shapes live in services/backend/src/api/{auth,community,space,profile,data}.rs.

WebAuthn note: register/login start/finish carry the ceremony payloads as the TYPED hound-passkey wire types ([CreationChallengeResponse] / [RequestChallengeResponse] from the server, [RegisterPublicKeyCredential] / [PublicKeyCredential] from the authenticator) and (de)serialise the request / response bodies through hound_passkey::dto — the SAME vocabulary the backend uses, so the wire stays byte-identical. The client still does not interpret the ceremony content; it threads the typed value between the authenticator and the backend.

uuid_v7_millis 🔒
Extract the unix-ms timestamp embedded in a UUIDv7. The backend keys all ids with Uuid::now_v7(), whose high 48 bits are the creation time in ms — so a message’s id is its send time, sparing us a separate field the backend never sends. Returns 0 for non-v7 uuids (the timestamp bits are still read but meaningless).

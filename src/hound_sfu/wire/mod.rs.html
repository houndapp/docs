1//! # Wire protocol
2//!
3//! Everything in this module is about turning bytes on the wire into typed
4//! Rust values and back again. No network I/O happens here — callers hand us
5//! `&[u8]` slices and we hand back structs.
6//!
7//! ## Layout
8//!
9//! ```text
10//! frame.rs  — 16-byte FrameHeader (peer_id, seq, track type, timestamp, …)
11//! codec.rs  — codec negotiation types (VideoCodecId, AudioCodecId, …)
12//! error.rs  — WireError: every parse failure with a clear description
13//! ```
14//!
15//! The separation between `frame.rs` and `codec.rs` is intentional: a frame
16//! header knows *which track* a payload belongs to, but it doesn't care about
17//! *how* the payload was encoded. Codec information is negotiated once at
18//! connection start (via the Hello/HelloAck handshake) and then remembered by
19//! the room; it is not repeated in every packet.
20
21pub mod codec;
22pub mod error;
23pub mod frag;
24pub mod frame;
25pub mod hello;

1//! # SFU connection handler
2//!
3//! This module owns the iroh `Endpoint` and the `accept()` loop. When a new
4//! client connects it:
5//!
6//! 1. Opens the signalling stream and reads the [`Hello`] message.
7//! 2. Verifies the relay token and negotiates codecs.
8//! 3. Assigns the client a peer ID and sends [`HelloAck`].
9//! 4. Spawns two tasks per connection:
10//!    * **reader** — receives QUIC datagrams and reliable streams, parses the
11//!      frame header, and calls `Room::forward`.
12//!    * **writer** — reads from a `mpsc` channel of outgoing frames and writes
13//!      them as datagrams or reliable streams depending on the keyframe flag.
14//!
15//! ## Sans-IO seam
16//!
17//! The forwarding logic lives in [`crate::room::state::Room`] and is fully
18//! tested without iroh via [`crate::room::state::MockSink`]. This module is the
19//! thin glue that wires iroh connections to that pure state machine.
20//!
21//! ## Connection lifecycle
22//!
23//! ```text
24//! accept() ──► handshake() ──► spawn reader + writer tasks
25//!                │                     │
26//!                ▼                     ▼
27//!           Hello / HelloAck     Room::forward
28//!           (signalling stream)  (frame routing)
29//! ```
30

47// ─── ALPN ────────────────────────────────────────────────────────────────────
48
49/// ALPN protocol identifier for hound SFU connections.
50///
51/// iroh uses ALPN to route incoming connections to the correct handler. Any
52/// client that wants to speak the hound voice protocol must present this ALPN
53/// in its TLS handshake.
54pub const HOUND_ALPN: &[u8] = b"hound-sfu/1";
55
56/// How long [`reject`] waits for a rejected client to read its NACK and close
57/// the connection before the handler gives up and tears everything down.
58const NACK_DRAIN_TIMEOUT: std::time::Duration = std::time::Duration::from_secs(5);
59
60/// How long the SFU waits, after accepting a connection, for the client's
61/// `Hello` to arrive over the signalling stream before giving up. Generous
62/// enough to absorb relay/hole-punching setup, short enough that a client that
63/// connected but can't deliver data doesn't pin a handler task indefinitely.
64const HANDSHAKE_TIMEOUT: std::time::Duration = std::time::Duration::from_secs(15);
65
66// ─── SfuSink ─────────────────────────────────────────────────────────────────
67
68/// [`FrameSink`] implementation backed by a real iroh connection.
69///
70/// Frames queued via `try_send_datagram` / `try_send_reliable` are forwarded
71/// to the corresponding writer task over a bounded `mpsc` channel. The writer
72/// task runs in its own `tokio::task` and writes to the iroh connection without
73/// blocking the forwarding hot path.
74///
75/// The channel is intentionally small (128 frames). If the remote peer is slow
76/// to consume data the channel fills up and `try_send_*` returns `false`,
77/// dropping the frame. Real-time media prefers freshness over completeness.
78struct SfuSink {
79    datagram_tx: mpsc::Sender<Bytes>,
80    reliable_tx: mpsc::Sender<Bytes>,
81}
82
83impl FrameSink for SfuSink {
84    fn try_send_datagram(&self, frame: Bytes) -> bool {
85        self.datagram_tx.try_send(frame).is_ok()
86    }
87
88    fn try_send_reliable(&self, frame: Bytes) -> bool {
89        self.reliable_tx.try_send(frame).is_ok()
90    }
91}

95/// Thread-safe map from space_id → Room.
96///
97/// A room is created when the first participant joins a space and dropped when
98/// the last participant leaves. The `Arc<Mutex<Room>>` allows multiple
99/// connection-handler tasks to access the same room concurrently.
100pub type RoomRegistry = Arc<Mutex<HashMap<Uuid, Arc<Mutex<Room>>>>>;
101
102/// Create a new, empty room registry.
103pub fn new_room_registry() -> RoomRegistry {
104    Arc::new(Mutex::new(HashMap::new()))
105}

109/// The SFU server.
110///
111/// Holds the iroh endpoint and room registry. Call [`Sfu::run`] to start
112/// accepting connections.
113pub struct Sfu {
114    pub endpoint: Endpoint,
115    pub rooms: RoomRegistry,
116    relay_hmac_secret: String,
117}
118
119impl Sfu {
120    /// Bind a new iroh endpoint and return an `Sfu` ready to accept connections.
121    ///
122    /// `relay_url` is the iroh relay server used for NAT traversal. `secret_key`
123    /// is the iroh secret key for this node (determines the `EndpointId`).
124    /// `relay_hmac_secret` is the shared HMAC secret used to verify relay tokens.
125    /// `bind_addr` is the **exact** IPv4 socket the QUIC endpoint binds to.
126    ///
127    /// ## Why a specific bind address (not `0.0.0.0`)
128    ///
129    /// The SFU runs with `hostNetwork: true`, so the node has many interface
130    /// addresses: its public IP plus a pile of private/cluster ones (CNI,
131    /// docker bridge, the Kubernetes node InternalIP). iroh advertises the
132    /// addresses it can reach a peer on; **bound to `0.0.0.0` it enumerates and
133    /// advertises every interface address**, so clients spend the call probing
134    /// unreachable private IPs — paths open, fail validation, get abandoned, and
135    /// the churn destabilises the connection (the reliable `Hello` often never
136    /// lands). Bound to a **specific** address, iroh advertises only that one
137    /// (see `iroh::socket::update_direct_addresses`). So the caller resolves the
138    /// node's public IPv4 and passes `public_ip:LISTEN_PORT` here, giving clients
139    /// exactly one reachable address to dial. The port is fixed (not ephemeral)
140    /// so it matches the single UDP port opened on the node.
141    ///
142    /// If the public IP can't be determined the caller may pass `0.0.0.0`, which
143    /// restores the old enumerate-everything behaviour (correct, just noisy).
144    pub async fn bind(
145        relay_url: RelayUrl,
146        secret_key: SecretKey,
147        relay_hmac_secret: String,
148        bind_addr: std::net::SocketAddrV4,
149    ) -> eyre::Result<Self> {
150        let endpoint = Endpoint::builder(iroh::endpoint::presets::N0)
151            .secret_key(secret_key)
152            .relay_mode(RelayMode::Custom(iroh::RelayMap::from(relay_url)))
153            .alpns(vec![HOUND_ALPN.to_vec()])
154            .clear_ip_transports()
155            .bind_addr(std::net::SocketAddr::V4(bind_addr))
156            .map_err(|e| eyre::eyre!("invalid SFU bind address {bind_addr}: {e}"))?
157            .bind()
158            .await?;
159
160        Ok(Self::from_endpoint(endpoint, relay_hmac_secret))
161    }
162
163    /// Build an `Sfu` around an already-bound iroh [`Endpoint`].
164    ///
165    /// [`Sfu::bind`] is the production constructor: it builds an endpoint wired
166    /// to the iroh relay for NAT traversal. Tests need a *hermetic* endpoint
167    /// (relay disabled, direct loopback addressing) so they can run without
168    /// network access — they construct that endpoint themselves and hand it
169    /// here. Keeping the accept loop in [`Sfu::run`] identical for both paths is
170    /// the whole point: the e2e tests exercise the same connection-handling glue
171    /// that runs in production.
172    ///
173    /// The caller is responsible for having set `HOUND_ALPN` on the endpoint.
174    pub fn from_endpoint(endpoint: Endpoint, relay_hmac_secret: String) -> Self {
175        Self {
176            endpoint,
177            rooms: new_room_registry(),
178            relay_hmac_secret,
179        }
180    }
181
182    /// Accept connections in a loop until the endpoint closes.
183    ///
184    /// Each accepted connection is handed off to [`handle_connection`] in its
185    /// own tokio task. This function only returns on error or endpoint close.
186    pub async fn run(&self) -> eyre::Result<()> {
187        tracing::info!(
188            node_id = %self.endpoint.id(),
189            "SFU accepting connections"
190        );
191
192        loop {
193            let Some(incoming) = self.endpoint.accept().await else {
194                tracing::info!("SFU endpoint closed — stopping accept loop");
195                break;
196            };
197
198            let conn = match incoming.await {
199                Ok(c) => c,
200                Err(e) => {
201                    tracing::warn!("incoming connection failed: {e}");
202                    continue;
203                }
204            };
205
206            let rooms = self.rooms.clone();
207            let secret = self.relay_hmac_secret.clone();
208
209            tokio::spawn(async move {
210                if let Err(e) = handle_connection(conn, rooms, secret).await {
211                    tracing::warn!("connection handler error: {e:#}");
212                }
213            });
214        }
215
216        Ok(())
217    }
218
219    /// The iroh `EndpointId` for this SFU node.
220    ///
221    /// Return this in the backend's `VoiceJoinResponse.sfu_node_id` so that
222    /// clients know which node to dial.
223    pub fn node_id(&self) -> EndpointId {
224        self.endpoint.id()
225    }
226}

230/// Handle one incoming iroh connection from a voice client.
231async fn handle_connection(
232    conn: iroh::endpoint::Connection,
233    rooms: RoomRegistry,
234    relay_hmac_secret: String,
235) -> eyre::Result<()> {
236    let peer_endpoint_id = conn.remote_id();
237    let connected_at = std::time::Instant::now();
238    tracing::info!(%peer_endpoint_id, "connection accepted");
239
240    // ── Signalling handshake (reliable stream) ────────────────────────────
241    // Bound the whole "accept the bi-stream and read the Hello" phase. If the
242    // QUIC connection is established but the client's Hello never arrives — e.g.
243    // an unstable path where the relay link is flapping and direct paths fail —
244    // this otherwise hangs forever, leaking a handler task per stuck client and
245    // logging nothing past "connection accepted". The timeout converts that into
246    // a clear, bounded failure.
247    let (mut send_stream, mut recv_stream, msg_len) =
248        match tokio::time::timeout(HANDSHAKE_TIMEOUT, async {
249            let (send_stream, mut recv_stream) = conn.accept_bi().await?;
250            let mut len_buf = [0u8; 4];
251            recv_stream.read_exact(&mut len_buf).await?;
252            eyre::Ok((send_stream, recv_stream, u32::from_le_bytes(len_buf)))
253        })
254        .await
255        {
256            Ok(Ok(parts)) => parts,
257            Ok(Err(e)) => return Err(e),
258            Err(_) => {
259                tracing::warn!(
260                    %peer_endpoint_id,
261                    timeout_secs = HANDSHAKE_TIMEOUT.as_secs(),
262                    "handshake timed out waiting for Hello — dropping connection"
263                );
264                return Ok(());
265            }
266        };
267
268    let len_buf = msg_len.to_le_bytes();
269    let msg_len = msg_len as usize;
270    if msg_len > 4096 {
271        eyre::bail!("Hello message too large: {msg_len} bytes");
272    }
273    let mut msg_buf = vec![0u8; 4 + msg_len];
274    msg_buf[..4].copy_from_slice(&len_buf);
275    recv_stream.read_exact(&mut msg_buf[4..]).await?;
276
277    let hello: Hello = match decode_message(&msg_buf) {
278        Ok(h) => h,
279        Err(e) => {
280            return reject(
281                &conn,
282                &mut send_stream,
283                HelloNackReason::TokenInvalid,
284                &format!("malformed Hello: {e}"),
285            )
286            .await;
287        }
288    };
289
290    // ── Token verification ────────────────────────────────────────────────
291    let now = chrono::Utc::now().timestamp();
292    let claims = match verify_relay_token(&hello.token, &relay_hmac_secret, now) {
293        Ok(c) => c,
294        Err(e) => {
295            let reason = match &e {
296                crate::auth::TokenError::Expired { .. } => HelloNackReason::TokenExpired,
297                _ => HelloNackReason::TokenInvalid,
298            };
299            return reject(&conn, &mut send_stream, reason, &e.to_string()).await;
300        }
301    };
302
303    // ── Codec negotiation ─────────────────────────────────────────────────
304    let codecs = match negotiate_codecs(&hello.codecs) {
305        Some(c) => c,
306        None => {
307            return reject(
308                &conn,
309                &mut send_stream,
310                HelloNackReason::CodecMismatch,
311                "no common codec for one or more tracks",
312            )
313            .await;
314        }
315    };
316
317    // ── Room assignment ───────────────────────────────────────────────────
318    // Writer channels — the writer task drains these and pushes to iroh.
319    // Outgoing-frame channels for this peer's writer tasks. The reliable one is
320    // generous: dropping a reliable frame means dropping a keyframe, which leaves
321    // the receiver's decoder stuck until the *next* keyframe (visible as a frozen
322    // tile). The datagram channel can be smaller — dropping a delta is a brief
323    // blip the next keyframe fixes.
324    let (datagram_tx, mut datagram_rx) = mpsc::channel::<Bytes>(256);
325    let (reliable_tx, mut reliable_rx) = mpsc::channel::<Bytes>(256);
326
327    let sink = Arc::new(SfuSink {
328        datagram_tx,
329        reliable_tx,
330    });
331
332    let (peer_id, peers_snapshot) = {
333        let room_arc = {
334            let mut registry = rooms.lock().await;
335            registry
336                .entry(claims.space_id)
337                .or_insert_with(|| Arc::new(Mutex::new(Room::new(codecs))))
338                .clone()
339        };
340        let mut room_guard = room_arc.lock().await;
341        let peer_id = room_guard.assign_peer_id(claims.user_id, sink);
342        let peers: Vec<PeerSummary> = room_guard
343            .peer_summaries()
344            .into_iter()
345            .filter(|p| p.peer_id != peer_id)
346            .collect();
347        (peer_id, peers)
348    };
349
350    tracing::info!(
351        %claims.user_id,
352        %claims.space_id,
353        peer_id,
354        participants = peers_snapshot.len() + 1,
355        "peer joined room"
356    );
357
358    // ── HelloAck ──────────────────────────────────────────────────────────
359    let ack = ServerHandshake::Ack(HelloAck {
360        peer_id,
361        codecs,
362        peers: peers_snapshot,
363    });
364    let ack_bytes = encode_message(&ack)?;
365    send_stream.write_all(&ack_bytes).await?;
366    send_stream.finish()?;
367    drop(send_stream);
368
369    // ── Spawn writer tasks ────────────────────────────────────────────────
370    let conn_datagram = conn.clone();
371    tokio::spawn(async move {
372        while let Some(frame) = datagram_rx.recv().await {
373            if let Err(e) = conn_datagram.send_datagram(frame) {
374                tracing::debug!("datagram send error: {e}");
375            }
376        }
377    });
378
379    let conn_reliable = conn.clone();
380    tokio::spawn(async move {
381        while let Some(frame) = reliable_rx.recv().await {
382            match conn_reliable.open_uni().await {
383                Ok(mut stream) => {
384                    if let Err(e) = stream.write_all(&frame).await {
385                        tracing::debug!("reliable stream write error: {e}");
386                    }
387                    let _ = stream.finish();
388                }
389                Err(e) => tracing::debug!("open_uni error: {e}"),
390            }
391        }
392    });
393
394    // ── Reader loop ───────────────────────────────────────────────────────
395    // Datagrams and incoming reliable streams run concurrently.
396    let space_id = claims.space_id;
397    let rooms_reader = rooms.clone();
398
399    let result = tokio::select! {
400        r = read_datagrams(&conn, peer_id, space_id, rooms_reader.clone()) => r,
401        r = read_reliable_streams(&conn, peer_id, space_id, rooms_reader.clone()) => r,
402    };
403
404    if let Err(e) = result {
405        tracing::debug!(%claims.user_id, "connection closed: {e}");
406    }
407
408    // ── Cleanup ───────────────────────────────────────────────────────────
409    {
410        let mut registry = rooms.lock().await;
411        let empty = if let Some(room_arc) = registry.get(&space_id) {
412            let mut room = room_arc.lock().await;
413            room.remove_peer(peer_id);
414            tracing::info!(
415                %claims.user_id,
416                peer_id,
417                participants = room.peer_count(),
418                duration_secs = connected_at.elapsed().as_secs(),
419                "peer left room"
420            );
421            room.is_empty()
422        } else {
423            false
424        };
425        if empty {
426            registry.remove(&space_id);
427            tracing::info!(%space_id, "room closed (no participants)");
428        }
429    }
430
431    Ok(())
432}

434/// Drain QUIC datagrams from the connection and forward them via the room.
435async fn read_datagrams(
436    conn: &iroh::endpoint::Connection,
437    from: PeerId,
438    space_id: Uuid,
439    rooms: RoomRegistry,
440) -> eyre::Result<()> {
441    loop {
442        let datagram = conn.read_datagram().await?;
443        if datagram.len() < HEADER_SIZE {
444            tracing::debug!(from, "datagram too short ({} B) — skipping", datagram.len());
445            continue;
446        }
447        let header = match FrameHeader::parse(&datagram) {
448            Ok(h) => h,
449            Err(e) => {
450                tracing::debug!("bad frame header: {e}");
451                continue;
452            }
453        };
454        let is_keyframe = header.is_keyframe();
455        let frame = Bytes::copy_from_slice(&datagram);
456
457        if let Some(room_arc) = rooms.lock().await.get(&space_id).cloned() {
458            room_arc.lock().await.forward(from, frame, is_keyframe);
459        }
460    }
461}

463/// Drain incoming reliable (uni-directional) streams and forward them via the room.
464async fn read_reliable_streams(
465    conn: &iroh::endpoint::Connection,
466    from: PeerId,
467    space_id: Uuid,
468    rooms: RoomRegistry,
469) -> eyre::Result<()> {
470    loop {
471        let mut stream = conn.accept_uni().await?;
472        // Read the full frame (streams carry exactly one frame). A whole keyframe
473        // now arrives on a single stream, so allow up to 4 MB.
474        let data = stream.read_to_end(4 * 1024 * 1024).await?;
475        if data.len() < HEADER_SIZE {
476            continue;
477        }
478        let header = match FrameHeader::parse(&data) {
479            Ok(h) => h,
480            Err(e) => {
481                tracing::debug!("bad reliable frame header: {e}");
482                continue;
483            }
484        };
485        let is_keyframe = header.is_keyframe();
486        let frame = Bytes::from(data);
487
488        if let Some(room_arc) = rooms.lock().await.get(&space_id).cloned() {
489            room_arc.lock().await.forward(from, frame, is_keyframe);
490        }
491    }
492}

494/// Reject a connection: send a [`HelloNack`] on the signalling stream, then wait
495/// for the client to drain it before the connection is torn down.
496///
497/// ## Why wait?
498///
499/// QUIC's `CONNECTION_CLOSE` frame can overtake buffered stream data. If we send
500/// the NACK and immediately return — dropping the `Connection` — the client
501/// typically observes a generic `ConnectionLost` and never sees *why* it was
502/// rejected (expired token? wrong secret? codec mismatch?). That turns every
503/// rejection into an indistinguishable silent failure on the client.
504///
505/// Per iroh's own guidance, the only reliable delivery signal the sender has is
506/// the peer closing the connection after it has read the data. So after writing
507/// the NACK we await [`Connection::closed`], bounded by a timeout so a
508/// misbehaving or vanished client can't pin the task open forever.
509///
510/// [`HelloNack`]: crate::wire::hello::HelloNack
511/// [`Connection::closed`]: iroh::endpoint::Connection::closed
512async fn reject(
513    conn: &iroh::endpoint::Connection,
514    stream: &mut iroh::endpoint::SendStream,
515    reason: HelloNackReason,
516    message: &str,
517) -> eyre::Result<()> {
518    tracing::warn!(?reason, %message, "rejecting connection");
519    let nack = ServerHandshake::Nack(crate::wire::hello::HelloNack {
520        reason,
521        message: message.to_owned(),
522    });
523    let bytes = encode_message(&nack)?;
524    stream.write_all(&bytes).await?;
525    stream.finish()?;
526
527    // Let the client read the NACK and close the connection itself. The timeout
528    // is a safety valve — we don't care about the close reason, only that we
529    // gave the bytes a chance to land.
530    let _ = tokio::time::timeout(NACK_DRAIN_TIMEOUT, conn.closed()).await;
531    Ok(())
532}

1//! # MoQ relay — the native forward plane
2//!
3//! The MoQ forward plane: a thin, forward-only relay that fans every publisher's
4//! encoded media out to the other subscribers in the same space. This is the
5//! only media path the SFU serves — clients dial the [`RELAY_ALPN`] (`moql`)
6//! ALPN, negotiated via [`crate::sfu::Sfu`].
7//!
8//! ## Forward-only, no transcode
9//!
10//! The relay never decodes media. It re-emits the encoded objects a publisher
11//! writes to every subscriber of that track, byte-for-byte. moq-net owns the
12//! actual fan-out and per-subscriber prioritisation: each GOP is one QUIC
13//! uni-stream and the consumer's `next_group` priority-drops a stale group at
14//! its boundary, so a receiver never freezes replaying stale deltas while a
15//! fresh keyframe group waits. We do not hand-roll routing tables, NACK caches,
16//! or SSRC/seq rewrites — that is exactly what MoQ's group/object model gives
17//! us for free.
18//!
19//! ## How moq-net does the forwarding (the Origin)
20//!
21//! moq-net routes through a shared [`moq_net::Origin`]: a tree of broadcast
22//! paths. A [`moq_net::Server`] session is wired with two halves of that origin:
23//!
24//! * `with_consume(OriginProducer)` — "consume any **remote** broadcasts,
25//!   inserting them into this origin" (the publisher half: a connected publisher
26//!   announces a broadcast and moq-net inserts it into our shared origin).
27//! * `with_publish(OriginConsumer)` — "publish any **local** broadcasts from
28//!   this origin to the remote" (the subscriber half: a connected subscriber's
29//!   SUBSCRIBE is served from whatever the shared origin holds).
30//!
31//! Give every connection both halves of **one shared root origin** and moq-net
32//! itself forwards: publisher A announces `space/X/peer/A/video` into the origin
33//! → subscriber B's session sees the announce and serves the objects. The relay
34//! code never copies a byte; it only owns the origin and the auth gate.
35//!
36//! ## Auth gate (the mandatory bit)
37//!
38//! moq-net itself does not authenticate. Before any media flows we:
39//!
40//! 1. **Validate the relay-token JWT-equivalent** the client presents on a
41//!    dedicated control bi-stream, reusing [`crate::auth::verify_relay_token`]
42//!    (the same HMAC-SHA256 `{user}:{space}:{expiry}:{mac}` scheme the backend
43//!    mints). A bad/expired/missing token is rejected before the MoQ handshake
44//!    runs. The whole exchange is bounded by [`RelayConfig::auth_deadline`] (see
45//!    [`handle_connection`]) — a client that completes the QUIC handshake but
46//!    never opens/finishes the token stream cannot hold the connection open
47//!    indefinitely.
48//! 2. **Admit the session against [`SessionLimits`]** — a global cap and a
49//!    per-user cap, checked right after a successful token verification and
50//!    before the moq-net handshake runs (see [`SessionRegistry`]). Over-cap is a
51//!    clean [`AUTH_REJECT`], not a dropped connection.
52//! 3. **Scope the connection to its space** — always, both halves. From the
53//!    validated [`crate::auth::RelayTokenClaims`] we derive the namespace prefix
54//!    `space/{space_id}` and hand the moq-net session an origin **scoped** to it
55//!    via [`moq_net::OriginProducer::scope`] / [`moq_net::OriginConsumer::scope`].
56//!    A scoped producer's `publish_broadcast` returns `false` for any path
57//!    outside the prefix, and a scoped consumer never sees announcements outside
58//!    it — so a participant authenticated for space A **cannot** announce or
59//!    subscribe to space B's namespace. moq-net enforces the boundary; we just
60//!    set it from the token.
61//! 4. **Scope the INGEST half to the caller's own peer namespace, always.** The
62//!    two halves of a session are scoped independently (see
63//!    [`handle_connection`]): the SERVE half (`with_publish`, subscribing)
64//!    stays `space/{space_id}` — a participant must be able to subscribe to
65//!    anyone in the call. The INGEST half (`with_consume`, announcing) is
66//!    tightened to `space/{space_id}/peer/{user_id}`, so a connection
67//!    authenticated as user A can only announce broadcasts under its own
68//!    `user_id` sub-path — it can never publish (or displace, via moq-net's
69//!    route tie-break) under another peer's path. This is what makes the
70//!    `peer/{user_id}/{peer_id}/{track}` tail of [`NAMESPACE_CONVENTION`] a
71//!    real boundary rather than just a naming convention.
72//!
73//! This is **call-membership + per-user-namespace** auth. Per-track allow-listing
74//! (which *peer* may publish which *track* within a space, driven by
75//! `hound-control`'s `SessionServer` roster) remains a later refinement.
76//!
77//! ## Ongoing per-connection budgets (after admission)
78//!
79//! Auth and [`SessionRegistry::admit`] are one-time checks at connection
80//! start. A connection that passes both can still misbehave for the rest of
81//! its life — flooding ingress bytes, or announcing an unbounded number of
82//! broadcast paths — so [`handle_connection`] also spawns a per-connection
83//! [`ingress_watchdog`] that runs for the session's whole lifetime, enforcing
84//! [`IngressLimits`] and logging a periodic ops summary. See that function's
85//! doc for the enforcement details and close-code scheme.

100/// MoQ ALPN the relay accepts. Must be a moq-net ALPN: in raw mode
101/// `web-transport-iroh` reports the iroh connection's ALPN as the MoQ protocol
102/// and moq-net rejects an unrecognised one with `Error::UnknownAlpn`. `moql` is
103/// moq-net's version-negotiation ALPN and matches the client
104/// (`hound_transport::ALPN`). This is the only ALPN the SFU endpoint advertises.
105pub const RELAY_ALPN: &[u8] = b"moql";

107/// Marker documenting that [`RELAY_ALPN`] is the moq-net negotiation ALPN, not a
108/// free-to-choose `hound-*` value (the latter fails the handshake).
109pub const RELAY_ALPN_PREFIX: &[u8] = b"moq";

111/// The broadcast-namespace convention every announce/subscribe must obey.
112///
113/// ```text
114/// space/{space_id}/peer/{user_id}/{peer_id}/{track}
115/// ```
116///
117/// `user_id` is the AUTHENTICATED identity from the relay token / `PeerInfo`
118/// roster (a UUID) — the segment the relay enforces. `peer_id` is the
119/// control-plane-assigned `u32` tail that keeps a single user's multiple
120/// devices/sessions distinct within their own namespace.
121///
122/// The relay ALWAYS scopes each authenticated connection's SERVE (subscribe)
123/// half to the `space/{space_id}` prefix taken from its relay token — moq-net
124/// refuses any subscribe outside it (no cross-space leakage). The INGEST
125/// (publish) half is ALWAYS additionally tightened to
126/// `space/{space_id}/peer/{user_id}`, so a connection can only announce under
127/// its own authenticated user's sub-path — see [`handle_connection`] and
128/// [`peer_scope`]. Per-track allow-listing (e.g. gating who may publish a
129/// specific `hound-control` `TrackId`) remains a later refinement.
130pub const NAMESPACE_CONVENTION: &str = "space/{space_id}/peer/{user_id}/{peer_id}/{track}";

132/// Maximum relay-token length accepted on the control stream. A valid token is
133/// `36 + 1 + 36 + 1 + ~10 + 1 + 64 ≈ 150` bytes; this bound stops a malicious
134/// client pinning memory with a giant "token".
135const MAX_TOKEN_LEN: usize = 512;

137/// One-byte control-stream replies the relay sends after token verification.
138/// The client gates its MoQ handshake on reading [`AUTH_OK`].
139const AUTH_OK: u8 = 1;
140const AUTH_REJECT: u8 = 0;

142/// How long the relay waits, after sending an [`AUTH_REJECT`], for the client to
143/// read the verdict and close the connection before the handler gives up. A
144/// safety valve so a vanished client can't pin the task; we only care that the
145/// reject byte had a chance to land.
146const AUTH_REJECT_DRAIN: Duration = Duration::from_secs(5);
147
148/// Deadline for the whole relay-token control-stream handshake (`authenticate`:
149/// `accept_bi` + the two `read_exact`s). QUIC's automatic keepalives mean the
150/// idle timeout never fires on its own, so a bound here is the only thing
151/// stopping a client that completes the QUIC handshake but never opens (or
152/// never finishes) the token stream from holding the connection — and the task
153/// serving it — open indefinitely. [`SessionRegistry`] admission only ever runs
154/// after a successful auth (see `handle_connection`), so this deadline is also
155/// what keeps an unauthenticated connection from ever holding a session slot or
156/// blocking the SIGTERM drain wait: bounding "unauthenticated" bounds the
157/// exposure. Production default (see [`RelayConfig::default`]); tests
158/// substitute a much shorter value to exercise the timeout deterministically.
159pub const AUTH_DEADLINE: Duration = Duration::from_secs(10);

182/// Ongoing per-connection ingress budget, enforced continuously by
183/// [`ingress_watchdog`] for the life of an admitted session — unlike
184/// [`SessionLimits`] (checked once at admission), this is a running check
185/// against traffic the connection generates after it's already in the call.
186#[derive(Debug, Clone, Copy)]
187pub struct IngressLimits {
188    /// Sustained ingress cap in bytes/sec, read from `conn.stats().udp_rx.bytes`
189    /// (cumulative QUIC bytes received — the actual cost to the node, not just
190    /// media payload). `0` disables the budget. The default (~24 Mbit/s) is
191    /// generous headroom over the heaviest legitimate publisher (source + low
192    /// simulcast video, screenshare, audio).
193    pub max_bytes_per_sec: u64,
194    /// How often [`ingress_watchdog`] samples `conn.stats()`. Kept separate
195    /// from `window` (rather than derived from it) so tests can shrink both
196    /// independently instead of waiting out the production window.
197    pub sample_interval: Duration,
198    /// Sustained-rate averaging span handed to [`IngressBudget::new`] — a
199    /// spike shorter than this self-corrects (see that type's doc); a rate
200    /// held for the whole window trips.
201    pub window: Duration,
202    /// Maximum distinct broadcast paths this connection's authenticated
203    /// ingest scope (`space/{s}/peer/{user_id}`) may have announced at once
204    /// before [`ingress_watchdog`] closes it. `0` disables enforcement.
205    /// Bounds unbounded origin-tree growth / task spawn from one connection
206    /// announcing broadcasts without limit. NOTE: scoped to the
207    /// authenticated `(space, user)` ingest namespace, not the raw QUIC
208    /// connection identity — see [`ingress_watchdog`]'s doc for why, and the
209    /// practical effect when the same user holds multiple concurrent
210    /// sessions (bounded separately by [`SessionLimits::max_per_user`]).
211    pub max_broadcasts_per_session: usize,
212}
213
214impl Default for IngressLimits {
215    /// Production defaults — see `main.rs`'s `MAX_INGRESS_BYTES_PER_SEC` /
216    /// `MAX_BROADCASTS_PER_SESSION` CLI flags, which override
217    /// `max_bytes_per_sec` / `max_broadcasts_per_session`. `sample_interval`
218    /// and `window` are not exposed as CLI flags (only tests need to shrink
219    /// them).
220    fn default() -> Self {
221        Self {
222            max_bytes_per_sec: 3_000_000,
223            sample_interval: Duration::from_secs(5),
224            window: Duration::from_secs(30),
225            max_broadcasts_per_session: 16,
226        }
227    }
228}

270/// Why [`SessionRegistry::admit`] refused a session — carries the count that
271/// triggered the rejection, purely for the log line at the call site.
272#[derive(Debug, Clone, Copy, PartialEq, Eq)]
273enum AdmitRejected {
274    /// The global cap was at (or, transiently, just over) this count.
275    Global(usize),
276    /// This user's own cap was at this count.
277    PerUser(usize),
278}
279
280/// Shared session accounting: a global count plus a per-user count, gating
281/// admission at [`SessionLimits`] and released via [`SessionGuard`]. Cheap to
282/// clone (every field is `Arc`-backed) — each accept-loop task holds its own
283/// clone.
284///
285/// Only *authenticated* sessions are ever admitted here (see
286/// [`handle_connection`]), so [`SessionRegistry::active_sessions`] — the
287/// counter `main`'s SIGTERM drain wait polls — never counts a connection that
288/// never passed the auth gate.
289#[derive(Clone)]
290pub struct SessionRegistry {
291    active: Arc<AtomicUsize>,
292    per_user: Arc<Mutex<HashMap<Uuid, usize>>>,
293    limits: SessionLimits,
294}
295
296impl SessionRegistry {
297    pub fn new(limits: SessionLimits) -> Self {
298        Self {
299            active: Arc::new(AtomicUsize::new(0)),
300            per_user: Arc::new(Mutex::new(HashMap::new())),
301            limits,
302        }
303    }
304
305    /// Shared handle on the global active-session counter.
306    pub fn active_sessions(&self) -> Arc<AtomicUsize> {
307        self.active.clone()
308    }
309
310    /// Try to admit one session for `user_id`.
311    ///
312    /// Reserves global capacity with `fetch_add` first and rolls it back on
313    /// rejection, rather than check-then-increment, so two concurrent
314    /// accept-loop tasks racing the last slot can't both observe "space
315    /// available" and overshoot the cap. `Ok` also bumps the per-user count and
316    /// returns the RAII guard that reverses both counters on drop; `Err` leaves
317    /// every counter exactly as it found it.
318    fn admit(&self, user_id: Uuid) -> Result<SessionGuard, AdmitRejected> {
319        let active = self.active.fetch_add(1, Ordering::SeqCst) + 1;
320        if active > self.limits.max_sessions {
321            self.active.fetch_sub(1, Ordering::SeqCst);
322            return Err(AdmitRejected::Global(active - 1));
323        }
324
325        let mut per_user = self.per_user.lock().unwrap();
326        let count = per_user.entry(user_id).or_insert(0);
327        if *count >= self.limits.max_per_user {
328            let count = *count;
329            drop(per_user);
330            self.active.fetch_sub(1, Ordering::SeqCst);
331            return Err(AdmitRejected::PerUser(count));
332        }
333        *count += 1;
334        drop(per_user);
335
336        Ok(SessionGuard {
337            active: self.active.clone(),
338            per_user: self.per_user.clone(),
339            user_id,
340        })
341    }
342}

344/// RAII guard for one admitted session: decrements the global count and its
345/// user's per-user entry on drop (removing the entry once it reaches zero), so
346/// a panic or early return anywhere in the connection handler still releases
347/// capacity — admission can never wedge on a phantom session.
348pub struct SessionGuard {
349    active: Arc<AtomicUsize>,
350    per_user: Arc<Mutex<HashMap<Uuid, usize>>>,
351    user_id: Uuid,
352}
353
354impl Drop for SessionGuard {
355    fn drop(&mut self) {
356        self.active.fetch_sub(1, Ordering::SeqCst);
357        let mut per_user = self.per_user.lock().unwrap();
358        if let std::collections::hash_map::Entry::Occupied(mut entry) = per_user.entry(self.user_id)
359        {
360            *entry.get_mut() -= 1;
361            if *entry.get() == 0 {
362                entry.remove();
363            }
364        }
365    }
366}

385/// Create a fresh, wildcard-scoped root [`OriginProducer`] — the shared origin
386/// through which moq-net forwards every broadcast.
387///
388/// Every accepted relay connection gets a *space-scoped view* of this same
389/// origin (see [`handle_relay_connection`]), so a publisher and a subscriber in
390/// the same space meet here and moq-net fans the objects out. [`Relay`] builds
391/// one internally; the dual-ALPN SFU loop ([`crate::sfu::Sfu`]) builds one too so
392/// it can serve the relay on its own endpoint without constructing a [`Relay`].
393pub(crate) fn new_root_origin() -> OriginProducer {
394    Origin::random().produce()
395}
396
397/// Build the `space/{space_id}` namespace prefix a connection is scoped to.
398///
399/// This is the call-membership boundary: everything a participant may announce
400/// or subscribe lives under this prefix, and moq-net rejects anything outside
401/// it. The SERVE (subscribe) half is ALWAYS scoped to this — see [`peer_scope`]
402/// for the tighter INGEST-half prefix.
403fn space_scope(claims: &RelayTokenClaims) -> String {
404    format!("space/{}", claims.space_id)
405}
406
407/// Build the `space/{space_id}/peer/{user_id}` namespace prefix the INGEST
408/// (publish) half is ALWAYS scoped to. `user_id` comes from the validated relay
409/// token, never the client — so a connection can only announce broadcasts under
410/// its own authenticated user's sub-path, closing the "any space member can
411/// announce under (and displace) another peer's path" hole.
412fn peer_scope(claims: &RelayTokenClaims) -> String {
413    format!("space/{}/peer/{}", claims.space_id, claims.user_id)
414}

416/// The MoQ relay server.
417///
418/// Owns the iroh [`Endpoint`], the shared [`OriginProducer`] root through which
419/// moq-net forwards every broadcast, the relay-token HMAC secret, and the
420/// [`SessionRegistry`] admission accounting. Call [`Relay::run`] to accept
421/// connections.
422///
423/// This is a standalone-relay convenience used by tests; the production binary
424/// drives the same [`handle_connection`] logic through [`crate::sfu::Sfu`]
425/// instead so the service endpoint is a single iroh node.
426pub struct Relay {
427    endpoint: Endpoint,
428    /// The shared root origin (wildcard scope). Every accepted connection gets a
429    /// space-scoped view of this same origin, so a publisher in space X and a
430    /// subscriber in space X meet here and moq-net fans the objects out.
431    origin: OriginProducer,
432    relay_hmac_secret: String,
433    registry: SessionRegistry,
434    auth_deadline: Duration,
435    /// Per-connection ingress-watchdog budget — see [`IngressLimits`].
436    ingress: IngressLimits,
437}

439impl Relay {
440    /// Build a relay around an already-bound iroh [`Endpoint`].
441    ///
442    /// The caller is responsible for having advertised [`RELAY_ALPN`] on the
443    /// endpoint. Mirrors [`crate::sfu::Sfu::from_endpoint`] so the hermetic
444    /// tests exercise the same accept/auth glue that runs in production.
445    pub fn from_endpoint(
446        endpoint: Endpoint,
447        relay_hmac_secret: String,
448        config: RelayConfig,
449    ) -> Self {
450        Self {
451            endpoint,
452            origin: new_root_origin(),
453            relay_hmac_secret,
454            registry: SessionRegistry::new(config.limits),
455            auth_deadline: config.auth_deadline,
456            ingress: config.ingress,
457        }
458    }
459
460    /// The iroh node id clients dial to reach this relay.
461    pub fn node_id(&self) -> iroh::EndpointId {
462        self.endpoint.id()
463    }
464
465    /// Shared handle on the active-session counter — see
466    /// [`SessionRegistry::active_sessions`]. Exposed for tests asserting that a
467    /// rejected/timed-out connection never touches it.
468    pub fn active_sessions(&self) -> Arc<AtomicUsize> {
469        self.registry.active_sessions()
470    }
471
472    /// Accept connections in a loop until the endpoint closes.
473    ///
474    /// Each connection is handed to [`handle_connection`] in its own task. The
475    /// shared origin and secret are cloned per connection (both cheap: the
476    /// origin is `Arc`-backed internally, the secret is a short `String`).
477    pub async fn run(&self) -> eyre::Result<()> {
478        tracing::info!(node_id = %self.endpoint.id(), "MoQ relay accepting connections");
479
480        loop {
481            let Some(incoming) = self.endpoint.accept().await else {
482                tracing::info!("MoQ relay endpoint closed — stopping accept loop");
483                break;
484            };
485
486            let conn = match incoming.await {
487                Ok(c) => c,
488                Err(e) => {
489                    tracing::warn!("incoming relay connection failed: {e}");
490                    continue;
491                }
492            };
493
494            let origin = self.origin.clone();
495            let secret = self.relay_hmac_secret.clone();
496            let registry = self.registry.clone();
497            let auth_deadline = self.auth_deadline;
498            let ingress = self.ingress;
499            tokio::spawn(async move {
500                if let Err(e) =
501                    handle_connection(conn, origin, secret, registry, auth_deadline, ingress).await
502                {
503                    tracing::warn!("relay connection handler error: {e}");
504                }
505            });
506        }
507
508        Ok(())
509    }
510}

538/// Handle one incoming MoQ relay connection: authenticate, admit, scope, run
539/// the moq-net server session.
540///
541/// ## Stream ordering (why the token stream is accepted first)
542///
543/// The client opens the **relay-token control bi-stream first** and awaits the
544/// relay's [`AUTH_OK`] before starting its MoQ handshake. The relay accepts that
545/// bi-stream (`conn.accept_bi`) before handing the connection to
546/// [`moq_net::Server::accept`] (which opens *its own* control stream). QUIC
547/// bi-streams are accepted in open order, so the token exchange always lands
548/// before the MoQ control stream. If auth fails the connection is closed and the
549/// MoQ handshake never runs.
550///
551/// ## Ordering of auth vs. admission vs. the verdict byte
552///
553/// [`authenticate`] verifies the token but deliberately does NOT write
554/// [`AUTH_OK`] on success — it hands back the still-open send half instead. That
555/// lets this function run the [`SessionRegistry::admit`] cap check *before*
556/// committing to a verdict, so an over-cap connection still gets a clean single-
557/// byte [`AUTH_REJECT`] (the same wire shape as a bad token) rather than
558/// silently being admitted and then killed, or racing a verdict it already sent.
559async fn handle_connection(
560    conn: iroh::endpoint::Connection,
561    origin: OriginProducer,
562    relay_hmac_secret: String,
563    registry: SessionRegistry,
564    auth_deadline: Duration,
565    ingress: IngressLimits,
566) -> Result<(), RelayError> {
567    let peer = conn.remote_id();
568    tracing::info!(%peer, "relay connection accepted");
569
570    // ── Relay-token auth (control bi-stream, bounded) ──────────────────────
571    let (claims, mut auth_send) = match tokio::time::timeout(
572        auth_deadline,
573        authenticate(&conn, &relay_hmac_secret),
574    )
575    .await
576    {
577        Ok(Ok(pair)) => pair,
578        Ok(Err(e)) => {
579            tracing::warn!(%peer, "relay auth rejected: {e}");
580            // Don't slam `conn.close` here: a `CONNECTION_CLOSE` frame can
581            // overtake the buffered reject byte, so the client would observe
582            // a generic connection error instead of the explicit reject
583            // verdict. Wait (bounded) for the client to read the verdict and
584            // close the connection itself, giving the reject byte a chance
585            // to land.
586            let _ = tokio::time::timeout(AUTH_REJECT_DRAIN, conn.closed()).await;
587            return Err(e);
588        }
589        Err(_) => {
590            // The client completed the QUIC handshake but never opened, or
591            // never finished, the token control stream — nothing was ever
592            // sent to drain, so close outright rather than leave the task
593            // and connection open behind QUIC's automatic keepalives.
594            tracing::warn!(%peer, deadline = ?auth_deadline, "relay auth handshake timed out — closing");
595            conn.close(0u32.into(), b"auth timeout");
596            return Err(RelayError::Auth("auth handshake timed out".into()));
597        }
598    };
599
600    // ── Session admission (after auth, before moq-net) ──────────────────────
601    let _session_guard = match registry.admit(claims.user_id) {
602        Ok(guard) => guard,
603        Err(reason) => {
604            tracing::warn!(
605                %peer, %claims.user_id, %claims.space_id, ?reason,
606                "relay session rejected — over capacity"
607            );
608            let _ = reject(&mut auth_send).await;
609            let _ = tokio::time::timeout(AUTH_REJECT_DRAIN, conn.closed()).await;
610            return Err(RelayError::Auth(format!(
611                "session rejected: over capacity ({reason:?})"
612            )));
613        }
614    };
615    auth_send
616        .write_all(&[AUTH_OK])
617        .await
618        .map_err(|e| RelayError::Iroh(format!("write auth ok: {e}")))?;
619    let _ = auth_send.finish();
620
621    // ── Namespace scoping (the call-membership + per-user boundary) ────────
622    // The two halves are scoped INDEPENDENTLY from the shared root origin (not
623    // chained) because they need different prefixes:
624    //   * SERVE (subscribe, `with_publish`) — `space/{id}`: a participant must
625    //     be able to subscribe to anyone else in the call.
626    //   * INGEST (announce, `with_consume`) — `space/{id}/peer/{user_id}`: a
627    //     connection can only announce under its own authenticated identity.
628    let scope = space_scope(&claims);
629    let scope_path = Path::new(&scope);
630    let scoped_serve_producer = origin
631        .scope(std::slice::from_ref(&scope_path))
632        .ok_or_else(|| RelayError::Moq(format!("origin scope rejected namespace {scope:?}")))?;
633    // `.consume()` derives an OriginConsumer inheriting the producer's scope —
634    // this is the `with_publish` half, so subscribers here see any broadcast
635    // announced anywhere in the space, regardless of the ingest scope below.
636    let scoped_consume = scoped_serve_producer.consume();
637
638    let ingest_scope = peer_scope(&claims);
639    let ingest_path = Path::new(&ingest_scope);
640    let scoped_publish = origin
641        .scope(std::slice::from_ref(&ingest_path))
642        .ok_or_else(|| {
643            RelayError::Moq(format!("origin scope rejected namespace {ingest_scope:?}"))
644        })?;
645    // A second cursor over the same ingest scope, dedicated to the watchdog's
646    // broadcast-count budget (see `ingress_watchdog`). `consume()` takes `&self`
647    // so this doesn't disturb `scoped_publish`, which is still moved into
648    // `with_consume` below.
649    let ingest_announces = scoped_publish.consume();
650
651    tracing::info!(
652        %peer, %claims.user_id, %claims.space_id,
653        serve_namespace = %scope, ingest_namespace = %ingest_scope,
654        "relay connection authorised"
655    );
656
657    // ── MoQ server handshake + session ─────────────────────────────────────
658    // `web-transport-iroh` raw mode: speak MoQ directly over iroh, no HTTP/3.
659    let wt = web_transport_iroh::Session::raw(conn.clone());
660    let session = moq_net::Server::new()
661        .with_consume(scoped_publish)
662        .with_publish(scoped_consume)
663        .accept(wt)
664        .await
665        .map_err(|e| RelayError::Moq(format!("server accept: {e}")))?;
666
667    // Ingress watchdog: bandwidth budget + broadcast-count quota + the
668    // periodic ops line, for the life of this session. Aborted the moment the
669    // connection closes so it never outlives what it's watching.
670    let watchdog_conn = conn.clone();
671    let watchdog = tokio::spawn(ingress_watchdog(
672        watchdog_conn,
673        ingest_announces,
674        claims.user_id,
675        claims.space_id,
676        ingress,
677    ));
678
679    // The session drives forwarding on background tasks (moq-net spawns them in
680    // `lite::start`). Hold it alive until the connection closes, then drop it so
681    // its broadcasts unannounce and subscribers see end-of-track.
682    let _session = session;
683    conn.closed().await;
684    watchdog.abort();
685    tracing::info!(%peer, %claims.user_id, "relay connection closed");
686    Ok(())
687}

689/// Per-connection ops surface + policy enforcement, run for the life of one
690/// authenticated relay session (spawned in [`handle_connection`], aborted the
691/// instant the connection closes). Three concerns share one loop so there is
692/// exactly one task watching a connection's health:
693///
694/// * **Ingress budget** — samples `conn.stats().udp_rx.bytes` (cumulative
695///   QUIC bytes received) every [`IngressLimits::sample_interval`] and feeds
696///   an [`IngressBudget`] to catch a sustained flood
697///   ([`IngressLimits::max_bytes_per_sec`]).
698/// * **Broadcast quota** — counts this connection's own currently-announced
699///   broadcast paths from `ingest_announces`' event stream, bounding runaway
700///   `space/{s}/peer/{user_id}` fan-out
701///   ([`IngressLimits::max_broadcasts_per_session`]).
702///
703///   Why an announce-event count and not moq-net's `Stats` aggregator (the
704///   module built for exactly this kind of counting): `Stats`'s per-broadcast
705///   counters are private and only reach the outside world by publishing a
706///   JSON snapshot onto a broadcast that a consumer must then subscribe to
707///   and parse — there is no synchronous in-process accessor. Standing up a
708///   scratch origin plus a self-subscribing background task just to read back
709///   two integers is exactly the kind of indirection not worth adding for a
710///   number this cheap to derive directly. `ingest_announces` is a consumer
711///   over the SAME shared origin subtree `scoped_publish` (this connection's
712///   ingest half) publishes into, via the ordinary [`OriginConsumer`] announce
713///   API every other subscriber in this codebase already uses — so counting
714///   its live announce/unannounce events is both cheaper and simpler.
715///
716///   The tradeoff: that subtree is keyed by the authenticated `(space, user)`
717///   pair, not by QUIC connection identity, so if the SAME user holds more
718///   than one concurrent session in the SAME space (allowed up to
719///   [`SessionLimits::max_per_user`]), every one of those connections'
720///   watchdogs sees the group's COMBINED broadcast count, not just its own.
721///   In the common case (one session per user per space) the two coincide
722///   exactly. When they don't, the quota still bounds something real — one
723///   identity's total ingest fan-out into a space — just not quite what
724///   "per-session" literally promises.
725/// * **Ops log** — every [`OPS_LOG_INTERVAL`], one `info!` line summarising
726///   both, so "which connection is the problem" is answerable by grepping one
727///   connection's lines rather than reasoning about aggregate metrics.
728///
729/// Either policy trip closes the connection with a distinct close code
730/// ([`CLOSE_INGRESS_BUDGET_EXCEEDED`] / [`CLOSE_BROADCAST_QUOTA_EXCEEDED`])
731/// rather than the generic `0` other close paths use, so a client inspecting
732/// the code/reason on `conn.closed()` can tell a policy close from a network
733/// failure.
734async fn ingress_watchdog(
735    conn: iroh::endpoint::Connection,
736    mut ingest_announces: OriginConsumer,
737    user_id: Uuid,
738    space_id: Uuid,
739    limits: IngressLimits,
740) {
741    let mut budget = IngressBudget::new(limits.max_bytes_per_sec, limits.window);
742    let mut live_broadcasts: HashSet<PathOwned> = HashSet::new();
743    // Once the announce cursor closes (the session ended) stop polling it —
744    // a closed `OriginConsumer` future resolves `None` immediately, and
745    // without this guard that would spin the select loop.
746    let mut announces_open = true;
747
748    let mut sample_ticker = tokio::time::interval(limits.sample_interval);
749    sample_ticker.set_missed_tick_behavior(tokio::time::MissedTickBehavior::Delay);
750    let mut log_ticker = tokio::time::interval(OPS_LOG_INTERVAL);
751    log_ticker.set_missed_tick_behavior(tokio::time::MissedTickBehavior::Delay);
752
753    loop {
754        tokio::select! {
755            _ = sample_ticker.tick() => {
756                let bytes = conn.stats().udp_rx.bytes;
757                if let Some(rate) = budget.sample(bytes, Instant::now()) {
758                    tracing::warn!(
759                        %user_id, %space_id, bytes_per_sec = rate, cap = limits.max_bytes_per_sec,
760                        "relay connection exceeded its ingress budget — closing"
761                    );
762                    conn.close(
763                        CLOSE_INGRESS_BUDGET_EXCEEDED.into(),
764                        b"ingress budget exceeded",
765                    );
766                    return;
767                }
768            }
769            update = ingest_announces.announced(), if announces_open => {
770                match update {
771                    Some((path, Some(_))) => {
772                        live_broadcasts.insert(path);
773                    }
774                    Some((path, None)) => {
775                        live_broadcasts.remove(&path);
776                    }
777                    None => announces_open = false,
778                }
779                if limits.max_broadcasts_per_session > 0
780                    && live_broadcasts.len() > limits.max_broadcasts_per_session
781                {
782                    tracing::warn!(
783                        %user_id, %space_id, broadcasts = live_broadcasts.len(),
784                        cap = limits.max_broadcasts_per_session,
785                        "relay connection exceeded its broadcast quota — closing"
786                    );
787                    conn.close(
788                        CLOSE_BROADCAST_QUOTA_EXCEEDED.into(),
789                        b"broadcast quota exceeded",
790                    );
791                    return;
792                }
793            }
794            _ = log_ticker.tick() => {
795                tracing::info!(
796                    %user_id, %space_id,
797                    bytes_per_sec = ?budget.observed_rate(),
798                    broadcasts = live_broadcasts.len(),
799                    "relay connection ingress"
800                );
801            }
802        }
803    }
804}

806/// Read and verify the relay token off the dedicated control bi-stream.
807///
808/// Wire format (client → relay): `u16 LE length` + `token bytes`. The token
809/// scheme is the backend's HMAC `{user_id}:{space_id}:{expiry}:{mac}`, verified
810/// by the shared [`crate::auth::verify_relay_token`].
811///
812/// On a format/verification failure the one-byte [`AUTH_REJECT`] reply is
813/// written and the stream finished here. On success this returns the claims AND
814/// the still-open send half WITHOUT writing a verdict — the caller
815/// ([`handle_connection`]) still has to run the post-auth session-cap check
816/// before it knows whether the final byte is [`AUTH_OK`] or [`AUTH_REJECT`];
817/// writing here would foreclose that choice.
818async fn authenticate(
819    conn: &iroh::endpoint::Connection,
820    secret: &str,
821) -> Result<(RelayTokenClaims, iroh::endpoint::SendStream), RelayError> {
822    let (mut send, mut recv) = conn
823        .accept_bi()
824        .await
825        .map_err(|e| RelayError::Iroh(format!("accept token stream: {e}")))?;
826
827    let mut len_buf = [0u8; 2];
828    recv.read_exact(&mut len_buf)
829        .await
830        .map_err(|e| RelayError::Iroh(format!("read token len: {e}")))?;
831    let len = u16::from_le_bytes(len_buf) as usize;
832    if len == 0 || len > MAX_TOKEN_LEN {
833        let _ = reject(&mut send).await;
834        return Err(RelayError::Auth(format!(
835            "token length {len} out of bounds (1..={MAX_TOKEN_LEN})"
836        )));
837    }
838
839    let mut token_buf = vec![0u8; len];
840    recv.read_exact(&mut token_buf)
841        .await
842        .map_err(|e| RelayError::Iroh(format!("read token: {e}")))?;
843    let token = String::from_utf8(token_buf)
844        .map_err(|_| RelayError::Auth("token is not valid UTF-8".into()))?;
845
846    let now = chrono::Utc::now().timestamp();
847    match verify_relay_token(&token, secret, now) {
848        Ok(claims) => Ok((claims, send)),
849        Err(e) => {
850            let _ = reject(&mut send).await;
851            Err(RelayError::Auth(e.to_string()))
852        }
853    }
854}
855
856/// Send the one-byte rejection and finish the control stream. Best-effort: the
857/// caller has already decided to reject, so a write error here is not fatal.
858async fn reject(send: &mut iroh::endpoint::SendStream) -> Result<(), RelayError> {
859    send.write_all(&[AUTH_REJECT])
860        .await
861        .map_err(|e| RelayError::Iroh(format!("write auth reject: {e}")))?;
862    let _ = send.finish();
863    Ok(())
864}

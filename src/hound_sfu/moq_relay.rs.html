1//! # MoQ relay — the native forward plane
2//!
3//! The MoQ forward plane: a thin, forward-only relay that fans every publisher's
4//! encoded media out to the other subscribers in the same space. It lives
5//! **alongside** the browser-path handlers in [`crate::sfu`] — it does not touch
6//! them, share their ALPN, or share their room state. A client picks the path by the
7//! ALPN it dials:
8//!
9//! * [`crate::sfu::HOUND_ALPN`] (`hound-sfu/1`) → the frozen datagram/uni-stream
10//!   browser path.
11//! * [`RELAY_ALPN_PREFIX`]-prefixed MoQ ALPN (`moql`) → this relay.
12//!
13//! ## Forward-only, no transcode
14//!
15//! The relay never decodes media. It re-emits the encoded objects a publisher
16//! writes to every subscriber of that track, byte-for-byte. moq-net owns the
17//! actual fan-out and per-subscriber prioritisation: each GOP is one QUIC
18//! uni-stream and the consumer's `next_group` priority-drops a stale group at
19//! its boundary, so a receiver never freezes replaying stale deltas while a
20//! fresh keyframe group waits. We do not hand-roll routing tables, NACK caches,
21//! or SSRC/seq rewrites — that is exactly what MoQ's group/object model gives
22//! us for free.
23//!
24//! ## How moq-net does the forwarding (the Origin)
25//!
26//! moq-net routes through a shared [`moq_net::Origin`]: a tree of broadcast
27//! paths. A [`moq_net::Server`] session is wired with two halves of that origin:
28//!
29//! * `with_consume(OriginProducer)` — "consume any **remote** broadcasts,
30//!   inserting them into this origin" (the publisher half: a connected publisher
31//!   announces a broadcast and moq-net inserts it into our shared origin).
32//! * `with_publish(OriginConsumer)` — "publish any **local** broadcasts from
33//!   this origin to the remote" (the subscriber half: a connected subscriber's
34//!   SUBSCRIBE is served from whatever the shared origin holds).
35//!
36//! Give every connection both halves of **one shared root origin** and moq-net
37//! itself forwards: publisher A announces `space/X/peer/A/video` into the origin
38//! → subscriber B's session sees the announce and serves the objects. The relay
39//! code never copies a byte; it only owns the origin and the auth gate.
40//!
41//! ## Auth gate (the mandatory bit)
42//!
43//! moq-net itself does not authenticate. Before any media flows we:
44//!
45//! 1. **Validate the relay-token JWT-equivalent** the client presents on a
46//!    dedicated control bi-stream, reusing [`crate::auth::verify_relay_token`]
47//!    (the same HMAC-SHA256 `{user}:{space}:{expiry}:{mac}` scheme the backend
48//!    mints and the frozen path verifies). A bad/expired/missing token is
49//!    rejected before the MoQ handshake runs.
50//! 2. **Scope the connection to its space.** From the validated
51//!    [`crate::auth::RelayTokenClaims`] we derive the namespace prefix
52//!    `space/{space_id}` and hand the moq-net session an origin **scoped** to it
53//!    via [`moq_net::OriginProducer::scope`] / [`moq_net::OriginConsumer::scope`].
54//!    A scoped producer's `publish_broadcast` returns `false` for any path
55//!    outside the prefix, and a scoped consumer never sees announcements outside
56//!    it — so a participant authenticated for space A **cannot** announce or
57//!    subscribe to space B's namespace. moq-net enforces the boundary; we just
58//!    set it from the token.
59//!
60//! This is **call-membership** auth. Per-track allow-listing (which *peer* may
61//! publish which *track* within a space, driven by `hound-control`'s
62//! `SessionServer` roster) is a later refinement — see the
63//! [`NAMESPACE_CONVENTION`] seam.
64

71/// MoQ ALPN the relay accepts. Must be a moq-net ALPN: in raw mode
72/// `web-transport-iroh` reports the iroh connection's ALPN as the MoQ protocol
73/// and moq-net rejects an unrecognised one with `Error::UnknownAlpn`. `moql` is
74/// moq-net's version-negotiation ALPN and matches the client
75/// (`hound_transport::ALPN`). Kept separate from [`crate::sfu::HOUND_ALPN`] so
76/// the relay and the browser path never collide on one connection.
77pub const RELAY_ALPN: &[u8] = b"moql";
78
79/// Marker documenting that [`RELAY_ALPN`] is the moq-net negotiation ALPN, not a
80/// free-to-choose `hound-*` value (the latter fails the handshake).
81pub const RELAY_ALPN_PREFIX: &[u8] = b"moq";
82
83/// The broadcast-namespace convention every announce/subscribe must obey.
84///
85/// ```text
86/// space/{space_id}/peer/{peer_id}/{track}
87/// ```
88///
89/// The relay scopes each authenticated connection to the `space/{space_id}`
90/// prefix taken from its relay token. moq-net then refuses any publish or
91/// subscribe outside that prefix. The `peer/{peer_id}/{track}` tail is **not**
92/// enforced yet: per-track / per-peer allow-listing (e.g. preventing peer A from
93/// impersonating peer B's namespace, or gating screenshare tracks) is a future
94/// `hound-control` `SessionServer`-driven refinement. The
95/// seam to add it is [`space_scope`] → tighten the scope prefix to
96/// `space/{space_id}/peer/{peer_id}` for a publisher once the roster is wired.
97pub const NAMESPACE_CONVENTION: &str = "space/{space_id}/peer/{peer_id}/{track}";

99/// Maximum relay-token length accepted on the control stream. A valid token is
100/// `36 + 1 + 36 + 1 + ~10 + 1 + 64 ≈ 150` bytes; this bound stops a malicious
101/// client pinning memory with a giant "token".
102const MAX_TOKEN_LEN: usize = 512;
103
104/// One-byte control-stream replies the relay sends after token verification.
105/// The client gates its MoQ handshake on reading [`AUTH_OK`].
106const AUTH_OK: u8 = 1;
107const AUTH_REJECT: u8 = 0;
108
109/// How long the relay waits, after sending an [`AUTH_REJECT`], for the client to
110/// read the verdict and close the connection before the handler gives up. A
111/// safety valve so a vanished client can't pin the task; we only care that the
112/// reject byte had a chance to land.
113const AUTH_REJECT_DRAIN: std::time::Duration = std::time::Duration::from_secs(5);
114
115/// Errors from the relay's connection-acceptance / auth path. moq-net session
116/// errors after a successful handshake are logged inside the spawned session
117/// task, not surfaced here.
118#[derive(Debug, thiserror::Error)]
119pub enum RelayError {
120    /// An iroh stream/connection operation failed during the handshake.
121    #[error("iroh: {0}")]
122    Iroh(String),
123    /// The relay token was missing, malformed, or failed verification.
124    #[error("auth: {0}")]
125    Auth(String),
126    /// The moq-net server handshake failed.
127    #[error("moq: {0}")]
128    Moq(String),
129}

131/// Create a fresh, wildcard-scoped root [`OriginProducer`] — the shared origin
132/// through which moq-net forwards every broadcast.
133///
134/// Every accepted relay connection gets a *space-scoped view* of this same
135/// origin (see [`handle_relay_connection`]), so a publisher and a subscriber in
136/// the same space meet here and moq-net fans the objects out. [`Relay`] builds
137/// one internally; the dual-ALPN SFU loop ([`crate::sfu::Sfu`]) builds one too so
138/// it can serve the relay on its own endpoint without constructing a [`Relay`].
139pub(crate) fn new_root_origin() -> OriginProducer {
140    Origin::random().produce()
141}
142
143/// Build the `space/{space_id}` namespace prefix a connection is scoped to.
144///
145/// This is the call-membership boundary: everything a participant may announce
146/// or subscribe lives under this prefix, and moq-net rejects anything outside
147/// it. Per-peer tightening (the `hound-control` refinement) would append
148/// `/peer/{peer_id}` here.
149fn space_scope(claims: &RelayTokenClaims) -> String {
150    format!("space/{}", claims.space_id)
151}

153/// The MoQ relay server.
154///
155/// Owns the iroh [`Endpoint`] (a separate endpoint / ALPN from the frozen
156/// browser-path SFU), the shared [`OriginProducer`] root through which moq-net
157/// forwards every broadcast, and the relay-token HMAC secret. Call [`Relay::run`]
158/// to accept connections.
159pub struct Relay {
160    endpoint: Endpoint,
161    /// The shared root origin (wildcard scope). Every accepted connection gets a
162    /// space-scoped view of this same origin, so a publisher in space X and a
163    /// subscriber in space X meet here and moq-net fans the objects out.
164    origin: OriginProducer,
165    relay_hmac_secret: String,
166}
167
168impl Relay {
169    /// Build a relay around an already-bound iroh [`Endpoint`].
170    ///
171    /// The caller is responsible for having advertised [`RELAY_ALPN`] on the
172    /// endpoint. Mirrors [`crate::sfu::Sfu::from_endpoint`] so the hermetic
173    /// tests exercise the same accept/auth glue that runs in production.
174    pub fn from_endpoint(endpoint: Endpoint, relay_hmac_secret: String) -> Self {
175        Self {
176            endpoint,
177            origin: new_root_origin(),
178            relay_hmac_secret,
179        }
180    }
181
182    /// The iroh node id clients dial to reach this relay.
183    pub fn node_id(&self) -> iroh::EndpointId {
184        self.endpoint.id()
185    }
186
187    /// Accept connections in a loop until the endpoint closes.
188    ///
189    /// Each connection is handed to [`handle_connection`] in its own task. The
190    /// shared origin and secret are cloned per connection (both cheap: the
191    /// origin is `Arc`-backed internally, the secret is a short `String`).
192    pub async fn run(&self) -> eyre::Result<()> {
193        tracing::info!(node_id = %self.endpoint.id(), "MoQ relay accepting connections");
194
195        loop {
196            let Some(incoming) = self.endpoint.accept().await else {
197                tracing::info!("MoQ relay endpoint closed — stopping accept loop");
198                break;
199            };
200
201            let conn = match incoming.await {
202                Ok(c) => c,
203                Err(e) => {
204                    tracing::warn!("incoming relay connection failed: {e}");
205                    continue;
206                }
207            };
208
209            let origin = self.origin.clone();
210            let secret = self.relay_hmac_secret.clone();
211            tokio::spawn(async move {
212                if let Err(e) = handle_connection(conn, origin, secret).await {
213                    tracing::warn!("relay connection handler error: {e}");
214                }
215            });
216        }
217
218        Ok(())
219    }
220}

222/// Handle one incoming MoQ relay connection on a **shared** origin.
223///
224/// This is the per-connection relay logic factored out of [`Relay::run`] so the
225/// SFU service's dual-ALPN accept loop ([`crate::sfu::Sfu::run`]) can drive the
226/// relay on the *same* iroh endpoint as the frozen browser path: a `moql`
227/// connection is dispatched here with the SFU's shared origin and relay secret.
228/// [`Relay::run`] (used by the standalone relay tests) and the SFU loop both call
229/// it, so they exercise identical auth/scoping/session glue.
230pub(crate) async fn handle_relay_connection(
231    conn: iroh::endpoint::Connection,
232    origin: OriginProducer,
233    relay_hmac_secret: String,
234) -> Result<(), RelayError> {
235    handle_connection(conn, origin, relay_hmac_secret).await
236}

238/// Handle one incoming MoQ relay connection: authenticate, scope, run the
239/// moq-net server session.
240///
241/// ## Stream ordering (why the token stream is accepted first)
242///
243/// The client opens the **relay-token control bi-stream first** and awaits the
244/// relay's [`AUTH_OK`] before starting its MoQ handshake. The relay accepts that
245/// bi-stream (`conn.accept_bi`) before handing the connection to
246/// [`moq_net::Server::accept`] (which opens *its own* control stream). QUIC
247/// bi-streams are accepted in open order, so the token exchange always lands
248/// before the MoQ control stream. If auth fails the connection is closed and the
249/// MoQ handshake never runs.
250async fn handle_connection(
251    conn: iroh::endpoint::Connection,
252    origin: OriginProducer,
253    relay_hmac_secret: String,
254) -> Result<(), RelayError> {
255    let peer = conn.remote_id();
256    tracing::info!(%peer, "relay connection accepted");
257
258    // ── Relay-token auth (control bi-stream) ───────────────────────────────
259    let claims = match authenticate(&conn, &relay_hmac_secret).await {
260        Ok(claims) => claims,
261        Err(e) => {
262            tracing::warn!(%peer, "relay auth rejected: {e}");
263            // Don't slam `conn.close` here: a `CONNECTION_CLOSE` frame can
264            // overtake the buffered reject byte, so the client would observe a
265            // generic connection error instead of the explicit reject verdict.
266            // Wait (bounded) for the client to read the verdict and close the
267            // connection itself — the same delivery discipline the frozen
268            // browser path uses for its NACK (see `crate::sfu::reject`).
269            let _ = tokio::time::timeout(AUTH_REJECT_DRAIN, conn.closed()).await;
270            return Err(e);
271        }
272    };
273
274    // ── Namespace scoping (the call-membership boundary) ───────────────────
275    // Derive a space-scoped view of the shared origin. moq-net enforces the
276    // prefix: a publisher scoped here cannot announce outside `space/{id}` and a
277    // subscriber scoped here never sees another space's broadcasts.
278    let scope = space_scope(&claims);
279    let scope_path = Path::new(&scope);
280    let scoped_publish = origin
281        .scope(std::slice::from_ref(&scope_path))
282        .ok_or_else(|| RelayError::Moq(format!("origin scope rejected namespace {scope:?}")))?;
283    // `with_consume` ingests remote announces INTO this scoped origin; the
284    // matching consumer (`with_publish`) serves them back OUT to subscribers.
285    // Both are the same scoped origin, so publishers and subscribers in this
286    // space meet and moq-net forwards between them.
287    let scoped_consume = scoped_publish.consume();
288
289    tracing::info!(%peer, %claims.user_id, %claims.space_id, namespace = %scope, "relay connection authorised");
290
291    // ── MoQ server handshake + session ─────────────────────────────────────
292    // `web-transport-iroh` raw mode: speak MoQ directly over iroh, no HTTP/3.
293    let wt = web_transport_iroh::Session::raw(conn.clone());
294    let session = moq_net::Server::new()
295        .with_consume(scoped_publish)
296        .with_publish(scoped_consume)
297        .accept(wt)
298        .await
299        .map_err(|e| RelayError::Moq(format!("server accept: {e}")))?;
300
301    // The session drives forwarding on background tasks (moq-net spawns them in
302    // `lite::start`). Hold it alive until the connection closes, then drop it so
303    // its broadcasts unannounce and subscribers see end-of-track.
304    let _session = session;
305    conn.closed().await;
306    tracing::info!(%peer, %claims.user_id, "relay connection closed");
307    Ok(())
308}

310/// Read and verify the relay token off the dedicated control bi-stream.
311///
312/// Wire format (client → relay): `u16 LE length` + `token bytes`. Reply
313/// (relay → client): one byte, [`AUTH_OK`] or [`AUTH_REJECT`]. The token scheme
314/// is the backend's HMAC `{user_id}:{space_id}:{expiry}:{mac}`, verified by the
315/// shared [`crate::auth::verify_relay_token`].
316async fn authenticate(
317    conn: &iroh::endpoint::Connection,
318    secret: &str,
319) -> Result<RelayTokenClaims, RelayError> {
320    let (mut send, mut recv) = conn
321        .accept_bi()
322        .await
323        .map_err(|e| RelayError::Iroh(format!("accept token stream: {e}")))?;
324
325    let mut len_buf = [0u8; 2];
326    recv.read_exact(&mut len_buf)
327        .await
328        .map_err(|e| RelayError::Iroh(format!("read token len: {e}")))?;
329    let len = u16::from_le_bytes(len_buf) as usize;
330    if len == 0 || len > MAX_TOKEN_LEN {
331        let _ = reject(&mut send).await;
332        return Err(RelayError::Auth(format!(
333            "token length {len} out of bounds (1..={MAX_TOKEN_LEN})"
334        )));
335    }
336
337    let mut token_buf = vec![0u8; len];
338    recv.read_exact(&mut token_buf)
339        .await
340        .map_err(|e| RelayError::Iroh(format!("read token: {e}")))?;
341    let token = String::from_utf8(token_buf)
342        .map_err(|_| RelayError::Auth("token is not valid UTF-8".into()))?;
343
344    let now = chrono::Utc::now().timestamp();
345    match verify_relay_token(&token, secret, now) {
346        Ok(claims) => {
347            send.write_all(&[AUTH_OK])
348                .await
349                .map_err(|e| RelayError::Iroh(format!("write auth ok: {e}")))?;
350            let _ = send.finish();
351            Ok(claims)
352        }
353        Err(e) => {
354            let _ = reject(&mut send).await;
355            Err(RelayError::Auth(e.to_string()))
356        }
357    }
358}

360/// Send the one-byte rejection and finish the control stream. Best-effort: the
361/// caller has already decided to reject, so a write error here is not fatal.
362async fn reject(send: &mut iroh::endpoint::SendStream) -> Result<(), RelayError> {
363    send.write_all(&[AUTH_REJECT])
364        .await
365        .map_err(|e| RelayError::Iroh(format!("write auth reject: {e}")))?;
366    let _ = send.finish();
367    Ok(())

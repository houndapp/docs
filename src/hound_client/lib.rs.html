1//! # hound-client — the unified, headless client core
2//!
3//! The single source of truth for every hound client. It owns auth, the REST data
4//! plane, the realtime (`/community`) plane, and the media/call engine; reduces
5//! them into the typed view-models in [`model`]; and exposes commands +
6//! snapshot streams. UI layers (via `hound-frb`) are thin views over this — no
7//! client logic lives in Dart.
8//!
9//! Layout:
10//! - [`app`] — the [`HoundClient`] orchestration facade: observable watch channels +
11//!   the realtime→reducer task.
12//! - [`error`] — the typed [`HoundError`] surface.
13//! - [`model`] — typed view-models the UI renders.
14//! - [`rest`] — the REST data plane.
15//! - [`realtime`] — the socket.io `/community` plane.
16//! - [`session`] — auth/session lifecycle: [`SessionStore`], token refresh, [`AuthState`].
17//! - [`reducer`] — pure folds of REST loads + realtime events into view-models.
18//! - [`call`] — the voice/video call orchestration engine: join →
19//!   control WS → MoQ → publish/subscribe → decode, reusing the engine/transport/
20//!   control/media crates. UI-free: the decode→render seam is a [`call::VideoSink`]
21//!   the FRB layer supplies.

41// them by a `hound_client::{MediaKind, Source, Codec}` path (a `CallTrack` carries

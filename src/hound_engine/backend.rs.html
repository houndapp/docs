1//! # The platform SPI (sealed)
2//!
3//! The traits a platform backend implements to plug its devices and codecs
4//! into the engine. **Sealed**: the private supertrait below can only be
5//! implemented inside this crate, so the set of backends is closed at the
6//! crate boundary — the app can call through these traits (via the [`crate::api`]
7//! facade) but can never implement them or smuggle a foreign "device" into
8//! the engine. Adding Windows/Linux parity means adding an in-crate
9//! `platform::windows` / `platform::linux` module that implements these
10//! (see the crate-root "adding a backend" notes).
11//!
12//! ## Lifecycle: open is construct, drop is stop
13//!
14//! There is deliberately **no `start()`** on these traits. A backend value
15//! that exists is running: construction performs the open/negotiate (and is
16//! where permission/absence errors surface, typed), and `Drop` releases the
17//! device/session on the owning thread. That makes a "called
18//! `next_frame` before `start`" runtime error unrepresentable.
19//!
20//! ## Threading contract (`!Send` by design)
21//!
22//! Most backend values are `!Send` (CoreAudio streams, AVFoundation session
23//! graphs, VideoToolbox sessions are tied to their setup context). The
24//! contract everywhere is: **construct, drive, and drop on one dedicated OS
25//! thread** — never an async executor, never a UI thread, never moved.
26

33/// A running screen/window source producing timestamped raw frames, optionally
34/// paired with an [`AppAudio`] stream of the source application's audio.
35///
36/// The video half mirrors [`VideoCapture`] exactly (construct-is-start, blocking
37/// [`Self::next_frame`] delivering tightly-packed I420 [`RawFrame`]s stamped on
38/// the capture clock, RAII stop on `Drop`), so the encode/publish path reuses
39/// the camera plumbing verbatim. The optional audio half is split out as a
40/// separate [`AppAudio`] handle the open hands back, so a consumer that only
41/// wants video never carries an audio leg.
42///
43/// ## Implementor contract
44/// * Construction opened the capture session and began delivery, reporting the
45///   geometry the consumer receives via [`Self::negotiated`] (the encode target;
46///   native frames are scaled down preserving aspect, like the camera path).
47/// * [`Self::next_frame`] blocks until the next frame or returns `Err` when the
48///   stream ended — which includes the source vanishing (window closed) or the
49///   user stopping the share via the OS overlay. Those are a typed **capture
50///   end**, never a panic.
51/// * The handle is `!Send` (the platform capture session is tied to its setup
52///   context): construct, drive, and drop on one dedicated capture thread.
53pub trait ScreenCapture: sealed::Sealed {
54    /// Block until the next frame is available, or `Err` at stream end.
55    fn next_frame(&mut self) -> Result<RawFrame, CaptureError>;
56
57    /// Bounded-wait variant of [`Self::next_frame`]: `Ok(None)` when `timeout`
58    /// elapses with no frame. Screen capture is damage-driven (a static source
59    /// legitimately produces nothing, unlike a camera), so consumers that need
60    /// liveness — preview tees, keyframe synthesis for late subscribers — poll
61    /// with this instead of blocking indefinitely.
62    fn next_frame_timeout(
63        &mut self,
64        timeout: std::time::Duration,
65    ) -> Result<Option<RawFrame>, CaptureError>;
66
67    /// Whether the captured SOURCE still exists (the shared window / display is
68    /// still in the shareable inventory). Damage-driven capture cannot distinguish
69    /// "static content" from "source gone" by frame flow alone, so an idle
70    /// consumer probes this (throttled) and ENDS the share when it reports false —
71    /// otherwise the last frame is re-emitted forever. Default `true` for backends
72    /// without an inventory query (their stop/error path covers source loss).
73    fn source_alive(&self) -> bool {
74        true
75    }
76
77    /// The source's current content size (points) as observed on delivered
78    /// frames, when the platform reports it. A running capture never resizes
79    /// itself, so a change means the shared window was resized — the session's
80    /// cue to restart the share at the new geometry. `None` = no frame seen
81    /// yet, or no such signal on this platform.
82    fn content_size(&self) -> Option<(u32, u32)> {
83        None
84    }
85
86    /// The geometry + pixel format delivered frames carry (the encode target,
87    /// post the native→target scale).
88    fn negotiated(&self) -> (Dimensions, PixelFormat);
89}

91/// A running per-application audio source delivering fixed-size 48 kHz mono PCM
92/// frames — the audio leg of a [`ScreenCapture`] window/app share.
93///
94/// Deliberately the **same contract** as [`AudioInput`] (the mic path): 20 ms
95/// [`crate::types::SAMPLES_PER_FRAME`]-sample 48 kHz mono frames stamped on the
96/// capture clock, so the Opus encoder and publish path reuse the mic plumbing
97/// without a special case. Whatever the platform delivers (SCK hands back
98/// deinterleaved float32 stereo at the configured rate), the backend downmixes
99/// to mono and resamples to 48 kHz off the capture callback before delivery.
100///
101/// ## Implementor contract
102/// * [`Self::next_frame`] blocks until the next frame, or returns `Err` when the
103///   underlying screen capture ends (the audio leg dies with its video parent).
104/// * `!Send`, same threading contract as the parent [`ScreenCapture`].
105pub trait AppAudio: sealed::Sealed {
106    /// Block until the next fixed-size 48 kHz mono PCM frame is available.
107    fn next_frame(&mut self) -> Result<PcmFrame, AudioError>;
108}

116/// A running microphone source delivering fixed-size 48 kHz mono PCM frames.
117///
118/// ## Implementor contract
119/// * Construction opened the device and negotiated as close to 48 kHz mono
120///   i16 as it allows; it MUST fail loudly (typed [`AudioError`]) on
121///   no-device / permission-denied — never succeed-then-deliver-silence.
122/// * [`Self::next_frame`] blocks until the next
123///   [`crate::types::SAMPLES_PER_FRAME`]-sample frame is assembled from the
124///   device callback, stamped on the capture clock supplied at open. Returns
125///   `Err` when the stream ends or the device is lost.
126/// * Whatever the device's native rate/channels, delivered frames are always
127///   48 kHz mono (the backend downmixes/resamples off the RT callback).
128pub trait AudioInput: sealed::Sealed {
129    /// Block until the next fixed-size PCM frame is available.
130    fn next_frame(&mut self) -> Result<PcmFrame, AudioError>;
131
132    /// The (rate, channels) the *device* actually negotiated — the device
133    /// truth for diagnostics. Delivery is always 48 kHz mono regardless.
134    fn negotiated(&self) -> (SampleRate, ChannelCount);
135}

137/// A running speaker sink that plays 48 kHz mono PCM frames.
138///
139/// ## Implementor contract
140/// * [`Self::play`] enqueues a frame for playout into the bounded ring the
141///   real-time output callback drains. It MUST NOT block the caller on the
142///   device; on overflow it drops the oldest (never an unbounded queue).
143pub trait AudioOutput: sealed::Sealed {
144    /// Enqueue one PCM frame for playout (non-blocking; bounded ring).
145    fn play(&mut self, frame: &PcmFrame) -> Result<(), AudioError>;
146
147    /// Read and reset the count of output samples the device's RT callback had to
148    /// silence-pad because the ring ran dry since the last call — the definitive
149    /// end-of-pipeline AUDIBLE GAP (a choppy dropout). Steady state is 0; a rising
150    /// count is the playout stutter, made measurable. Default 0 for sinks that do
151    /// not (or cannot) track it.
152    fn take_output_underruns(&self) -> u64 {
153        0
154    }
155
156    /// Current buffered playout depth in milliseconds (the device ring's fill). The
157    /// mix bus targets a small cushion here so the ring never drains to zero (which
158    /// is what made it underrun on pacing jitter). Default 0 for sinks that don't
159    /// track it — the bus then falls back to one chunk per tick.
160    fn buffered_ms(&self) -> u32 {
161        0
162    }
163
164    /// Whether the underlying device stream has errored (e.g. the default output
165    /// device changed / disconnected) and the sink must be reopened. The cpal stream
166    /// is dead after such an error but this object lives on, so the mix bus checks
167    /// this each tick and reopens — otherwise output stays silent. Default false.
168    fn is_dead(&self) -> bool {
169        false
170    }
171
172    /// The opened device's native sample rate (Hz). The mix bus uses this only as a
173    /// heuristic: a rate other than the engine rate means the device was opened
174    /// resampled — in practice a Bluetooth headset in hands-free (HFP) mode, whose
175    /// SCO link is often not yet carrying audio when the stream first opens (silent
176    /// first connect that a reopen fixes). Default = engine rate, so sinks that don't
177    /// report it (and every test fake) opt out of the warmup reopen entirely.
178    fn device_rate(&self) -> u32 {
179        crate::types::SAMPLE_RATE.hz()
180    }
181}

183/// A running camera/screen source producing timestamped raw frames.
184///
185/// ## Implementor contract
186/// * Construction negotiated a format and began delivery, reporting the
187///   geometry the consumer will actually receive via [`Self::negotiated`]
188///   (cameras rarely honour the exact ask; backends scale to the requested
189///   encode target so the encoder geometry always matches).
190/// * [`Self::next_frame`] blocks until the next frame or returns `Err` when
191///   the stream ended / the device was lost. Frames are stamped on the
192///   capture clock supplied at open.
193pub trait VideoCapture: sealed::Sealed {
194    /// Block until the next frame is available.
195    fn next_frame(&mut self) -> Result<RawFrame, CaptureError>;
196
197    /// The geometry + pixel format delivered frames carry (the encode
198    /// target, post any internal scale).
199    fn negotiated(&self) -> (Dimensions, PixelFormat);
200}

202/// A video encoder: raw frames in, encoded chunks out, with keyframe and
203/// bitrate control. One per outbound video track.
204///
205/// ## Implementor contract
206/// * [`Self::encode`] may return **zero or more** chunks for one input frame:
207///   look-ahead encoders buffer internally, so early frames can yield nothing
208///   and a later frame can flush several. The caller must not assume 1:1.
209/// * [`Self::request_keyframe`] forces the *next* encoded frame to be a
210///   keyframe (IDR). Callers coalesce PLI storms before driving it.
211/// * [`Self::set_bitrate`] retargets rate control live; best-effort.
212/// * [`Self::flush`] drains any buffered frames at stream end.
213pub trait VideoEncode: sealed::Sealed {
214    /// The codec this encoder produces (for packet headers).
215    fn codec(&self) -> VideoCodec;
216
217    /// A short human label for the encode path actually in use, including whether
218    /// it is hardware- or software-backed (e.g. `"VideoToolbox (hardware)"`,
219    /// `"openh264 (software)"`). For diagnostics — surfaced in the call logs so a
220    /// software-encode fallback (CPU-heavy) is visible alongside the decode path.
221    fn backend(&self) -> &'static str;
222
223    /// Encode one raw frame. Returns any encoded chunks now ready.
224    fn encode(&mut self, frame: &RawFrame) -> Result<Vec<EncodedChunk>, CaptureError>;
225
226    /// Force the next encoded frame to be a keyframe (IDR/key).
227    fn request_keyframe(&mut self);
228
229    /// Retarget the rate controller. Best-effort.
230    fn set_bitrate(&mut self, bitrate: Bitrate) -> Result<(), CaptureError>;
231
232    /// Drain buffered frames (call once at stream end).
233    fn flush(&mut self) -> Result<Vec<EncodedChunk>, CaptureError>;
234}

236/// A video decoder: encoded chunks in, raw frames out. The receive-side
237/// complement to [`VideoEncode`]; one per inbound video track.
238///
239/// ## Implementor contract
240/// * [`Self::decode`] takes one encoded access unit and returns **zero or
241///   one** decoded [`RawFrame`]. Zero is normal and not an error: a decoder
242///   cannot emit anything until it has seen the in-band bootstrap (H.264
243///   SPS/PPS, AV1 sequence header), so pre-keyframe chunks are dropped.
244/// * Decoded frames are [`PixelFormat::I420`] (backends that decode to NV12
245///   natively convert before returning) at the decoded picture dimensions.
246pub trait VideoDecode: sealed::Sealed {
247    /// The codec this decoder consumes.
248    fn codec(&self) -> VideoCodec;
249
250    /// Decode one encoded access unit. `None` while pre-bootstrap/buffering.
251    fn decode(&mut self, chunk: &[u8]) -> Result<Option<RawFrame>, CaptureError>;
252
253    /// A short human label for the decode path actually in use, including whether
254    /// it is hardware- or software-backed (e.g. `"VideoToolbox (hardware)"`,
255    /// `"openh264 (software)"`). For diagnostics — surfaced in the call logs so a
256    /// CPU-burning software-decode fallback (e.g. AV1 with no HW block) is visible.
257    fn backend(&self) -> &'static str;
258}

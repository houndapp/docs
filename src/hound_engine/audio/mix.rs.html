1//! # Mixed-playout audio bus — one device stream + one coherent AEC reference
2//!
3//! All remote peers play through ONE mixed output stream. Echo cancellation
4//! requires it: [`Apm::process_render`] must be fed the one coherent stream the
5//! speaker actually plays — interleaving independent per-peer render references
6//! corrupts AEC3's canceller for any N>1. A single mixed playout makes the
7//! render reference exact for any N by construction, and halves the device
8//! handles (one stream per call, not one per peer).
9//!
10//! ## The tick model (PULL-based: the RT thread is the sole release clock)
11//!
12//! One dedicated playback thread per call owns the single [`CpalPlayback`]. It
13//! ticks on a **wall-clock 10 ms cadence** (the APM chunk size,
14//! [`MIX_CHUNK_SAMPLES`]) — driven by the OUTPUT side, never by peer frame
15//! arrival, because the cpal ring absorbs the jitter between the tick and the
16//! DAC. On each tick the bus, for EACH registered producer:
17//!
18//! 1. **pops every due frame from that peer's jitter buffer** on its OWN
19//!    real-time clock — `playout.pop_due(clock.now() + lookahead)` — appending the
20//!    released PCM to the slot's sample carry-queue, then drains up to
21//!    [`MIX_CHUNK_SAMPLES`] from the carry,
22//! 2. **sums** the producers sample-for-sample in **i32** then **saturating-clamps
23//!    to i16** (no wrap distortion on overload),
24//! 3. feeds the MIXED 10 ms chunk to [`Apm::process_render`] **once** (the single
25//!    coherent reference), and
26//! 4. pushes the mixed chunk to the [`CpalPlayback`] ring.
27//!
28//! ### Why pull, not push
29//!
30//! A push design — a per-peer pump task (a normal async task) calling `pop_due`
31//! and PUSHING samples into the slot for this RT thread to drain — puts a
32//! **second, non-real-time scheduler on the release path**: under video-encode
33//! load the pump gets descheduled, feeds its slot late, and the RT drain finds
34//! the slot short — an audible per-peer gap (`mix_starves`) with the device ring
35//! still full (`output_underruns == 0`). No slot lead or cushion can fix that,
36//! because the producer's *timing* is the variable. Pull — the RT thread is the
37//! ONLY thing that calls `pop_due`, on its own steady clock — removes that
38//! scheduler from the release path entirely. The pump's sole job is to FILL
39//! the jitter buffer (network-paced; its scheduling never affects playout),
40//! and the jitter buffer IS the cushion that absorbs the pump's arrival jitter.
41//! Both push (pump) and pop (this thread) read the same `Copy` [`CaptureClock`],
42//! so the buffer's sender→local timeline mapping is consistent across the two
43//! threads.
44//!
45//! When **zero** producers are registered the bus STILL ticks (pushing silence),
46//! so the device stream and the AEC reference stay warm for the whole call —
47//! AEC3's delay/echo state does not have to re-converge each time the room goes
48//! briefly quiet or the last peer's pipeline is mid-rebuild.
49//!
50//! ## Registration + bounds
51//!
52//! A per-peer pipeline, instead of owning a playback thread, registers a
53//! [`BusProducer`] and PUSHES its decoded, timestamped PCM into the slot's jitter
54//! buffer via [`BusProducer::push`] (which also folds the arrival into the slot's
55//! [`AdaptiveJitter`] and reports back the depth/jitter for stats). The RT thread
56//! pulls from that buffer. The jitter buffer is bounded (drop-oldest at its frame
57//! cap) and the sample carry-queue is bounded to ~100 ms ([`SLOT_CAPACITY_SAMPLES`])
58//! drop-oldest, so one stalled or runaway peer can never grow the bus's memory.
59//! Dropping a [`BusProducer`] (its pipeline torn down / aborted) unregisters its
60//! slot from the bus, so a left peer stops contributing to the mix with no explicit
61//! teardown call.
62//!
63//! ## Hot-swap / vanish / reopen
64//!
65//! The bus thread owns the speaker [`DeviceChoice`] hot-swap, the vanished-device
66//! fallback (handled inside [`CpalPlayback::start`]), and the reopen backoff —
67//! in ONE place targeting the ONE stream. `call_set_speaker` nudges the shared `watch` sender;
68//! the bus thread re-opens the single device in place without disturbing the mix.
69//!
70//! ## `!Send` discipline
71//!
72//! `cpal::Stream` is `!Send` on CoreAudio, so the [`CpalPlayback`] is constructed,
73//! driven, and dropped on the bus's own thread and never crosses a thread
74//! boundary. Producers (`Send`/`Sync`) only ever touch the bounded slot queues.
75//!
76//! ## Decoupling
77//!
78//! This module is deliberately free of session-specific types: it takes raw mono
79//! i16 PCM (`&[i16]`) in and plain handles out. The only audio dependencies are
80//! [`Apm`] (the render-reference sink) and [`CpalPlayback`] (the device), both
81//! behind narrow calls.

98/// Opens the output sink for a [`DeviceChoice`] on the bus thread, returning a
99/// boxed [`AudioOutput`] or `None` on failure. Production passes
100/// [`cpal_sink_factory`] ([`CpalPlayback`]); tests inject a fake so they never
101/// touch a real (possibly sandbox-blocked) audio device. Runs ONLY on the bus
102/// thread, so the `!Send` `cpal::Stream` never crosses a thread boundary.

105/// The production sink factory: open a [`CpalPlayback`] for the choice (with the
106/// vanish-fallback inside `open`), boxed as [`AudioOutput`]. `None` on an open
107/// failure (logged) — the bus keeps ticking the AEC reference and retries.
108fn cpal_sink_factory() -> SinkFactory {
109    Box::new(

114                    "voice: mix-bus playback opened"

116                Some(Box::new(playback) as Box<dyn AudioOutput>)
117            }
118            Err(e) => {
119                tracing::error!("voice: mix-bus playback start failed: {e}");
120                None
121            }
122        },
123    )
124}

149/// Samples in one mix tick: 10 ms @ 48 kHz mono = **480**. Identical to the APM's
150/// native chunk ([`APM_FRAME_SAMPLES`]) so the mixed chunk feeds `process_render`
151/// as exactly one block, and the tick cadence matches the canceller's frame size.
152pub const MIX_CHUNK_SAMPLES: usize = APM_FRAME_SAMPLES;
153
154/// The wall-clock period the bus thread ticks at: 10 ms, matching
155/// [`MIX_CHUNK_SAMPLES`] at 48 kHz. Output-side cadence — the cpal ring absorbs
156/// the difference between this and the DAC's real pull timing.
157const MIX_TICK: Duration = Duration::from_millis(10);
158
159/// [`MIX_TICK`] in milliseconds — the chunk duration the output cushion is counted in.
160const MIX_TICK_MS: u32 = 10;
161
162/// How far PAST strictly-due the RT thread pops each tick: exactly one [`MIX_TICK`]
163/// (10 ms). A 20 ms audio frame spans two 10 ms ticks, so its due-instant lands
164/// between ticks for some phase offsets; a one-tick lookahead releases the next
165/// frame into the carry-queue *before* the carry can bottom out, so the slot
166/// always has a full chunk to drain regardless of how the frame-due phase aligns
167/// to the tick grid. Because the RT thread (not the pump) owns this read, the
168/// lookahead is immune to producer scheduling — it just smooths the frame/tick
169/// phase. Net latency is unchanged: a frame released `lookahead` early simply
170/// waits that long in the carry-queue instead of the jitter buffer.
171const MIX_PULL_LOOKAHEAD_US: i64 = MIX_TICK_MS as i64 * 1_000;
172
173/// Target depth of the output device ring the bus keeps filled (~3 chunks). The
174/// device drains the ring on its hardware clock while this thread refills it on a
175/// software clock; without a cushion the ring sits at ~0 and underruns on the
176/// slightest pacing jitter (or when encode load deschedules this thread). 30 ms
177/// absorbs normal jitter at a small, constant added output latency (AEC3 tolerates
178/// the fixed delay); the thread-priority promotion handles longer stalls.
179const OUTPUT_CUSHION_MS: u32 = 30;
180
181/// Per-producer slot bound: ~100 ms of 48 kHz mono audio (4800 samples). A
182/// drop-oldest policy keeps a stalled/runaway peer from growing the bus's memory;
183/// 100 ms is ~5 mix chunks of slack, enough to ride out a scheduling hiccup
184/// without unbounded growth.
185const SLOT_CAPACITY_SAMPLES: usize = (SAMPLE_RATE.hz() as usize / 1000) * 100;
186
187/// Initial rate-limit on device-reopen attempts when no speaker is open (a down
188/// device must not hammer `start` every tick); first retry after ~500 ms.
189const REOPEN_BACKOFF: Duration = Duration::from_millis(500);
190
191/// Ceiling for the reopen backoff. A *persistently* unavailable output device
192/// (e.g. an ALSA `hw:` held exclusively by another process on Linux — where each
193/// failed open/close churn can itself prolong the `EBUSY`) must not retry at
194/// 2 Hz forever, flooding the log and thrashing the device. The interval doubles
195/// from [`REOPEN_BACKOFF`] up to this cap, then holds — a slow heartbeat that
196/// still recovers automatically once the device frees, without the storm.
197const REOPEN_BACKOFF_MAX: Duration = Duration::from_secs(5);

199/// A mix tick this late (µs) or more is recorded as a [`GlitchKind::LateTick`]. Half a
200/// tick (5 ms) — below this is ordinary scheduler noise the output cushion absorbs;
201/// above it the RT feeder genuinely slipped and the device ring risks draining.
202const TICK_LATE_GLITCH_US: u128 = (MIX_TICK_MS as u128 * 1000) / 2;

218/// The per-peer jitter state the PUMP mutates (push) and the RT thread reads (pop),
219/// behind one mutex held only for the bounded buffer op. Owning the
220/// [`AdaptiveJitter`] here keeps push (fold arrival → retarget depth) and pop (the
221/// release schedule) operating on one consistent buffer under one lock.
222struct PlayoutState {
223    buffer: PlayoutBuffer<MixFrame>,
224    adapt: AdaptiveJitter,
225}

227/// What a [`BusProducer::push`] reports back so the caller can surface per-peer
228/// audio-health stats without a second lock — everything the debug report needs to
229/// attribute a glitch to the jitter-buffer stage.
230pub struct PushStatus {
231    /// Current smoothed inter-arrival jitter estimate (µs, RFC 3550 `J`).
232    pub jitter_us: i64,
233    /// Frames currently queued in the jitter buffer (the playout depth gauge).
234    pub buffered_frames: u32,
235    /// Current jitter-buffer depth (µs) — what playout is delayed by.
236    pub target_delay_us: i64,
237    /// Depth the estimator recommends (`floor + 4·J`). `recommended ≫ target` =
238    /// buffer too shallow for the link (the grow step can't keep up) → underflows.
239    pub recommended_delay_us: i64,
240    /// Cumulative frames dropped as arrived-too-late (network late-arrival / loss).
241    pub late_drops: u64,
242    /// Cumulative frames dropped because the buffer was full (backed-up / clock skew).
243    pub overflow_drops: u64,
244    /// Cumulative mix-tick underflows for THIS slot (a primed slot that came up short
245    /// when the RT bus drained it — the per-peer audible gap). Read from the slot's
246    /// running starve count, so it reflects the latest value as of this push.
247    pub underflow_total: u64,
248}

255/// What an [`AudioGlitch`] is — the discrete audible-gap events the bus records, each
256/// already attributed to a stage so the debug report reads a cause, not a raw count.
257#[derive(Debug, Clone, Copy, PartialEq, Eq)]
258pub enum GlitchKind {
259    /// A primed slot came up short when the RT bus drained it: the per-peer jitter
260    /// buffer had nothing due (network starvation / buffer too shallow).
261    Underflow,
262    /// The 10 ms mix tick fired late (the RT feeder thread was descheduled) — a
263    /// device-side gap affecting ALL audio. `detail_us` is how late.
264    LateTick,
265}

279    pub kind: GlitchKind,

290/// What [`Slot::pull_into`] reports back to the bus loop for one tick.
291struct PullOutcome {
292    /// Samples this slot contributed to the mix this tick. (Read by tests asserting
293    /// the drain contract; the production loop only acts on `starved`.)
294    #[allow(dead_code)]
295    contributed: usize,
296    /// Whether a primed slot came up short (an underflow this tick).
297    starved: bool,

300/// One producer's per-peer slot. The PUMP pushes decoded, timestamped PCM into
301/// `state.buffer` (the jitter buffer); the BUS THREAD pops the due frames on its
302/// own real-time clock into `carry` and drains [`MIX_CHUNK_SAMPLES`] per tick. The
303/// two mutexes are each held only for a bounded op, never across IO; the producer
304/// is the async per-peer pump, the consumer is the bus's own thread (neither is a
305/// cpal RT callback).
306struct Slot {
307    /// Jitter buffer + adaptive depth. Pump writes (push), bus reads (pop_due).
308    state: Mutex<PlayoutState>,
309    /// Sample carry-queue between the 20 ms popped frames and the 10 ms drained
310    /// chunks. Touched ONLY by the bus thread, so it is uncontended — the mutex is
311    /// for the `Arc<Slot>` interior mutability, not cross-thread coordination.
312    carry: Mutex<VecDeque<i16>>,
313    /// Shared bus counter of mix STARVATION events: a tick where this slot was
314    /// primed (not idle) but its carry came up SHORT of a full chunk — a per-peer
315    /// audible gap. Held as a shared `Arc` so a dropped slot's events still count
316    /// toward the bus total. THIS is the per-peer stutter signal (distinct from the
317    /// output-ring underrun, which is the device-side gap). With the pull model a
318    /// rising count means the JITTER BUFFER itself ran dry (real network loss /
319    /// total starvation), not a producer-scheduling race.
320    starves: Arc<AtomicU64>,
321    /// THIS slot's own cumulative starve count (the shared `starves` above is the
322    /// bus-wide sum). Used only to attribute the aggregate to a source kind
323    /// ([`Self::label`]) so the stutter log can say WHICH audio starves — e.g. a
324    /// screen-share's intermittent app audio vs the remote mic (voice).
325    own_starves: AtomicU64,
326    /// A human label for the slot's source kind ("mic" / "screen_audio"), for the
327    /// per-kind starve breakdown. Static — the call sites pass a string literal.
328    label: &'static str,
329    /// DTX-aware starvation classifier (warmup gate + gap-onset window + recent-energy
330    /// silence discriminator). Touched ONLY by the bus thread each tick, so the mutex
331    /// is uncontended interior mutability over the `Arc<Slot>`, never cross-thread
332    /// coordination. See [`crate::audio::starve`].
333    starve: Mutex<StarveTracker>,
334    /// The control-plane peer id this slot's audio belongs to ([`NO_PEER_TAG`] if
335    /// registered untagged), so a recorded underflow [`AudioGlitch`] can name the peer.
336    peer_tag: u32,
337}

339impl Slot {
340    fn new(
341        starves: Arc<AtomicU64>,
342        playout_floor_us: i64,
343        label: &'static str,
344        peer_tag: u32,
345    ) -> Self {
346        Self {
347            state: Mutex::new(PlayoutState {
348                buffer: PlayoutBuffer::new(playout_floor_us),
349                adapt: AdaptiveJitter::with_floor(playout_floor_us),
350            }),
351            carry: Mutex::new(VecDeque::with_capacity(SLOT_CAPACITY_SAMPLES)),
352            starves,
353            own_starves: AtomicU64::new(0),
354            label,
355            starve: Mutex::new(StarveTracker::new()),
356            peer_tag,
357        }
358    }
359
360    /// A slot with a throwaway starvation counter and a zero playout floor (release
361    /// ASAP) — for the mix/drain unit tests that seed the carry directly.
362    #[cfg(test)]
363    fn unmetered() -> Self {
364        Self::new(Arc::new(AtomicU64::new(0)), 0, "test", NO_PEER_TAG)
365    }
366
367    /// Producer side (pump): fold this frame's arrival into the adaptive jitter depth
368    /// and push it into the jitter buffer, both under one lock so the depth retarget
369    /// and the insert stay consistent. Returns the post-push depth + jitter estimate
370    /// for the caller's per-peer stats. `now_us` is the local receive time and
371    /// `ts_us` the sender's capture timestamp — BOTH on the shared [`CaptureClock`]
372    /// the bus thread pops against.
373    fn push(&self, now_us: i64, ts_us: i64, samples: &[i16]) -> PushStatus {
374        // Own the samples BEFORE taking the lock: this heap allocation must not happen
375        // inside `state.lock()`, which the RT mix pull also takes every 10 ms tick —
376        // an alloc under that lock would let a fill push briefly block the sacred
377        // playout thread.
378        let frame = MixFrame {
379            ts_us,
380            samples: samples.to_vec(),
381        };
382        // `ever_fed` is set when audio first REACHES the carry (the first release
383        // below), NOT here: a frame pushed now sits in the jitter buffer for ~target
384        // ms before it is due, and that intentional warmup delay must not count as
385        // starvation. Setting it on push would flag the whole warmup as a gap.
386        let Ok(mut st) = self.state.lock() else {
387            // poisoned ⇒ a prior op panicked; degrade rather than propagate.
388            return PushStatus {
389                jitter_us: 0,
390                buffered_frames: 0,
391                target_delay_us: 0,
392                recommended_delay_us: 0,
393                late_drops: 0,
394                overflow_drops: 0,
395                underflow_total: 0,
396            };
397        };
398        let PlayoutState { buffer, adapt } = &mut *st;
399        adapt.on_frame(buffer, now_us, ts_us);
400        buffer.push(now_us, frame);
401        PushStatus {
402            jitter_us: adapt.jitter_us(),
403            buffered_frames: buffer.len() as u32,
404            target_delay_us: buffer.target_delay_us(),
405            recommended_delay_us: adapt.recommended_delay_us(),
406            late_drops: buffer.late_drops(),
407            overflow_drops: buffer.overflow_drops(),
408            // The slot's running starve count (updated by the RT pop) — read here so
409            // the per-peer underflow gauge advances without a second lock.
410            underflow_total: self.own_starves.load(Ordering::Relaxed),
411        }
412    }
413
414    /// Bus side (one tick): release every frame whose playout time has arrived (up
415    /// to `now_us + MIX_PULL_LOOKAHEAD_US`) into the carry-queue, then accumulate up
416    /// to `out.len()` carry samples into `out` (summing in i32), leaving the rest of
417    /// `out` untouched. Returns a [`PullOutcome`] (samples contributed + whether this
418    /// was an underflow), so the bus loop can record a per-peer glitch. Bumps the
419    /// shared starvation counter when a primed, non-idle slot comes up short — which,
420    /// under the pull model, means the jitter buffer genuinely had nothing due
421    /// (network starvation), not a producer-scheduling race.
422    fn pull_into(&self, now_us: i64, out: &mut [i32]) -> PullOutcome {
423        // 1) Release due frames from the jitter buffer into the carry (bounded).
424        if let Ok(mut st) = self.state.lock() {
425            let due = st.buffer.pop_due(now_us + MIX_PULL_LOOKAHEAD_US);
426            drop(st); // release the jitter lock before touching the carry
427            if !due.is_empty()
428                && let Ok(mut q) = self.carry.lock()
429            {
430                for frame in due {
431                    // Bulk-append the frame's samples, then trim the oldest past
432                    // capacity in ONE drain (drop-oldest backpressure) — a single
433                    // memcpy + bounds check per frame instead of one per sample.
434                    q.extend(frame.samples);
435                    if q.len() > SLOT_CAPACITY_SAMPLES {
436                        let excess = q.len() - SLOT_CAPACITY_SAMPLES;
437                        q.drain(..excess);
438                    }
439                }
440            }
441        }
442
443        // 2) Drain a chunk from the carry into the mix accumulator, tracking the peak
444        // |sample| released so the starve classifier can tell speech from silence.
445        let want = out.len();
446        let mut peak_abs: i32 = 0;
447        let n = {
448            let Ok(mut q) = self.carry.lock() else {
449                return PullOutcome {
450                    contributed: 0,
451                    starved: false,
452                };
453            };
454            let n = want.min(q.len());
455            // Bulk-drain the oldest n samples into the mix accumulator (sum in i32),
456            // one pass instead of a per-sample pop_front.
457            for (slot, s) in out.iter_mut().zip(q.drain(..n)) {
458                *slot += s as i32;
459                let a = (s as i32).abs();
460                if a > peak_abs {
461                    peak_abs = a;
462                }
463            }
464            n
465        };
466
467        // 3) Starvation accounting (bus thread only) via the DTX-aware classifier: a
468        // dry/short carry counts as audible stutter only if real audio was recently
469        // playing — a silent peer's comfort-noise gaps are not stutter. The tracker
470        // owns the warmup gate + gap-onset window + silence discrimination.
471        let counted = self
472            .starve
473            .lock()
474            .map(|mut tr| {
475                tr.observe(Tick {
476                    contributed: n,
477                    want,
478                    peak_abs,
479                })
480            })
481            .unwrap_or(false);
482        if counted {
483            self.starves.fetch_add(1, Ordering::Relaxed);
484            self.own_starves.fetch_add(1, Ordering::Relaxed);
485        }
486        PullOutcome {
487            contributed: n,
488            starved: counted,
489        }
490    }
491
492    /// Test-only: seed the carry-queue directly (bypassing the jitter buffer) so the
493    /// pure sum/clamp/drain unit tests can drive the accumulator without scheduling.
494    /// The first `pull_into` that drains a sample marks the slot fed (via the
495    /// classifier), so starvation accounting goes live exactly as in production.
496    #[cfg(test)]
497    fn seed_carry(&self, samples: &[i16]) {
498        let mut q = self.carry.lock().unwrap();
499        for &s in samples {
500            if q.len() >= SLOT_CAPACITY_SAMPLES {
501                q.pop_front();
502            }
503            q.push_back(s);
504        }
505    }
506}

508/// A handle a per-peer pipeline holds to push its decoded PCM into the mix. Pushing
509/// is non-blocking and bounded (the jitter buffer + the ~100 ms carry are both
510/// drop-oldest). Dropping the handle unregisters the producer's slot from the bus
511/// (so a torn-down peer stops contributing with no explicit call) — the Drop-safe
512/// teardown the ghost discipline requires.
513pub struct BusProducer {
514    id: u64,
515    slot: Arc<Slot>,
516    registry: Arc<Registry>,
517}
518
519impl BusProducer {
520    /// Push one decoded audio frame into this producer's jitter buffer: `now_us` is
521    /// the local receive time and `ts_us` the sender's capture timestamp (BOTH on the
522    /// session's shared [`CaptureClock`], which the bus thread pops against), and
523    /// `samples` the 48 kHz mono PCM. Folds the arrival into the adaptive jitter depth
524    /// and returns the resulting depth/jitter for per-peer stats. The bus thread
525    /// releases the frame on its own real-time clock — the pump's scheduling never
526    /// affects playout timing.
527    pub fn push(&self, now_us: i64, ts_us: i64, samples: &[i16]) -> PushStatus {
528        self.slot.push(now_us, ts_us, samples)
529    }
530}

532impl Drop for BusProducer {
533    /// Unregister this producer's slot from the bus on drop, so a peer whose
534    /// pipeline ended (or was aborted) stops contributing to the mix without an
535    /// explicit teardown call.
536    fn drop(&mut self) {
537        self.registry.unregister(self.id);
538    }
539}

541/// The set of live producer slots, shared between the bus thread (which iterates
542/// it every tick) and `register`/`unregister` (called from the async subscribe
543/// loop as peers come and go). A `std::sync::Mutex` is right: the critical section
544/// is a `HashMap` insert/remove or a cheap clone of the slot `Arc`s, never held
545/// across IO.
546#[derive(Default)]
547struct Registry {
548    slots: Mutex<HashMap<u64, Arc<Slot>>>,
549}
550
551impl Registry {
552    /// Snapshot the current slots (cheap `Arc` clones) so the tick iterates
553    /// without holding the registry lock across the mix/APM/playback work.
554    fn snapshot(&self) -> Vec<Arc<Slot>> {
555        self.slots
556            .lock()
557            .map(|g| g.values().cloned().collect())
558            .unwrap_or_default()
559    }
560
561    fn unregister(&self, id: u64) {
562        if let Ok(mut g) = self.slots.lock() {
563            g.remove(&id);
564        }
565    }
566}

568/// The mixed-playout bus for one call. Owns the dedicated playback thread (which
569/// owns the single [`CpalPlayback`] and runs the 10 ms mix tick) and the producer
570/// registry. Per-peer pipelines [`register`](Self::register) a [`BusProducer`] and
571/// push their due PCM into it; the bus sums them, feeds the mixed chunk to the APM
572/// render reference once, and plays it.
573///
574/// Dropping the bus signals its thread to stop (the `!Send` `CpalPlayback` is then
575/// dropped on its owning thread) — Drop-safe like the per-peer pipelines.
576pub struct PlayoutBus {
577    registry: Arc<Registry>,
578    next_id: AtomicU64,
579    /// Jitter-buffer depth floor (µs) for every new slot — the latency floor the
580    /// adaptive jitter grows from. Set by the call (`PLAYOUT_TARGET_US`). (The shared
581    /// capture clock itself is owned by the bus thread, not stored here — only the
582    /// thread reads `now()` each tick.)
583    playout_floor_us: i64,
584    stop: Arc<AtomicBool>,
585    /// Cumulative per-peer mix-STARVATION events shared with every [`Slot`] (a
586    /// primed slot that came up short at tick time — the per-peer audible gap). The
587    /// app polls this (diffing per window) to surface a live "mix starvation" stat
588    /// so playout stutter is a measured number, not a vibe. See [`Slot::starves`].
589    mix_starves: Arc<AtomicU64>,
590    /// Cumulative output-ring underrun SAMPLES (the device RT callback played
591    /// silence because the ring ran dry — the device-side gap). Accumulated each
592    /// tick from the sink's [`AudioOutput::take_output_underruns`].
593    output_underruns: Arc<AtomicU64>,
594    /// How LATE each 10 ms mix tick fired versus its schedule (µs), as a windowed
595    /// percentile histogram drained by the stats emitter. This is the RT-health
596    /// signal: if the bus thread is descheduled (e.g. a 60 fps screen-share encode
597    /// hogging the CPU), ticks slip, the device ring drains, and ALL audio stutters —
598    /// a p95 well above ~0 localizes the gap to the feeder thread, not the network.
599    tick_late_lat: Arc<Mutex<hound_media::LatencyHistogram>>,
600    /// Bounded ring of discrete audio-glitch events (drop-oldest), recorded by the bus
601    /// thread at the instant each gap happens and drained by the client each stats
602    /// window into a rolling history the debug report renders as a glitch log.
603    glitches: Arc<Mutex<VecDeque<AudioGlitch>>>,
604    /// The session capture clock (the same `Copy` clock the bus thread stamps glitches
605    /// against), so the client can compute each glitch's age (`now_us − at_us`).
606    clock: CaptureClock,
607    /// Mix-tick counter, surfaced for tests so a caller can confirm the bus keeps
608    /// ticking (silence) with zero producers. Test-only — the live path never
609    /// reads it.
610    #[cfg(test)]
611    ticks: Arc<AtomicU64>,
612}

758    /// kind. Lets the stats loop show WHICH audio is starving — e.g. a screen
759    /// share's intermittent app audio (often a metric artifact, inaudible on the

761    /// authoritative aggregate is [`Self::mix_starves`].

824/// Pull one mixed [`MIX_CHUNK_SAMPLES`]-sample chunk from the snapshot's slots at
825/// local time `now_us`: each slot releases its due jitter-buffered frames (on the
826/// RT clock) into its carry-queue and accumulates a chunk in i32 (no overflow on N
827/// loud peers), then saturating-clamp to i16 (no wrap distortion on overload). A
828/// slot with nothing due contributes silence. Returns the mixed chunk.
829///
830/// Allocation-light (one reused scratch); the unit-testable heart of the bus. Pure
831/// given the slots' state — `now_us` makes the release schedule explicit/testable.
832fn mix_chunk(
833    slots: &[Arc<Slot>],
834    now_us: i64,
835    scratch: &mut [i32; MIX_CHUNK_SAMPLES],
836    glitches: &mut Vec<AudioGlitch>,
837) -> [i16; MIX_CHUNK_SAMPLES] {
838    scratch.fill(0);
839    for slot in slots {
840        let outcome = slot.pull_into(now_us, scratch);
841        if outcome.starved {
842            // Record a per-peer underflow at the moment it happened, attributed to
843            // the slot's peer + source kind (the bus loop drains these into the ring).
844            glitches.push(AudioGlitch {
845                at_us: now_us,
846                peer_tag: slot.peer_tag,
847                label: slot.label,
848                kind: GlitchKind::Underflow,
849                detail_us: 0,
850            });
851        }
852    }
853    let mut out = [0i16; MIX_CHUNK_SAMPLES];
854    for (o, &acc) in out.iter_mut().zip(scratch.iter()) {
855        *o = acc.clamp(i16::MIN as i32, i16::MAX as i32) as i16;
856    }
857    out
858}

860/// Pre-fill a freshly-opened output sink's ring with [`OUTPUT_CUSHION_MS`] of
861/// SILENCE. This establishes the device cushion ONCE, at open, so the steady
862/// one-chunk-per-tick mix keeps the ring ~stable (RT-paced ticks ≈ the device drain
863/// rate) WITHOUT ever draining the per-peer slots faster than real time to refill it.
864/// Silence (not slot audio) means the pre-fill costs nothing from any peer's stream;
865/// it is a fixed ~cushion of added output latency, which AEC absorbs.
866fn prefill_output(playback: &mut Option<Box<dyn AudioOutput>>) {
867    if let Some(p) = playback.as_mut() {
868        let silent = chunk_to_frame(vec![0i16; MIX_CHUNK_SAMPLES]);
869        for _ in 0..(OUTPUT_CUSHION_MS / MIX_TICK_MS) {
870            let _ = p.play(&silent);
871        }
872    }
873}

875/// Arm the one-shot [`super::ROUTE_WARMUP_DELAY`] timer iff the just-opened sink runs at a
876/// non-engine rate (a resampled device — in practice a Bluetooth HFP headset whose
877/// SCO link may not be carrying audio yet). 48 kHz devices return `None` (no warmup
878/// reopen), so the common path is untouched. See [`super::ROUTE_WARMUP_DELAY`].
879fn warmup_reopen_at(playback: &Option<Box<dyn AudioOutput>>, now: Instant) -> Option<Instant> {
880    match playback.as_ref() {
881        Some(p) if p.device_rate() != SAMPLE_RATE.hz() => Some(now + super::ROUTE_WARMUP_DELAY),
882        _ => None,
883    }
884}

886/// Wrap a mixed 10 ms chunk in the [`PcmFrame`] shape the output sink and
887/// `Apm::process_render` take. The timestamp is unused on the render/playout side
888/// (the APM render path ignores it; the cpal ring is time-agnostic), so 0 is fine.
889fn chunk_to_frame(samples: Vec<i16>) -> PcmFrame {
890    PcmFrame::at_engine_rate(samples, TimestampUs::ZERO)
891}

893/// The bus thread body: own the single output sink (opened via `sink_factory`,
894/// with the speaker hot-swap / vanish-fallback / reopen-backoff), and every
895/// [`MIX_TICK`] sum the producers into one chunk, feed it
896/// to the APM render reference once, and play it. Ticks silence with zero
897/// producers so the device stream + AEC reference stay warm. The `!Send` sink
898/// never leaves this thread.
899#[allow(clippy::too_many_arguments)]
900fn mix_bus_loop(
901    stop: Arc<AtomicBool>,
902    mut speaker_rx: watch::Receiver<DeviceChoice<SpeakerId>>,
903    apm: Option<Arc<Apm>>,
904    registry: Arc<Registry>,
905    clock: CaptureClock,
906    ticks: Arc<AtomicU64>,
907    output_underruns: Arc<AtomicU64>,
908    tick_late_lat: Arc<Mutex<hound_media::LatencyHistogram>>,
909    glitch_ring: Arc<Mutex<VecDeque<AudioGlitch>>>,
910    mut sink_factory: SinkFactory,
911) {
912    // Promote this thread to real-time scheduling: it feeds the output device ring on
913    // a 10 ms software clock, and if video-encode load deschedules it the ring drains
914    // and the device plays silence (the under-load playout stutter the instrumentation
915    // showed spiking when a 60 fps screen share starts). The cpal output callback is
916    // already RT; this lets its feeder keep up. Best-effort — a failure just logs and
917    // runs at normal priority (the output cushion still absorbs ordinary jitter). The
918    // handle must stay alive for the thread's lifetime to hold the policy.
919    let _rt_handle = match audio_thread_priority::promote_current_thread_to_real_time(
920        MIX_CHUNK_SAMPLES as u32,
921        SAMPLE_RATE.hz(),
922    ) {
923        Ok(h) => {
924            tracing::info!("voice: mix-bus thread promoted to real-time priority");
925            Some(h)
926        }
927        Err(e) => {
928            tracing::warn!("voice: mix-bus RT priority unavailable ({e:?}) — normal priority");
929            None
930        }
931    };
932
933    let mut choice = speaker_rx.borrow().clone();
934    let mut playback = sink_factory(&choice);
935    prefill_output(&mut playback);
936    // One-shot reopen for a non-48k (BT/HFP) output whose route may not be live yet.
937    let mut warmup_reopen = warmup_reopen_at(&playback, Instant::now());
938    let mut last_open_attempt = Instant::now();
939    // Exponential reopen backoff: doubles from REOPEN_BACKOFF up to
940    // REOPEN_BACKOFF_MAX while the device stays down, resets on a successful
941    // open or a hot-swap. Stops the persistent-failure log/device-churn storm.
942    let mut reopen_backoff = REOPEN_BACKOFF;
943
944    // Reused scratch so the per-tick mix allocates nothing in the i32 accumulate.
945    let mut scratch = [0i32; MIX_CHUNK_SAMPLES];
946    // Reused per-tick glitch buffer — empty in steady state (no alloc), filled only on
947    // the rare tick that has an underflow / late slip, then drained into the ring.
948    let mut tick_glitches: Vec<AudioGlitch> = Vec::new();
949    // Reused wrapper buffer for the mixed chunk fed to the APM + device each tick, so
950    // the RT thread allocates nothing per tick.
951    let mut mixed_buf: Vec<i16> = Vec::with_capacity(MIX_CHUNK_SAMPLES);
952    let mut next_tick = Instant::now() + MIX_TICK;
953
954    while !stop.load(Ordering::Relaxed) {
955        // Pace to the next 10 ms boundary (output-side cadence). Sleeping the
956        // remainder keeps the tick near wall-clock without busy-spinning; the cpal
957        // ring absorbs the residual jitter between this and the DAC.
958        let now = Instant::now();
959        // How late this tick fired vs its 10 ms schedule: 0 when on-time/early, > 0
960        // only when the RT thread was descheduled past its deadline (the device ring
961        // then drains → all audio stutters). Record BEFORE sleeping so the value is
962        // the real slip, not the post-sleep alignment. One uncontended lock/tick,
963        // same discipline as the per-slot locks already on this path.
964        let late_us = now.saturating_duration_since(next_tick).as_micros();
965        if let Ok(mut h) = tick_late_lat.lock() {
966            h.record(late_us.min(u64::MAX as u128) as u64);
967        }
968        if now < next_tick {
969            std::thread::sleep(next_tick - now);
970        }
971        // Advance the schedule; if we fell badly behind (e.g. the thread was
972        // descheduled), re-anchor to now so we don't spin catching up.
973        next_tick += MIX_TICK;
974        if next_tick < Instant::now() {
975            next_tick = Instant::now() + MIX_TICK;
976        }
977
978        // Hot-swap: a changed speaker choice re-opens the single output device in
979        // place (vanish-fallback lives in `CpalPlayback::start`).
980        if speaker_rx.has_changed().unwrap_or(false) {
981            choice = speaker_rx.borrow_and_update().clone();
982            tracing::info!(?choice, "voice: speaker hot-swap requested (mix bus)");
983            // Dropping the old sink releases the device on this (owning)
984            // thread before the new one opens.
985            drop(playback.take());
986            playback = sink_factory(&choice);
987            prefill_output(&mut playback);
988            // A user-picked device may also be a BT headset needing the warmup reopen.
989            warmup_reopen = warmup_reopen_at(&playback, Instant::now());
990            last_open_attempt = Instant::now();
991            reopen_backoff = REOPEN_BACKOFF;
992        }
993
994        // One-shot warmup reopen: a non-48k (BT/HFP) output stream that opened onto a
995        // not-yet-live SCO route plays silence and never errors, so the `is_dead`
996        // reopen below never fires. Reopen it ONCE after the route has had time to
997        // settle (the automatic equivalent of the rejoin that "fixes" it). Not
998        // re-armed afterwards — a single nudge, no reopen loop.
999        if let Some(at) = warmup_reopen
1000            && Instant::now() >= at
1001        {
1002            warmup_reopen = None;
1003            if playback.is_some() {
1004                tracing::info!(
1005                    "voice: warmup reopen of non-48k (BT/HFP) output — first-connect audio fix"
1006                );
1007                drop(playback.take());
1008                playback = sink_factory(&choice);
1009                prefill_output(&mut playback);
1010                last_open_attempt = Instant::now();
1011                reopen_backoff = REOPEN_BACKOFF;
1012            }
1013        }
1014
1015        // Mix EXACTLY ONE chunk per tick on THIS real-time clock: each peer releases
1016        // its due jitter-buffered frames (`pop_due(clock.now() + lookahead)`) and we
1017        // sum one 10 ms chunk + one AEC render reference. This thread is the SOLE
1018        // release authority — the per-peer pumps only FILL the jitter buffers, so no
1019        // non-real-time scheduler sits on the playout path (one there gaps playout
1020        // audibly whenever it is descheduled).
1021        // The device ring is cushioned ONCE at open (a silence pre-fill); thereafter
1022        // the device draining at its hardware clock and this RT-paced one-chunk-per-tick
1023        // push keep it ~stable.
1024        let now_us = clock.now();
1025        let slots = registry.snapshot();
1026        // Mix + collect any per-peer underflow glitches this tick (into the reused
1027        // buffer — empty in steady state).
1028        tick_glitches.clear();
1029        let mixed = mix_chunk(&slots, now_us, &mut scratch, &mut tick_glitches);
1030        // A meaningfully-late tick (the RT feeder slipped) is itself a glitch affecting
1031        // ALL audio — record it once with how late it was. Sub-threshold scheduling
1032        // noise (≲ half a tick) is ignored so the log stays signal.
1033        if late_us >= TICK_LATE_GLITCH_US {
1034            tick_glitches.push(AudioGlitch {
1035                at_us: now_us,
1036                peer_tag: NO_PEER_TAG,
1037                label: "tick",
1038                kind: GlitchKind::LateTick,
1039                detail_us: late_us.min(i64::MAX as u128) as i64,
1040            });
1041        }
1042        if !tick_glitches.is_empty()
1043            && let Ok(mut ring) = glitch_ring.lock()
1044        {
1045            for g in tick_glitches.drain(..) {
1046                if ring.len() >= GLITCH_RING_CAP {
1047                    ring.pop_front();
1048                }
1049                ring.push_back(g);
1050            }
1051        }
1052        // Wrap the mixed chunk for the APM render reference + the device, reusing
1053        // `mixed_buf`'s allocation across ticks (the frame is built from the taken
1054        // buffer and the buffer is reclaimed below) so the RT tick performs no
1055        // per-tick heap allocation.
1056        mixed_buf.clear();
1057        mixed_buf.extend_from_slice(&mixed);
1058        let frame = chunk_to_frame(std::mem::take(&mut mixed_buf));
1059        if let Some(apm) = apm.as_ref() {
1060            apm.process_render(&frame);
1061        }
1062        if let Some(p) = playback.as_mut() {
1063            let _ = p.play(&frame);
1064        }
1065        // Reclaim the buffer for the next tick (no allocator traffic on the RT path).
1066        mixed_buf = frame.samples;
1067        if let Some(p) = playback.as_ref() {
1068            // Fold the device-side gap (silence the RT callback had to insert since
1069            // last tick) into the shared counter the app surfaces.
1070            let under = p.take_output_underruns();
1071            if under > 0 {
1072                output_underruns.fetch_add(under, Ordering::Relaxed);
1073            }
1074            // A dead stream (the output device changed/disconnected) leaves this sink
1075            // object alive but draining nothing — drop it so the reopen path below
1076            // re-opens the current device. Otherwise output stays silent until a
1077            // manual speaker hot-swap.
1078            if p.is_dead() {
1079                tracing::warn!("voice: output stream died (device changed) — reopening");
1080                drop(playback.take());
1081                // Leave `last_open_attempt` alone (it is the last *successful* open,
1082                // long ago for a stream that ran fine) so the reopen path below fires
1083                // on the NEXT tick rather than after a backoff; reset the ladder so a
1084                // genuinely-down device still backs off from the fast cadence.
1085                reopen_backoff = REOPEN_BACKOFF;
1086            }
1087        }
1088        ticks.fetch_add(1, Ordering::Relaxed);
1089
1090        // No live device: retry opening the current choice on an exponential
1091        // backoff (the vanished named speaker falls back to default inside
1092        // `start`). On success reset to the fast cadence; on continued failure
1093        // grow toward the cap so a persistently-held device is a slow heartbeat,
1094        // not a 2 Hz storm.
1095        if playback.is_none()
1096            && !stop.load(Ordering::Relaxed)
1097            && last_open_attempt.elapsed() >= reopen_backoff
1098        {
1099            playback = sink_factory(&choice);
1100            prefill_output(&mut playback);
1101            last_open_attempt = Instant::now();
1102            reopen_backoff = if playback.is_some() {
1103                REOPEN_BACKOFF
1104            } else {
1105                (reopen_backoff * 2).min(REOPEN_BACKOFF_MAX)
1106            };
1107        }
1108    }
1109
1110    // Dropping the sink releases the device on this (owning) thread.
1111    drop(playback);
1112}

1385    /// 20 ms Opus frame at 48 kHz mono.
1386    const FRAME_SAMPLES_20MS: usize = (SAMPLE_RATE.hz() as usize / 1000) * 20;
1387    /// 10 ms synthetic pull tick.
1388    const TICK_US: i64 = MIX_TICK_MS as i64 * 1_000;
1389

1391    const FRAME_TS_US: i64 = FRAME_SAMPLES_20MS as i64 * 1_000_000 / 48_000;

1393    /// Drive a slot: every 10 ms tick, push whatever frames `arrive_at(tick)` says
1394    /// land then (by frame index — frame N has capture ts N·20 ms), then pull a chunk.
1395    /// `peak` sets the level (active vs silence). Returns total starves over `ticks`.
1396    fn run_arrival_schedule(
1397        floor_us: i64,
1398        ticks: usize,
1399        peak: i16,
1400        mut arrive_at: impl FnMut(usize) -> Vec<usize>,

1407            for frame_idx in arrive_at(tick) {
1408                slot.push(now, frame_idx as i64 * FRAME_TS_US, &samples);
1409            }
1410            let mut out = [0i32; MIX_CHUNK_SAMPLES];
1411            let _ = slot.pull_into(now, &mut out);
1412        }
1413        starves.load(Ordering::Relaxed)
1414    }

1540    // ── End-to-end audio playout-chain continuity (the crackle guard) ────────────
1541    //
1542    // Deterministic, sans-IO: drive a peer's audio through the REAL pull-based chain
1543    // — the PUMP pushes 20 ms tone frames into the slot's jitter buffer at their
1544    // (possibly delayed) arrival, and the MIX thread pulls exactly one 10 ms chunk
1545    // per STEADY tick via `pull_into` (the production cadence). Both sides share one
1546    // timeline (the `CaptureClock` invariant). Audio crackle, as a number, is a
1547    // silence/short chunk mid-stream — a `Slot` starvation. The decisive property the
1548    // pull model gives us: the mix's release clock is independent of the pump's push
1549    // timing, so pump jitter/descheduling (the live crackle cause) only delays a FILL,
1550    // which the jitter buffer absorbs — it can NOT race the release. These pin that.
1551
1552    const CHAIN_FRAME_US: i64 = 20_000; // 20 ms Opus frame
1553    const CHAIN_FRAME_SAMPLES: usize = 960; // 20 ms @ 48 kHz mono
1554    const CHAIN_TARGET_US: i64 = 60_000; // jitter cushion (PLAYOUT_TARGET_US)
1555    const CHAIN_TICK_US: i64 = (MIX_TICK_MS as i64) * 1_000; // 10 ms mix tick
1556    const CHAIN_TONE: i16 = 5_000;
1557
1558    /// Drive ONE peer through the real pull chain for `ticks` mix ticks. `arrival(k)`
1559    /// returns the shared-clock time at which the pump PUSHES frame `k` (its ideal
1560    /// arrival is `k·20 ms`; returning later models pump descheduling/jitter). The mix
1561    /// pulls exactly one chunk per steady 10 ms tick. Returns the per-tick mixed chunks
1562    /// and the slot starvation count.
1563    fn run_pull_chain(
1564        ticks: i64,
1565        target_us: i64,
1566        arrival: impl Fn(i64) -> i64,
1567    ) -> (Vec<[i16; MIX_CHUNK_SAMPLES]>, u64) {
1568        let starves = Arc::new(AtomicU64::new(0));
1569        let slot = Arc::new(Slot::new(starves.clone(), target_us, "mic", NO_PEER_TAG));
1570        let mut scratch = [0i32; MIX_CHUNK_SAMPLES];
1571        let slots = [slot.clone()];
1572        let mut out = Vec::new();
1573        let mut next_frame = 0i64;
1574        for t in 0..ticks {
1575            let now = t * CHAIN_TICK_US;
1576            // Pump: push every frame whose (possibly delayed) arrival has come, at its
1577            // arrival time on the shared clock — exactly what the live pump does.
1578            while arrival(next_frame) <= now {
1579                let ts = next_frame * CHAIN_FRAME_US;
1580                slot.push(arrival(next_frame), ts, &[CHAIN_TONE; CHAIN_FRAME_SAMPLES]);
1581                next_frame += 1;
1582            }
1583            // Mix: pull exactly one chunk on the steady RT clock.
1584            out.push(mix_chunk(&slots, now, &mut scratch, &mut Vec::new()));
1585        }
1586        (out, starves.load(Ordering::Relaxed))
1587    }

1600    /// THE core guard: across EVERY frame-vs-tick phase alignment, an on-time stream
1601    /// pulled one-chunk-per-tick never starves and is continuous after warmup. A 20 ms
1602    /// frame straddles two 10 ms ticks, so its due-instant lands at every offset in the
1603    /// tick grid as the phase sweeps; the RT-side lookahead must cover all of them. (The
1604    /// old push-based slot with a 20 ms lead failed exactly here — a tick landing in the
1605    /// inter-frame trough played silence.)
1606    #[test]
1607    fn pull_chain_is_continuous_across_all_phases() {
1608        for phase_ms in 0..20i64 {
1609            let phase = phase_ms * 1_000;
1610            // Every frame (incl. the first, which sets the offset) arrives at ideal +
1611            // phase, so the frame-due phase relative to the tick grid sweeps 0..20 ms.
1612            let (mixed, starves) =
1613                run_pull_chain(200, CHAIN_TARGET_US, |k| k * CHAIN_FRAME_US + phase);
1614            assert_eq!(
1615                starves, 0,
1616                "phase {phase_ms} ms: a steady pulled chain must never starve"
1617            );
1618            // Warmup: first frame due at ~target; +phase; give generous slack.
1619            assert_continuous_after(&mixed, 10);
1620        }
1621    }

1623    /// The pull model's core guarantee, isolated: a non-real-time fill task
1624    /// descheduled under load (modelled as the pump running a CONSTANT delay behind
1625    /// real time — every frame pushed 45 ms late, ≪ the 60 ms buffer) must not gap
1626    /// playout. A *constant* delay is not jitter (the inter-arrival deviation is
1627    /// zero), so the buffer stays at its floor and every frame still beats its
1628    /// stable playout deadline. Under a push model a late pump feeds the slot late
1629    /// → starvation; under pull the RT thread pops the jitter buffer on its own
1630    /// clock, so a consistently-late pump is just a harmless late FILL — ZERO
1631    /// starvation, fully continuous.
1632    #[test]
1633    fn pull_chain_absorbs_constant_pump_delay() {
1634        let (mixed, starves) =
1635            run_pull_chain(200, CHAIN_TARGET_US, |k| k * CHAIN_FRAME_US + 45_000);
1636        assert_eq!(
1637            starves, 0,
1638            "a constantly-descheduled pump must not starve (the crackle root cause)"
1639        );
1640        assert_continuous_after(&mixed, 12);
1641    }

1//! # Acoustic echo cancellation (full-duplex APM)
2//!
3//! Wraps the [`sonora`] audio-processing module (a pure-Rust port of WebRTC's
4//! `modules/audio_processing`: AEC3 + noise suppression + AGC2 + high-pass) into
5//! the live call so speaker-mode calls (no headphones) do not feed the far-end
6//! audio back into the mic: the **render reference** (what we play out) is
7//! subtracted from the **capture** (the mic).
8//!
9//! ## Why `sonora` and not `webrtc-audio-processing`
10//!
11//! `webrtc-audio-processing` (the tonarino C++ binding) has a `-sys` crate that
12//! hard-requires `meson` to build the bundled WebRTC C++, and there is no
13//! system `libwebrtc-audio-processing` to dynamically link. `sonora` is the same
14//! WebRTC algorithms ported to pure Rust: it builds with only the toolchain. It
15//! is **v0.1.0** — newer and less battle-tested than the C++ original — so the
16//! toggle below degrades to pass-through if init ever fails.
17//!
18//! ## The timing contract (the one thing that must be right)
19//!
20//! * **Sample rate / channels:** the whole endpoint is 48 kHz mono i16
21//!   ([`SAMPLE_RATE`]/[`CHANNELS`]); the APM is configured to match, so there is
22//!   no resample on either the capture or render path.
23//! * **Frame size:** the APM's native chunk is **10 ms** (480 samples @ 48 kHz),
24//!   NOT our 20 ms Opus frame (960 samples). Every 20 ms [`PcmFrame`] is split
25//!   into **two** 10 ms passes ([`APM_FRAME_SAMPLES`]) — both for capture
26//!   ([`Apm::process_capture`]) and render ([`Apm::process_render`]). A frame
27//!   whose length is not a whole number of 10 ms chunks is passed through
28//!   unprocessed (defensive; the live path always hands exact 960-sample frames).
29//! * **Stream delay:** AEC3 needs an estimate of the capture-vs-render delay to
30//!   seed its delay search. We set a fixed [`INITIAL_STREAM_DELAY_MS`] once; AEC3
31//!   is adaptive and refines the true delay itself, so the estimate only has to
32//!   be in the right ballpark (it is not a hard alignment requirement like a
33//!   fixed-delay canceller). The render reference is tapped just before playout,
34//!   so the real loop delay is roughly the playout buffer + device output latency.
35//!
36//! ## Threading / locking
37//!
38//! One [`Apm`] is shared (`Arc`) between the **capture thread** (which calls
39//! [`Apm::process_capture`]) and the **mix-bus thread** (which calls
40//! [`Apm::process_render`] on each mixed chunk it plays).
41//! `sonora::AudioProcessing` is `Send` but not `Sync`, so it lives behind a
42//! `Mutex`. The processing calls are short (a few hundred µs for two 10 ms
43//! chunks) and the lock is never held across IO or `.await`. **Neither call
44//! happens inside a cpal RT callback** — capture processing is on the mic worker
45//! thread (between `next_frame` and the encode channel) and render processing is
46//! on the bus thread (between the mix and the playback ring),
47//! so the RT audio callbacks are never blocked on this lock.
48//!
49//! ### Render reference: the single mixed playout bus
50//!
51//! AEC3's render reference must be ONE coherent stream — exactly what the speaker
52//! plays. The [mixed-playout bus](crate::audio::mix) makes it so: all remote
53//! peers' due PCM is summed into one 10 ms chunk and fed to
54//! [`Apm::process_render`] **once per tick**, then played on the single
55//! `CpalPlayback`. Because there is exactly one render feed by construction,
56//! AEC is correct for ANY number of peers — interleaving independent per-peer
57//! references corrupts the canceller. The caller is the bus, not N per-peer
58//! pumps.
59
60use std::sync::Mutex;
61
62use sonora::config::{EchoCanceller, GainController2, NoiseSuppression, NoiseSuppressionLevel};
63use sonora::{AudioProcessing, Config, StreamConfig};
64
65use crate::types::{CHANNELS, PcmFrame, SAMPLE_RATE};
66
67/// Samples in one APM processing chunk: 10 ms @ 48 kHz mono = **480**. This is
68/// the APM's native frame size (`sample_rate / 100`), distinct from the 960-sample
69/// 20 ms Opus frame the rest of the pipeline carries. Every `PcmFrame` is split
70/// into `SAMPLES_PER_FRAME / APM_FRAME_SAMPLES` (= 2) chunks for the APM.
71pub const APM_FRAME_SAMPLES: usize = SAMPLE_RATE.hz() as usize / 100;
72
73/// Initial capture↔render stream-delay estimate handed to AEC3, in milliseconds.
74/// The true delay is roughly the playout jitter target (60 ms) plus the speaker
75/// device's output latency; 70 ms is a sane seed. AEC3 is **delay-adaptive** — it
76/// searches for and tracks the real delay from this seed — so this only has to be
77/// in the right ballpark, not exact. Exposed as a const so the live wiring and the
78/// tests agree on the value.
79pub const INITIAL_STREAM_DELAY_MS: i32 = 70;
80
81/// The live-call audio-processing module: shared echo canceller + noise
82/// suppressor + gain control. Capture frames (mic) and render frames (far-end
83/// playout) are both fed through it; it subtracts the render reference from the
84/// capture so speaker audio does not echo.
85///
86/// One instance is shared (`Arc<Apm>`) across the capture thread and every
87/// per-peer playback pump. Internally a `Mutex<AudioProcessing>` — the WebRTC APM
88/// is `Send` but single-threaded, and capture/render must serialize against each
89/// other anyway (they share the canceller's adaptive state).
90pub struct Apm {
91    inner: Mutex<AudioProcessing>,
92    config: StreamConfig,
93}
94
95impl Apm {
96    /// Build the APM for the endpoint's fixed 48 kHz mono format with AEC3, noise
97    /// suppression (high), and AGC2 (adaptive digital) enabled, seeded with the
98    /// initial stream-delay estimate. Returns `None` if the APM cannot be built
99    /// (so the caller falls back to pass-through and the call still works — AEC
100    /// failing must never break audio).
101    pub fn new() -> Option<Self> {
102        // sonora's builder does not return a Result, but constructing it could in
103        // principle panic on an invalid config; guard so an APM failure degrades to
104        // pass-through rather than taking down the capture thread.
105        let config = StreamConfig::new(SAMPLE_RATE.hz(), CHANNELS.count());
106        let apm_config = Config {
107            // AEC3, default transparent mode.
108            echo_canceller: Some(EchoCanceller::default()),
109            // Background noise suppression. High is a reasonable VoIP default.
110            noise_suppression: Some(NoiseSuppression {
111                level: NoiseSuppressionLevel::High,
112                ..Default::default()
113            }),
114            // AGC2 with adaptive digital gain — levels quiet talkers without the
115            // analog-mic loop (we do not own the OS mic gain here).
116            gain_controller2: Some(GainController2 {
117                adaptive_digital: Some(Default::default()),
118                ..Default::default()
119            }),
120            ..Default::default()
121        };
122
123        let build = std::panic::catch_unwind(std::panic::AssertUnwindSafe(|| {
124            let mut apm = AudioProcessing::builder()
125                .config(apm_config)
126                .capture_config(config)
127                .render_config(config)
128                .build();
129            // Seed the delay estimate; clamping is fine (best-effort).
130            let _ = apm.set_stream_delay_ms(INITIAL_STREAM_DELAY_MS);
131            apm
132        }));
133
134        match build {
135            Ok(apm) => Some(Self {
136                inner: Mutex::new(apm),
137                config,
138            }),
139            Err(_) => {
140                tracing::error!(
141                    target: "audio_endpoint",
142                    "APM init panicked — falling back to pass-through (no AEC)"
143                );
144                None
145            }
146        }
147    }
148
149    /// Process one captured mic [`PcmFrame`] **in place**: AEC (subtract the render
150    /// reference) + noise suppression + gain. The 20 ms frame is split into 10 ms
151    /// APM chunks; each is processed in place. A frame whose length is not a whole
152    /// number of 10 ms chunks is left untouched (defensive — the live path always
153    /// hands exact 960-sample frames). The lock is held only for the short
154    /// processing call, never across IO.
155    pub fn process_capture(&self, frame: &mut PcmFrame) {
156        if !self.chunkable(&frame.samples) {
157            return;
158        }
159        let Ok(mut apm) = self.inner.lock() else {
160            // A poisoned lock means a prior processing call panicked; degrade to
161            // pass-through rather than propagate (audio must keep flowing).
162            return;
163        };
164        let mut scratch = [0i16; APM_FRAME_SAMPLES];
165        for chunk in frame.samples.chunks_mut(APM_FRAME_SAMPLES) {
166            match apm.process_capture_i16(chunk, &mut scratch) {
167                Ok(()) => chunk.copy_from_slice(&scratch),
168                Err(e) => {
169                    // Out-of-range params clamp internally; a real error leaves the
170                    // chunk as captured. Log once-ish at debug (hot path).
171                    tracing::debug!(target: "audio_endpoint", "APM capture chunk error: {e}");
172                }
173            }
174        }
175    }
176
177    /// Feed one render (far-end / playout) [`PcmFrame`] to the APM as the echo
178    /// reference. Called **once per mix tick** by the [mixed-playout
179    /// bus](crate::audio::mix) with the SUMMED 10/20 ms chunk — the single
180    /// coherent stream the speaker plays. The APM accumulates it as the reverse
181    /// stream it subtracts from capture; output is discarded (we do not alter what
182    /// we play, only inform the canceller). Split into 10 ms chunks like capture.
183    ///
184    /// There is exactly one render feed by construction (the bus), so AEC works for
185    /// any number of peers — no per-source gate. (Before the bus, N per-peer pumps
186    /// fed this directly and the interleaved references corrupted AEC3; that gate
187    /// is gone.)
188    pub fn process_render(&self, frame: &PcmFrame) {
189        if !self.chunkable(&frame.samples) {
190            return;
191        }
192        let Ok(mut apm) = self.inner.lock() else {
193            return;
194        };
195        let mut scratch = [0i16; APM_FRAME_SAMPLES];
196        for chunk in frame.samples.chunks(APM_FRAME_SAMPLES) {
197            if let Err(e) = apm.process_render_i16(chunk, &mut scratch) {
198                tracing::debug!(target: "audio_endpoint", "APM render chunk error: {e}");
199            }
200        }
201    }
202
203    /// Whether `samples` is mono and a whole number of 10 ms APM chunks (so it can
204    /// be split cleanly). The live path always satisfies this (960 = 2 × 480);
205    /// anything else is passed through unprocessed.
206    fn chunkable(&self, samples: &[i16]) -> bool {
207        self.config.num_channels() == CHANNELS.count()
208            && !samples.is_empty()
209            && samples.len().is_multiple_of(APM_FRAME_SAMPLES)
210    }
211}

213#[cfg(test)]
214mod tests {
215    use super::*;

217    use crate::types::{SAMPLES_PER_FRAME, TimestampUs};

219    fn frame(samples: Vec<i16>) -> PcmFrame {
220        PcmFrame::at_engine_rate(samples, TimestampUs::ZERO)
221    }
222
223    /// A 960-sample 20 ms frame is exactly two 10 ms APM chunks.
224    #[test]
225    fn frame_splits_into_two_apm_chunks() {
226        assert_eq!(APM_FRAME_SAMPLES, 480);
227        assert_eq!(SAMPLES_PER_FRAME, 960);
228        assert_eq!(SAMPLES_PER_FRAME % APM_FRAME_SAMPLES, 0);
229        assert_eq!(SAMPLES_PER_FRAME / APM_FRAME_SAMPLES, 2);
230    }

232    /// Smoke: init, process synthetic render + capture frames, output stays sane
233    /// (finite, in i16 range, same length — never NaN/garbage or a length change).
234    #[test]
235    fn smoke_process_capture_and_render() {
236        let apm = Apm::new().expect("APM builds headless");
237
238        // A 440 Hz-ish tone as the far-end (render) reference.
239        let render: Vec<i16> = (0..SAMPLES_PER_FRAME)
240            .map(|i| ((i as f32 * 0.1).sin() * 8000.0) as i16)
241            .collect();
242        apm.process_render(&frame(render));
243
244        // Mic capture: some near-end speech-ish signal.
245        let mut cap = frame(
246            (0..SAMPLES_PER_FRAME)
247                .map(|i| ((i as f32 * 0.05).sin() * 4000.0) as i16)
248                .collect(),
249        );
250        let before_len = cap.samples.len();
251        apm.process_capture(&mut cap);
252
253        assert_eq!(cap.samples.len(), before_len, "frame length preserved");
254        // Output is real i16 (no panic, no garbage). RMS must be finite and within
255        // full scale — a sane processed frame, not silence-or-overflow.
256        let rms = cap.rms();
257        assert!(rms.is_finite(), "rms finite");
258        assert!((0.0..=1.0).contains(&rms), "rms within full scale: {rms}");
259    }

261    /// A non-chunkable frame (not a multiple of 480) is passed through unchanged.
262    #[test]
263    fn passes_through_odd_length() {
264        let apm = Apm::new().expect("APM builds headless");
265        let mut f = frame(vec![1234i16; 100]);
266        let before = f.samples.clone();
267        apm.process_capture(&mut f);
268        assert_eq!(f.samples, before, "odd-length frame untouched");
269    }

271    /// Echo-shaped test: when the captured signal is a delayed/attenuated copy of
272    /// the render reference (pure echo, no near-end voice), the APM should
273    /// measurably attenuate the captured energy as AEC3 converges. The canceller is
274    /// adaptive, so we feed many frames and assert the LATE frames are quieter than
275    /// the captured echo — with a generous bound (AEC3 needs convergence time and
276    /// this is a synthetic single-tap echo, not a real room).
277    #[test]
278    fn echo_reference_attenuates_capture() {
279        let apm = Apm::new().expect("APM builds headless");
280
281        // Far-end reference: a steady tone.
282        let make_tone = |phase: f32| -> Vec<i16> {
283            (0..SAMPLES_PER_FRAME)
284                .map(|i| (((i as f32 + phase) * 0.12).sin() * 10000.0) as i16)
285                .collect()
286        };
287
288        let mut captured_rms_late = 0.0f32;
289        let echo_input_rms = frame(make_tone(0.0)).rms();
290        let frames = 200; // ~4 s of audio — enough for AEC3 to converge.
291        for n in 0..frames {
292            let tone = make_tone(n as f32);
293            // Render the far end.
294            apm.process_render(&frame(tone.clone()));
295            // Capture == an attenuated copy of the far end (pure echo, no near-end).
296            let mut echo = frame(tone.iter().map(|&s| (s as i32 / 2) as i16).collect());
297            apm.process_capture(&mut echo);
298            if n >= frames - 20 {
299                captured_rms_late += echo.rms();
300            }
301        }
302        captured_rms_late /= 20.0;
303
304        // The processed echo (late frames) should be clearly below the raw echo
305        // input level. Half-amplitude echo has rms ≈ echo_input_rms/2; a converged
306        // AEC should push the residual well under that. Generous bound: at least
307        // some suppression below the un-attenuated reference level.
308        assert!(
309            captured_rms_late < echo_input_rms * 0.5,
310            "AEC should suppress the echo: late rms {captured_rms_late} vs reference {echo_input_rms}"
311        );
312    }

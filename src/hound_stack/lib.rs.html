1//! `hound-stack`: the dev/CI stack lifecycle library backing the
2//! `hound-stack` CLI (`src/main.rs`). One implementation shared by every
3//! caller that needs "bring the dev backend stack up, wait for it to be
4//! healthy, tear it down" — the `e2e` and `visual` subcommands, plus the
5//! `up`/`down` manual-dev commands. Replaces the hand-synced
6//! `tools/houndbot/e2e.sh` / `ui/integration_test/visual.sh` shell scripts.
7//!
8//! Noise contract (every flow): the console gets one line per lifecycle
9//! step; every subprocess's full output lands in a separated artifact file
10//! (`build.log` for all build phases, `compose.log`, `backend.log`,
11//! `sfu.log`, bot logs, test output); binaries are built quietly first and
12//! then exec'd so runtime logs hold runtime lines only; a failure surfaces
13//! the relevant log's tail inline.
14//!
15//! Module map:
16//! - [`ports`] — host port preflight (fail fast on a leftover/concurrent run).
17//! - [`compose`] — the `docker compose`/`docker-compose`/mise fallback chain,
18//!   output to `compose.log`.
19//! - [`wait`] — deadline-bounded polling (HTTP reachability, container
20//!   health, log lines) — no bare sleeps beyond the poll interval.
21//! - [`cargo`] — quiet builds to `build.log`; runtime logs never see
22//!   compiler chatter.
23//! - [`logs`] — log sectioning + failure-tail helpers.
24//! - [`backend`] — spawns the (prebuilt) host backend with the shared dev env.
25//! - [`sfu`] — forces the SFU's one-shot registration POST to re-land.
26//! - [`artifacts`] — `E2E_OUT_DIR`/`VISUAL_OUT_DIR`-style output dir
27//!   resolution.
28//! - [`tee`] — runs a child process, teeing its output to console + a log
29//!   file (the Rust equivalent of `cmd | tee log`).
30//! - [`lifecycle`] — [`lifecycle::LiveStack`]: bring-up plus idempotent,
31//!   drop-safe teardown (partial bring-up failures included), composing the
32//!   modules above.
33//! - [`e2e`] — the `hound-testkit` scenario suite flow.
34//! - [`visual`] — the visual snapshot harness flow (bootstraps via
35//!   `hound_testkit::Env` rather than re-implementing it in shell).
36//! - [`updown`] — manual `up`/`down` dev commands.

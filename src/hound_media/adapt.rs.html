1//! # Sender-side video adaptation (the BWE → encoder glue)
2//!
3//! The **deterministic, sans-IO** core that turns the signals a publisher can
4//! observe *without any new transport surface* into encoder decisions. It wraps
5//! three `hound_media` pieces and is the single place the cam-publish path
6//! consults each tick:
7//!
8//! * [`RateController`] — loss-based AIMD BWE. Produces a target send bitrate.
9//! * [`QualityLadder`] — maps that bitrate to a discrete video [`Rung`] (or
10//!   [`Quality::AudioOnly`] when even the lowest rung won't fit). *Audio is
11//!   sacred; video is the release valve.*
12//! * [`KeyframeThrottle`] — coalesces a PLI storm into one IDR and spaces forced
13//!   keyframes so a burst of receiver requests can't spike the bitrate.
14//!
15//! ## Why these signals
16//!
17//! Real BWE wants receiver-reported loss. [`VideoAdaptation`] folds three
18//! signals into one loss-equivalent each tick:
19//!
20//! * **Encoder/publisher backpressure.** The capture→publish channel is bounded;
21//!   when the encoder outruns the publisher (a slow/congested uplink) frames are
22//!   dropped at `try_send`. The dropped-fraction over an interval is a direct,
23//!   local overload signal — it behaves like loss for the AIMD controller.
24//! * **RTT.** `iroh::endpoint::Connection::rtt` is a cheap delay signal. A sharp
25//!   rise over the connection's own baseline is bufferbloat — congestion before
26//!   outright drops — so we fold an RTT-inflation term into the loss-equivalent.
27//! * **Receiver-reported loss.** The receiver→producer feedback loop (the
28//!   `MediaFeedback`/`MediaFeedbackReceived` control messages) reports the loss
29//!   fraction each subscriber observes on the downstream leg the relay hides —
30//!   the one signal the sender cannot see locally. When several receivers report,
31//!   the orchestrator pre-reduces to the **worst** leg (serve the worst
32//!   receiver) and passes that single fraction in.
33//!
34//! All three collapse to a single **loss-equivalent fraction** fed to the AIMD
35//! controller, keeping the controller's contract (one loss number per interval)
36//! intact. The mapping is the only policy here.
37//!
38//! ## Determinism
39//!
40//! Everything here is pure: time enters as `now_us`, every signal is an explicit
41//! input, and the only outputs are returned data ([`AdaptDecision`]). No clocks,
42//! no IO, no threads. The orchestrator samples the live signals and feeds them
43//! in; this core decides. That keeps the policy unit-testable with synthetic
44//! signals and no devices (see the tests below).
45

51/// How an RTT rise maps to loss-equivalent. Tuned conservatively: the delay
52/// signal nudges the controller toward backing off *before* drops appear, but
53/// drops (real overload) dominate when both fire.
54#[derive(Debug, Clone)]
55pub struct AdaptConfig {
56    /// RTT (µs) at or below which the delay term contributes nothing — the
57    /// connection's healthy floor. Anything under this is "no congestion".
58    pub rtt_floor_us: i64,
59    /// RTT (µs) inflation above the observed baseline at which the delay term
60    /// saturates to `rtt_max_loss`. Between floor and this it scales linearly.
61    pub rtt_inflation_span_us: i64,
62    /// Maximum loss-equivalent the delay term alone can contribute. Capped below
63    /// the controller's `decrease_loss` so RTT alone *nudges* (holds/slows climb)
64    /// but only real drops force the hard multiplicative back-off.
65    pub rtt_max_loss: f64,
66}
67
68impl Default for AdaptConfig {
69    fn default() -> Self {
70        Self {
71            rtt_floor_us: 50_000,           // 50 ms — healthy relay RTT
72            rtt_inflation_span_us: 150_000, // +150 ms over baseline ⇒ saturated
73            rtt_max_loss: 0.08,             // below default decrease_loss (0.10)
74        }
75    }
76}

78/// The signals one publish tick observes. Built by the orchestrator from the
79/// bounded-channel drop counters, the iroh connection RTT, and (when receivers
80/// report) the worst receiver-reported loss fraction this interval.
81#[derive(Debug, Clone, Copy, Default)]
82pub struct AdaptSignals {
83    /// Encoded access units produced since the last tick.
84    pub produced: u32,
85    /// Of those, how many were dropped at the bounded publish channel
86    /// (`try_send` failed) — the encoder outran the publisher.
87    pub dropped: u32,
88    /// Current connection RTT in microseconds, if the transport exposes it.
89    pub rtt_us: Option<i64>,
90    /// Worst receiver-reported loss fraction (0.0..=1.0) across all reporters this
91    /// interval, or `None` when no receiver reported. This is the one signal the
92    /// sender cannot observe locally — loss on a downstream leg the relay hides.
93    /// The orchestrator already reduced multiple reporters to the max (serve the
94    /// worst leg), so this is a single pre-reduced fraction.
95    pub remote_loss: Option<f64>,
96}
97
98impl AdaptSignals {
99    /// The drop fraction this tick (0.0 when nothing was produced). This is the
100    /// primary, most direct overload signal.
101    fn drop_fraction(&self) -> f64 {
102        if self.produced == 0 {
103            0.0
104        } else {
105            (self.dropped as f64 / self.produced as f64).clamp(0.0, 1.0)
106        }
107    }
108}

110/// What the encoder/publisher should do after a tick. Pure data — the
111/// orchestrator applies it (calls `set_bitrate`, pauses/resumes publish,
112/// forces a keyframe).
113#[derive(Debug, Clone, Copy, PartialEq)]
114pub struct AdaptDecision {
115    /// The BWE target send bitrate (bits/sec). The chosen rung's bitrate when on
116    /// video; the controller target when audio-only (so the climb-back is smooth).
117    pub target_bps: u64,
118    /// The selected quality. [`Quality::AudioOnly`] means pause cam encode/publish.
119    pub quality: Quality,
120    /// Whether the *quality* changed from the previous decision (rung up/down or
121    /// the AudioOnly⇄Video flip). The orchestrator only calls `set_bitrate` /
122    /// (un)pauses on a change, so a steady call costs nothing.
123    pub quality_changed: bool,
124}
125
126impl AdaptDecision {
127    /// Whether video is paused under this decision (AudioOnly).
128    pub fn is_audio_only(&self) -> bool {
129        matches!(self.quality, Quality::AudioOnly)
130    }
131}

133// ─── Encoder-rebuild geometry dwell ─────────────────────────────────────────────
134//
135// The follow-up to the resolution-adaptation hotfix: the sender's `VideoEncoder`
136// is opened once at a fixed geometry, so honoring a rung's resolution (not just
137// its bitrate/fps) means tearing down and rebuilding the encoder when the chosen
138// rung's geometry changes. That is expensive (a real device open/close) and
139// visible (forces a fresh IDR — a decoder can't switch resolution mid-GOP, so
140// every receiver must resync), so unlike bitrate/fps — which the SAME encoder
141// retargets for free every tick via `AdaptDecision::target_bps` /
142// `AdaptShared::target_fps` — a resolution change must not flap with ordinary
143// AIMD/delay-BWE noise. [`GeometryDwell`] is the dedicated hysteresis for that
144// one decision; the IO shell (the capture/encode loop) drives it independently of
145// [`VideoAdaptation::on_tick`], mirroring the `CodecSwitch`/`LayerHysteresis`
146// dwell pattern already used for the codec-switch and receive-side layer-switch
147// decisions (`crates/client/src/call/policy.rs`) — kept here instead so the
148// decision stays sans-IO and unit-tested in this crate.
149
150/// How long a candidate encode geometry must hold, unchanged, before the encoder
151/// actually rebuilds at it. Without a dwell, ordinary AIMD noise — or the
152/// delay-based estimator pinning low for a beat before recovering — would rebuild
153/// the encoder every few seconds. ~5 ticks at the 1 s adaptation cadence the
154/// client drives [`VideoAdaptation::on_tick`] at.
155pub const GEOMETRY_REBUILD_DWELL_US: i64 = 5_000_000;
156
157/// Make `v` even and at least 2 (encoders need even, non-zero luma extents).
158fn even_min2(v: u32) -> u32 {
159    (v & !1).max(2)
160}
161
162/// Fit a rung's target HEIGHT to the SOURCE capture's own aspect ratio, even-
163/// clamped. The camera and screen quality ladders both store a nominal 16:9
164/// `width` on each [`Rung`] purely for the bitrate model (see [`screen_rungs`]) —
165/// a screen source is not always 16:9 (a portrait phone mirror, a near-square
166/// shared window), so encoding at the rung's raw width would stretch it. Deriving
167/// the width from the source's own aspect instead — the same approach the low
168/// simulcast layer uses for its fixed low-resolution encode — keeps every encode
169/// at the right aspect regardless of what the ladder's rung table says. For the
170/// camera ladder, whose capture geometry and rungs are both already 16:9, this
171/// reproduces the rung's own width exactly.
172///
173/// `target_height` is clamped to never exceed the source's own height: a small
174/// source (a shared window narrower than the ladder's floor rung) must never be
175/// UPSCALED just because a low rung nominally asks for more pixels than the
176/// source has — the pre-adaptation code had the same "never upscale past capture"
177/// invariant (`w.min(capture_dims.width())`), and it still applies now that the
178/// width is aspect-derived instead of taken from the rung directly.
179fn fit_encode_dims(source: (u32, u32), target_height: u32) -> (u32, u32) {
180    let source_h = source.1.max(1);
181    let target_height = target_height.max(2).min(source_h);
182    let scale = target_height as f64 / source_h as f64;
183    let width = ((source.0.max(1) as f64) * scale).round().max(2.0) as u32;
184    (even_min2(width), even_min2(target_height))
185}

187/// Dwell-based hysteresis for the encoder-REBUILD geometry decision (pure,
188/// sans-IO — see [`GEOMETRY_REBUILD_DWELL_US`] for why this must not track the
189/// per-tick [`Rung`] directly). [`Self::observe`] feeds the freshly-decided rung
190/// HEIGHT each tick (`None` for [`Quality::AudioOnly`] — no video decision, so
191/// the last committed geometry carries through unchanged; encode is paused
192/// anyway) and returns the geometry the encoder should be built at right now:
193/// unchanged unless the SAME candidate has held for [`GEOMETRY_REBUILD_DWELL_US`]
194/// continuously — any intervening DIFFERENT candidate (a still-settling climb, or
195/// genuine flapping) resets the timer, so only a genuinely stable rung ever
196/// triggers a rebuild.
197#[derive(Debug, Clone)]
198pub struct GeometryDwell {
199    source: (u32, u32),
200    committed: (u32, u32),
201    pending: Option<((u32, u32), i64)>,
202}
203
204impl GeometryDwell {
205    /// `source` is the capture's own geometry — pass the encoder's ACTUAL
206    /// geometry at construction (the capture loop always opens its encoder at
207    /// native/source geometry before any adaptation decision exists), so the
208    /// first tick at the top rung is a no-op rather than a spurious rebuild.
209    pub fn new(source: (u32, u32)) -> Self {
210        let source = (source.0.max(1), source.1.max(1));
211        Self {
212            source,
213            committed: (even_min2(source.0), even_min2(source.1)),
214            pending: None,
215        }
216    }
217
218    /// Advance the dwell state one tick. `target_height` is the rung's height
219    /// this tick decided (`None` = [`Quality::AudioOnly`]); `now_us`/`dwell_us`
220    /// mirror the `CodecSwitch`/`LayerHysteresis` debounce contract elsewhere in
221    /// this codebase. Returns the committed `(width, height)` the encoder should
222    /// be built at.
223    pub fn observe(
224        &mut self,
225        target_height: Option<u32>,
226        now_us: i64,
227        dwell_us: i64,
228    ) -> (u32, u32) {
229        let Some(h) = target_height else {
230            return self.committed;
231        };
232        let want = fit_encode_dims(self.source, h);
233        if want == self.committed {
234            self.pending = None;
235            return self.committed;
236        }
237        match self.pending {
238            Some((p, since)) if p == want && now_us.saturating_sub(since) >= dwell_us => {
239                self.committed = want;
240                self.pending = None;
241            }
242            Some((p, _)) if p == want => {}
243            _ => self.pending = Some((want, now_us)),
244        }
245        self.committed
246    }
247
248    /// The currently-committed encode geometry (what the encoder should already
249    /// be built at).
250    pub fn committed(&self) -> (u32, u32) {
251        self.committed
252    }
253}

255/// Sender-side video adaptation: the BWE controller, the quality ladder, and the
256/// keyframe throttle behind one sans-IO façade.
257#[derive(Debug)]
258pub struct VideoAdaptation {
259    cfg: AdaptConfig,
260    rate: RateController,
261    ladder: QualityLadder,
262    keyframe: KeyframeThrottle,
263    /// Lowest RTT yet observed — the connection's own healthy baseline. The delay
264    /// term measures *inflation* above this, so a high-but-stable RTT link (a far
265    /// relay) is not perpetually penalised; only growth signals congestion.
266    rtt_baseline_us: Option<i64>,
267    /// The last decided quality, so [`Self::on_tick`] can report `quality_changed`
268    /// and the orchestrator only acts on transitions.
269    last_quality: Quality,
270}
271
272impl VideoAdaptation {
273    /// Build the adaptation stack for a camera publisher. `start_bps` seeds the
274    /// rate controller (typically the static cam target the encoder launched at)
275    /// and the ladder ceiling caps it at the publisher's native resolution rung
276    /// so we never try to upscale past what the camera produces.
277    pub fn new(start_bps: u64, ladder: QualityLadder, cfg: AdaptConfig) -> Self {
278        let rate = RateController::new(RateConfig {
279            start_bps,
280            // Derive the BWE ceiling from the ladder's top rung (+ climb margin)
281            // rather than a flat default, so the controller can climb all the way
282            // to the richest rung on a fast link — e.g. a screen share's 1440p120
283            // needs ~20 Mbps, which a 4 Mbps default would strand.
284            max_bps: ladder.bwe_ceiling_bps(),
285            ..Default::default()
286        });
287        Self {
288            cfg,
289            rate,
290            ladder,
291            keyframe: KeyframeThrottle::new(KeyframeThrottleConfig::default()),
292            rtt_baseline_us: None,
293            last_quality: Quality::Video(Rung {
294                // Placeholder until the first tick; `last_quality` only seeds the
295                // change-detector and is overwritten on the first `on_tick`.
296                bitrate_bps: start_bps,
297                width: 0,
298                height: 0,
299                fps: 0,
300            }),
301        }
302    }
303
304    /// The standard camera ladder for the common case. `cam_bitrate_bps` is the
305    /// encoder's static launch rate, used to seed the rate controller. The ladder's
306    /// top rung is 720p — exactly the cam capture config — so no ceiling cap is
307    /// needed: the climb naturally tops out at what the camera produces.
308    pub fn standard(cam_bitrate_bps: u64) -> Self {
309        Self::new(
310            cam_bitrate_bps,
311            QualityLadder::standard(),
312            AdaptConfig::default(),
313        )
314    }
315
316    /// A screen-share ladder for the sharer's [`ScreenQuality`] — their
317    /// motion↔legibility priority plus max-resolution/fps caps. `screen_bitrate_bps`
318    /// is the encoder's static launch rate, used to seed the rate controller. The
319    /// default `ScreenQuality` is the legibility ladder (drop **fps before
320    /// resolution** — the opposite trade-off to [`Self::standard`]). Everything else
321    /// (the AIMD controller, the RTT/backpressure/remote-loss fold, the keyframe
322    /// throttle) is identical to the camera path; only the bitrate→rung map differs.
323    pub fn screen_with(screen_bitrate_bps: u64, quality: ScreenQuality) -> Self {
324        Self::new(
325            screen_bitrate_bps,
326            QualityLadder::screen_for(quality),
327            AdaptConfig::default(),
328        )
329    }
330
331    /// Swap the quality ladder live when the sharer changes their [`ScreenQuality`]
332    /// on an active share (motion↔legibility, or a resolution/fps cap). Only the
333    /// bitrate→rung mapping changes: the AIMD controller's current estimate is
334    /// preserved, so the next [`Self::on_tick`] re-picks a rung from the new ladder
335    /// at the live bitrate (and writes the new `target_fps`/geometry downstream).
336    /// No encoder touch here — the capture thread reacts on the next tick.
337    pub fn set_screen_quality(&mut self, quality: ScreenQuality) {
338        self.ladder.reconfigure(screen_rungs(quality));
339        // The new ladder may top out higher (Source/120) or lower (a 480p cap), so
340        // move the controller ceiling to match — otherwise a richer top rung stays
341        // unreachable, or a stale high estimate over-drives a now-capped ladder.
342        self.rate.set_max_bps(self.ladder.bwe_ceiling_bps());
343    }
344
345    /// Fold this tick's signals into a single loss-equivalent for the AIMD
346    /// controller. Drops dominate; RTT inflation adds a capped nudge. Pure.
347    fn loss_equivalent(&mut self, sig: &AdaptSignals) -> f64 {
348        let drop_loss = sig.drop_fraction();
349
350        let rtt_loss = match sig.rtt_us {
351            Some(rtt) => {
352                // Track the healthy baseline (the min RTT seen), but never below
353                // the configured floor so a momentary sub-floor reading can't
354                // make every later sample look inflated.
355                let baseline = self
356                    .rtt_baseline_us
357                    .map(|b| b.min(rtt))
358                    .unwrap_or(rtt)
359                    .max(self.cfg.rtt_floor_us);
360                self.rtt_baseline_us = Some(baseline);
361
362                let inflation = (rtt - baseline).max(0);
363                if inflation == 0 || self.cfg.rtt_inflation_span_us <= 0 {
364                    0.0
365                } else {
366                    let frac = inflation as f64 / self.cfg.rtt_inflation_span_us as f64;
367                    (frac.clamp(0.0, 1.0)) * self.cfg.rtt_max_loss
368                }
369            }
370            None => 0.0,
371        };
372
373        // Receiver-reported loss on the downstream leg — already reduced to the
374        // worst reporter by the orchestrator. Clamp defensively.
375        let remote_loss = sig.remote_loss.unwrap_or(0.0).clamp(0.0, 1.0);
376
377        // The signals are not independent observations of the *same* loss, so
378        // take the strongest rather than summing (summing would double-count a
379        // congested link that both inflates RTT and drops, or a downstream leg
380        // whose loss already shows up as local backpressure). The max is also the
381        // right policy across signals that observe *different* legs: whichever
382        // bottleneck is worst is the one we must back off to clear — and for
383        // multiple receivers, the worst leg is already folded into remote_loss.
384        drop_loss.max(rtt_loss).max(remote_loss)
385    }
386
387    /// Advance the adaptation one tick: fold the signals into a loss-equivalent,
388    /// run the AIMD controller, pick a ladder rung, and report the decision. The
389    /// orchestrator applies the result (set encoder bitrate, pause/resume, etc.).
390    pub fn on_tick(&mut self, sig: &AdaptSignals, now_us: i64) -> AdaptDecision {
391        let loss = self.loss_equivalent(sig);
392        let target = self.rate.on_feedback(loss, now_us);
393        let quality = self.ladder.update(target);
394
395        let quality_changed = quality != self.last_quality;
396        self.last_quality = quality;
397
398        let target_bps = match quality {
399            Quality::Video(rung) => rung.bitrate_bps,
400            // Audio-only: report the controller target (not 0) so the consumer can
401            // see the recovery headroom building, and resume promptly when it
402            // climbs back over the lowest rung.
403            Quality::AudioOnly => target,
404        };
405
406        AdaptDecision {
407            target_bps,
408            quality,
409            quality_changed,
410        }
411    }

413    /// A receiver PLI arrived for our track (relayed via control). Coalesced —
414    /// many PLIs before the next [`Self::poll_keyframe`] still mean one IDR.
415    pub fn on_pli(&mut self) {
416        self.keyframe.on_pli();
417    }
418
419    /// Whether a forced keyframe is due now (a pending PLI and the min interval
420    /// elapsed). `true` at most once per interval, coalescing a storm into one
421    /// IDR. The orchestrator calls `encoder.request_keyframe()` on `true`.
422    pub fn poll_keyframe(&mut self, now_us: i64) -> bool {
423        self.keyframe.should_generate(now_us)
424    }
425
426    /// Record that a keyframe was emitted (forced or the encoder's own periodic
427    /// one) so the throttle measures its interval from the real emission and a
428    /// pending request is satisfied without a redundant forced IDR.
429    pub fn on_keyframe_emitted(&mut self, now_us: i64) {
430        self.keyframe.on_keyframe_emitted(now_us);
431    }

433    /// The last decided quality (a faithful projection of the last tick — it
434    /// reports `AudioOnly` iff that was the last decision, never a phantom video
435    /// rung). The orchestrator drives off [`AdaptDecision`] returned by
436    /// [`Self::on_tick`]; this getter rounds out the contract and is exercised by
437    /// the projection-faithfulness tests below. Test-only — the live path never
438    /// reads it.
439    #[cfg(test)]

441        self.ladder.current()
442    }

449    /// Steady, healthy call: no drops, flat RTT at the floor. The controller
450    /// climbs slowly (AIMD up), never backs off, and video stays up.
451    #[test]
452    fn healthy_link_climbs_and_keeps_video() {
453        let mut a = VideoAdaptation::standard(600_000);
454        let sig = AdaptSignals {
455            produced: 30,
456            dropped: 0,
457            rtt_us: Some(40_000), // below the 50 ms floor ⇒ no delay penalty
458            remote_loss: None,
459        };
460        let mut now = 0i64;
461        let first = a.on_tick(&sig, now);
462        assert!(
463            matches!(first.quality, Quality::Video(_)),
464            "a healthy link must keep video"
465        );
466        // Pump several intervals; the target must be non-decreasing (no back-off).
467        let mut last = first.target_bps;
468        for _ in 0..10 {
469            now += 1_000_000;
470            let d = a.on_tick(&sig, now);
471            assert!(matches!(d.quality, Quality::Video(_)));
472            assert!(
473                d.target_bps >= last || matches!(d.quality, Quality::Video(_)),
474                "healthy link should not force a back-off"
475            );
476            last = d.target_bps;
477        }
478    }

660    /// Receiver-reported loss with NO local signals (no drops, healthy RTT) must
661    /// still drive the controller down — this is the whole point of the feedback
662    /// loop: the sender is blind to a lossy downstream leg, and only the receiver
663    /// report reveals it. A sustained heavy remote loss collapses to AudioOnly.
664    #[test]
665    fn remote_loss_alone_drives_bitrate_down() {
666        let mut a = VideoAdaptation::standard(600_000);
667        // Establish a clean baseline (no drops, RTT at the floor).
668        let clean = AdaptSignals {
669            produced: 30,
670            dropped: 0,
671            rtt_us: Some(40_000),
672            remote_loss: None,
673        };
674        let baseline = a.on_tick(&clean, 0).target_bps;
675
676        // Now a receiver reports sustained 40% loss while every LOCAL signal stays
677        // healthy. With only local signals the controller would climb; the remote
678        // report alone must force it down instead.
679        let lossy = AdaptSignals {
680            produced: 30,
681            dropped: 0,
682            rtt_us: Some(40_000), // no bufferbloat locally
683            remote_loss: Some(0.40),
684        };
685        let mut now = 1_000_000i64;
686        let mut saw_audio_only = false;
687        let mut min_target = baseline;
688        for _ in 0..30 {
689            let d = a.on_tick(&lossy, now);
690            min_target = min_target.min(d.target_bps);
691            if d.is_audio_only() {
692                saw_audio_only = true;
693            }
694            now += 1_000_000;
695        }
696        assert!(
697            min_target < baseline,
698            "remote loss alone must walk the target down (baseline {baseline}, min {min_target})"
699        );
700        assert!(
701            saw_audio_only,
702            "sustained 40% remote loss must degrade to AudioOnly with no local signals"
703        );
704    }
